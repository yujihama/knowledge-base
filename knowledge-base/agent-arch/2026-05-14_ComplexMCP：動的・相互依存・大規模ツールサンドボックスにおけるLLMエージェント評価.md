## 要約

現状のLLMエージェントは独立したAPI呼び出しは得意だが、商用ソフトウェア自動化の「ラストマイル」——ツールが相互依存し、ステートフルで、環境ノイズが入る現実の業務フロー——では大きく性能が落ちる。本論文はその課題を定量化するベンチマーク「ComplexMCP」を提案する。

**設計**: Model Context Protocol（MCP）上に構築し、オフィススイートから金融システムまで7つのステートフルサンドボックスから計300以上のツールを提供。シード駆動アーキテクチャにより環境状態を動的に変化させ、APIのランダム障害を再現しながらも決定論的な評価を保証する。

**評価結果**: GPT-4系・Claude系を含む複数の最上位LLMを、フルコンテキスト方式とRAG方式の両パラダイムで評価。最高性能モデルでも成功率60%未満にとどまり、人間の90%に大きく及ばない。

**3つのボトルネック**: トラジェクトリの粒度分析で以下が特定された。①「ツール検索飽和（tool retrieval saturation）」——アクション空間が拡大するにつれ適切なツールを選べなくなる現象。②「過信（over-confidence）」——エージェントが必須の環境確認（前提条件チェック）をスキップして先に進む傾向。③「戦略的敗北主義（strategic defeatism）」——回復を試みる代わりに失敗を合理化して諦めてしまう傾向。

**監査エージェントへの示唆**: 監査ワークフローも「前工程の結果を受け取り・検証し・次工程に渡す」という相互依存ツールチェーンで構成される。ComplexMCPが示す「過信」と「戦略的敗北主義」は、監査エージェントが証跡確認や例外処理でも同様に陥りうるリスクであり、ガードレール設計（環境検証ステップの強制、リトライ戦略の明示）が不可欠と言える。RAGパラダイムでの評価結果は、大規模ツールカタログを持つ監査システムでのツール選択精度改善の参考にもなる。

## アイデア

- シード駆動アーキテクチャでAPIランダム障害を再現しつつ決定論的評価を保証するという設計は、再現性と現実性を両立する評価インフラのモデルケースになる
- 「戦略的敗北主義」という失敗モードの命名と分析は、エージェントのリカバリー戦略設計（リトライ・フォールバック・人間へのエスカレーション）を議論する際の共通語彙として有用
- フルコンテキスト vs RAGの比較評価により、ツール数が増えた際にどちらのパラダイムが検索飽和に強いかが定量的に示されており、大規模ツールカタログ設計の指針になる
