## 要約

Holosは、LLMを基盤とするマルチエージェントシステム（LaMAS）をウェブスケールで運用するための新しいアーキテクチャを提案した論文（2026年1月提出、38ページ）。従来のLaMASが抱える「スケーリング摩擦」「協調の崩壊」「価値の散逸」という三大課題（open-world issues）に対処することを目的としている。

Holosは5層アーキテクチャを採用しており、中核となる3モジュールは以下のとおり。①**Nuwaエンジン**：エージェントの高効率な生成・ホスティングを担うランタイム基盤。大量のエージェントを動的に生成・管理し、スケーリング摩擦を解消する。②**市場駆動型Orchestrator**：複数の異種エージェント間の協調を経済的なインセンティブメカニズムで実現し、単一障害点を排除した弾力性のある協調を提供する。③**内生的価値サイクル（endogenous value cycle）**：エージェント間のインセンティブ整合性（incentive compatibility）を保証する仕組みで、エージェントが自律的に行動しながらもシステム全体の目標と整合するよう設計されている。

Agentic Webの概念は、個々のエージェントが孤立したタスク解決者から「永続的なデジタルエンティティ」へと進化し、異種エージェントが自律的に相互作用・共進化するエコシステムを指す。Holosはミクロレベルの協調とマクロスケールの創発（emergence）の橋渡しを目指し、自己組織化・継続進化するAgentic Webの基盤を提供する。システムは公開されており、大規模エージェンティックエコシステムの研究テストベッドとして利用可能。図8枚・表4つを含む詳細な設計仕様が開示されている。

## アイデア

- 市場駆動型Orchestratorによるインセンティブ設計：エージェント間の協調を「経済的市場」として抽象化することで、中央集権的な制御なしに弾力性ある協調を実現するアプローチは、分散型エージェントシステムの設計パターンとして汎用性が高い
- 内生的価値サイクルによるインセンティブ整合性：エージェントが自律行動しつつシステム目標と整合する仕組みは、LLMエージェントの「目標漂流」問題への構造的な解法であり、長期運用システムに不可欠な概念
- 5層アーキテクチャと創発的スケーリング：マイクロ協調からマクロ創発へのギャップを5層構造で埋める設計思想は、単一のオーケストレーターに依存しないWebスケールエージェントシステムの参照アーキテクチャとなりうる
