## 要約

本記事はThe Gradientに掲載された論考で、現在のLLMベースチャットボットが「目的意識のある対話（Purposeful Dialogue）」を欠いているという問題を指摘する。MMLU・HumanEval・MATHといったベンチマークスコアの向上がユーザー体験の向上に比例していない現実を出発点に、マルチターン対話における課題を体系的に整理している。

目的意識のある対話とは、旅行計画エージェントや心理療法士ボットのように、特定の目標を中心とした複数ターンの会話を指す。交渉理論における「逐次的な取引は一括提示より良い結果をもたらす」という知見を援用し、マルチターン対話の優位性を説明する。SWE-benchのようなGitHubイシュー解決タスクでも、単一パス生成ではなく人間とのバックアンドフォースが必要である点を具体例として挙げる。

システムの構造的問題として、LLMの学習パイプラインを3段階で分析する。①事前学習では大規模コーパスで次トークン予測を行うが、対話的データは少量。②対話フォーマット導入では`tokenizer.apply_chat_template`で<system>/<INST>タグによる構造化を行うが、この形式は事前学習コーパスに存在しない。③RLHFはLeCunの「ケーキのチェリー」に相当するファインチューニングに過ぎず、KLペナルティとLoRAで重みを更新する。

一貫性の定量評価のために、著者らは2つのシステムプロンプト付きLMエージェントを相互に対話させ、任意長の対話を合成する環境を構築した。対話の主幹[a1, b1, …, a8, b8]の途中（例：5ターン目）に「脱線メッセージ」を挿入し、元の役割指示への復帰能力をテストする設計である。実験結果として、GPT-4・Claude・Llama等の主要モデルはいずれも長期対話での役割維持に困難を示した。

より高度な目的意識のある対話の実現に向け、記事は4つの技術的方向性を提示する。①プロンプトベースのアプローチ（多段階指示・スクラッチパッド思考）、②対話ツリーや状態機械による明示的な状態管理、③RLHFの強化版として対話全体の報酬を最適化するマルチターンRLHF（例：Jurassic-2、Gemini）、④ユーザーモデリングの組み込み。

これらの課題は監査エージェント開発に直接示唆を持つ。監査プロセスは本質的にマルチターンであり、監査人がAIと反復的にやり取りしながら証拠収集・リスク評価を進める形態に近い。RLHFの報酬設計を対話全体の目標達成にシフトさせるアプローチや、状態機械による対話管理は、LangGraphベースの監査エージェント設計においても応用可能な知見である。

## アイデア

- 2つのLMエージェントを互いに対話させ途中で脱線メッセージを挿入することでシステムプロンプト遵守の長期一貫性を定量評価するフレームワーク
- 対話フォーマット（<system>/<INST>タグ）が事前学習コーパスに存在しないため、RLHFで初めてそのフォーマットが訓練データに現れるという構造的ミスマッチ
- 単発タスクではなく対話全体の目標達成を報酬とするマルチターンRLHFへの移行が、エージェントの目的遂行能力を根本から変える可能性
