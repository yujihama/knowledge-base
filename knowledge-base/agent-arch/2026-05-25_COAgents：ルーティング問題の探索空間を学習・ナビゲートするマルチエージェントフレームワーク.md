## 要約

車両経路問題（VRP: Vehicle Routing Problem）は物流・配送システムに不可欠だが、組合せ爆発的な複雑性により大規模インスタンスでは計算困難が続いている。従来の手法は手作りヒューリスティクスによる局所改善と局所最小解脱出のための「ジャンプ」に依存しているが、多様なインスタンスへの汎化が困難だった。

COAgentsはこの探索プロセスをグラフとしてモデル化する協調型マルチエージェントフレームワーク。ノードが解（solution）を表し、エッジが局所的改善（refinement）または多様化のための大きな摂動（perturbation＝ジャンプ）に対応する。探索中に動的に構築される「Partial Search Graph（PSG）」上で、3種類のエージェントが協調して動作する：(1) Node Selection Agent（次に探索する解ノードを選択）、(2) Move Selection Agent（局所改善の手を選択し集中探索を誘導）、(3) Jump Agent（新領域への探索タイミングを決定）。

エンドツーエンド学習アプローチとは異なり、問題非依存の探索制御とコンパクトなドメイン固有エンコーディングを明確に分離することで、タスク横断的な適応性を実現している。この設計により、VRPTWなど新たな制約付き問題への拡張が容易になる。

実験はCVRP（容量制約付き車両経路問題）とVRPTW（時間窓付き）のベンチマークで実施。CVRPでは複数の learn-to-search ベースラインと競争力を維持しつつ、より難しいVRPTWインスタンスで学習ベース手法の新たな最先端を達成。最強のニューラルソルバーであるPOMOと比較してN=100で14%、N=50で44%のギャップ削減、ALNS比ではN=100で21%、N=50で40%の改善を達成している。

監査エージェント開発への示唆として、COAgentsの「探索空間をグラフとして表現し複数エージェントが役割分担して制御する」設計思想は、監査判断プロセスにおける証拠探索・仮説生成・検証の分業設計に応用可能。特にNode/Move/Jump Agentの分離はReActエージェントにおけるexploit/explore戦略の明示的実装パターンとして参考になる。

## アイデア

- 探索空間をグラフ（ノード=解、エッジ=遷移操作）として明示的に表現し、そのグラフ上でエージェントを訓練するという設計が、他の組合せ最適化問題（スケジューリング、配置問題）にも適用できる汎用パターンである点
- 問題非依存の探索制御（いつどこを探索するか）とドメイン固有エンコーディング（問題の特徴表現）を分離したアーキテクチャが、モジュール性と転移可能性を同時に高めており、新しい制約（時間窓など）の追加コストを大幅に削減できる点
- 集中探索（intensification）と多様化探索（diversification）を異なるエージェントに担わせ、Partial Search Graphで両者の実行タイミングを動的に学習させる仕組みが、従来の固定スケジュールベースのメタヒューリスティクスより柔軟な探索を実現している点
