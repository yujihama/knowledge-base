## 要約

本論文は、30年間「Webの主要消費者は人間である」という前提の下で構築されてきたWorld Wide Webを、AIエージェントを一級市民として扱う「エージェントファーストWeb」へ再設計するための10の設計原則を提案する。

現状の問題として、WebはAIエージェントをCAPTCHA、blanket blocking、コンテンツ抽出者としての扱いによって排除する構造になっている。これに対し、論文はアクセス層・経済層・コンテンツ層の三層にわたる設計変更を提案する。

【アクセス層】人間を代理するエージェントは、そのユーザーと同等のアクセス権を継承すべきという「エージェント＝人間代理」原則を採用する。具体的には、HTTPリクエストにブラウザヘッダーと同様のエージェント識別メタデータとレートリミットを付加し、同一ドメインから人間向けと機械最適化コンテンツの両方を提供するデュアルレイヤーアーキテクチャを構築する。

【経済層】エージェントの経済的義務はそれが代理する人間のそれを反映すべきとする「インテントベース階層フレームワーク」を提案する。ページビューではなくトークン単位でコンテンツを計量するトークンベースサブスクリプションモデルと、AI生成コンテンツを人間の意図に紐付ける「コミッション型コンテンツエコノミー」を導入する。

【コンテンツ層】最も深刻な問題として「認識論的再帰（epistemic recursion）」を指摘する。これはAI生成コンテンツがエージェントに消費され、さらなるAIコンテンツを生成するという自己参照ループであり、Web上の知識が人間の現実から進行的に乖離するリスクを持つ。対策として以下を提案する：(1) Agent Text Markup Language（ATML）という新たなマークアップ言語、(2) コンテンツへの人間監督度を4段階で示す階層モデル、(3) コンテンツの出所と編集履歴を追跡する暗号学的プロベナンスチェーン。

監査エージェント開発への示唆：監査エージェントがWebから情報収集する際、アクセス正当性・データ信頼性・人間監督の追跡可能性という課題が重要になる。ATMLや暗号プロベナンスチェーンは、エージェントが収集した証拠の信頼性評価フレームワークに直接応用できる。また、「インテントベース階層」の概念はエージェントの行動ログと人間の意図を紐付けるGRC設計にも転用可能。

## アイデア

- 認識論的再帰（epistemic recursion）の概念：AI生成コンテンツがAIに消費されてさらにAIコンテンツを生成する自己参照ループは、RAGシステムの訓練データ汚染問題と直結しており、知識ベース品質保証の設計に根本的な問いを投げかける
- ATMLと4段階人間監督モデル：コンテンツへの人間関与度を構造的にタグ付けするアプローチは、監査証跡における「AIアシスト vs 人間判断」の区別を自動化するフレームワークとして流用できる
- エージェント＝人間代理原則によるアクセス権継承：エージェントが操作する権限を明示的にユーザーにバインドする設計は、LangGraphやMCPでのツール認可設計（who authorized this action）に即応用可能な概念整理
