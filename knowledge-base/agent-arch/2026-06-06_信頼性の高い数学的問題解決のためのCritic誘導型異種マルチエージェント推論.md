## 要約

本論文は、LLMが複雑な数学的推論において依然として抱えるハルシネーション・中間推論誤り・信頼性欠如の問題に対処するため、Critic（批評者）ベースの異種マルチエージェントフレームワークを提案する。

【アーキテクチャの仕組み】
フレームワークはGenerator-Validatorの二段構成を採用する。Generatorは解を生成し、ValidatorはそのCorrectness（正誤）を判定するだけでなく、具体的な批評（Critique）を出力して解の再生成を誘導する。これにより誤りの連鎖伝播（Error Cascading）を防止し、適応的な誤り修正が可能になる。

「異種（Heterogeneous）」という点が重要で、専門性の異なる複数のLLMエージェントを組み合わせる。単一モデルのアンサンブルではなく、異なる特性・強みを持つモデルを協調させることで、単一モデルでは見落とす誤りを補完し合う構造になっている。

さらに、中間フィードバックに基づいてCritic駆動型の適応学習システムが推論プロセスを評価・誘導する。単純な最終回答の正誤チェックではなく、推論の途中段階にフィードバックを介入させる点が特徴的である。

【実験結果】
GSM8K（小学校レベルの数学文章題ベンチマーク）での評価で、シングルショットモデルおよびCriticなしモデルに対して最大13%の精度向上を達成した。

アブレーション研究の結果、性能向上の主因はモデルサイズではなくCriticベースのフィードバックループであることが示された。また、異種性（Heterogeneity）とCritiqueの組み合わせにより大規模モデルへの依存度が低下し、小規模モデルでも大規模モデルと同等の性能が発揮できることが確認された。

【監査エージェント開発への示唆】
LangGraphベースの監査エージェントにおいて、推論ステップのValidator Agentを導入する際の設計指針として直接応用できる。特に「正誤判定 + 批評出力 → 再生成」のループ構造は、監査手続きのステップ検証（証拠収集・リスク評価・判断の妥当性チェック）に転用可能。また、大型モデル不要という知見はローカルLLM環境（RTX 3090）での複数小型モデル協調構成の実現可能性を支持する。

## アイデア

- ValidatorがCorrectness判定だけでなくCritique（批評文）を出力して再生成を誘導する設計は、LLM-as-judgeの発展形として監査エージェントの中間ステップ検証に直接応用できる
- 異種モデル協調により小規模モデルが大規模モデルと同等性能を発揮できるという知見は、ローカルLLM環境での複数モデル協調アーキテクチャの設計根拠になる
- Error Cascadingの防止を中間フィードバックで実現するアプローチは、LangGraphの条件分岐ノード設計（推論途中での自己修正ループ）のパターンとして直接参照できる
