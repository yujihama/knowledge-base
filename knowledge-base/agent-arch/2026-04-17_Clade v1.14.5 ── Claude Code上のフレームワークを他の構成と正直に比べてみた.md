## 要約

CladeはClaude Code上で動作するMarkdownベースのマルチエージェントフレームワーク（v1.14.5）。本記事では「Claude Code生（設定なし）」「CLAUDE.mdだけ」「Clade」「CLAUDE.md自作」の4パターンを正直に比較し、Cladeの位置づけを明確化している。

Cladeの主な構成要素は以下の通り。①エージェント役割分担：インタビュアー・アーキテクト・プランナー・デベロッパー・テスター・コードレビュアー・セキュリティレビュアーの7役が専用指示書を持つ。②フェーズ間引き継ぎ：各フェーズの成果物（要件定義レポート・アーキテクチャレポート等）を`.claude/reports/`にタイムスタンプ付きで保存し、セッションを超えてコンテキストを維持する。③セッション管理：`/end-session`で残タスク・成果・失敗を記録し、`/init-session`で翌日から復元できる。④`/update`コマンドによるフレームワーク自動更新。

最大の設計的特徴は「全部Markdown（自然言語）のみ」であること。PythonやJavaScriptで実装された他フレームワークと異なり、エージェント定義・ルール・ワークフロー・スキルすべてをMarkdownで記述する。これにより「透明性（.claudeフォルダを開けば動作が一目でわかる）」と「ノーコードカスタマイズ」を実現する一方、最終制御はClaudeの解釈に委ねられるため稀に指示無視が発生する。

その他の特徴として、`/agent-workflow-builder`による繰り返し業務のワークフロー自動生成（保育士向け「発達経過まとめ」等の非開発業務にも対応）、`/agent-project-setup`によるコーディング規約の一括適用、`/agent-mcp-setup`によるMCPサーバのconversational設定が挙げられる。`.claude/`フォルダをGitにコミットすれば、チームの知見（ルール・スキル・レポート）をそのまま引き継げる。

正直な欠点として：Node.js・Git・Claude Code CLIのインストールが必要でノンエンジニアには敷居が高い、Claude Code専用、コマンド数が多い（ただし`/init-session`のトリアージ機能でsmall/medium/largeを選択すれば適切なワークフローへ誘導される）、各フェーズに承認ポイントがある（ヒューマン・イン・ザ・ループ設計）。

v1.14.2〜v1.14.5の更新内容：Windows/bash経由でのsetup.ps1文字化け修正（CP932→UTF-8）、`/init-session`へのタスク規模トリアージ追加（small/medium/large）、statuslineのrate limitリセット時刻バグ修正（秒単位Unixタイムスタンプをミリ秒として誤解釈していた問題を`×1000`で修正）。

監査エージェント開発への示唆：各フェーズのレポート自動保存と承認ポイント（ヒューマン・イン・ザ・ループ）の設計は、監査プロセスの証跡管理・承認フローと親和性が高い。`.claude/`ごとGit管理する手法は、監査エージェントの判断根拠・設計変更履歴の追跡に直接応用できる。

## アイデア

- 「全部Markdown」という設計思想：プログラミング言語実装と比較して透明性とカスタマイズ性を優先し、エージェント定義・ルール・ワークフロー全体を自然言語のみで記述することで非エンジニアでも中身を把握・編集できる構造にしている
- フェーズ間引き継ぎをファイルシステムで実現：セッションを超えたコンテキスト維持を`.claude/reports/`への成果物保存で解決しており、LLMのコンテキストウィンドウ制約をファイルI/Oで回避するアーキテクチャパターン
- /init-sessionのタスク規模トリアージ（small/medium/large）：毎回フルワークフローを起動する非効率を防ぐためのルーティング機構で、タスク複雑度に応じてエージェント起動コストを調整する設計
