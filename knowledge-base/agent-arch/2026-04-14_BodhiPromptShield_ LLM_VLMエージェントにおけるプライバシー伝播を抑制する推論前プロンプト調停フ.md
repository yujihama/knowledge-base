## 要約

LLM/VLMエージェントでは、ユーザーが入力した生のプロンプトが単一のモデル呼び出しに留まらず、RAGの検索クエリ、メモリへの書き込み、ツール呼び出し、ログ出力など複数のステージを横断して流通する。このクロスステージ伝播が従来の文書単位での匿名化パイプラインでは対処できないプライバシーリスクを生み出している。BodhiPromptShieldは、この問題に対処するポリシー認識型フレームワークで、推論前（Pre-Inference）の段階でプロンプト内の機密スパンを検出し、三種類のルーティング手法——型付きプレースホルダー（typed placeholders）、意味的抽象化（semantic abstraction）、セキュアシンボリックマッピング（secure symbolic mapping）——によって機密情報を仮置きする。その後、認可された境界に到達した時点でのみ元の情報を復元する「遅延復元（delayed restoration）」を採用しており、復元タイミング自体をセキュリティ変数として扱う点が既存の企業向け匿名化ツールとの差別化点である。評価はControlled Prompt-Privacy Benchmark（CPPB）を用いた制御実験で行われ、ステージごとのプライバシー伝播率を10.7%から7.1%へと抑制することに成功した。プライバシー漏洩率（PER）は9.3%、文脈保持率（AC）は0.94、タスク成功率（TSR）は0.92を達成し、汎用的な匿名化手法を上回る性能を示した。ただし、これらの数値はCPPB上の制御実験結果であり、公開ベンチマークへの転用可能性や正式なプライバシー保証を主張するものではない。監査エージェント開発への示唆として、エージェントが外部ツールや社内データベースと連携する際に、中間的な処理ステージで機密情報が意図せず露出するリスクは実務上も重要な論点であり、本フレームワークのような推論前調停レイヤーをLangGraphのノード間やMCPツール呼び出し前に挿入するアーキテクチャ設計は、GRC（ガバナンス・リスク・コンプライアンス）対応エージェントの設計に直接応用できる視点を提供する。コードはGitHubで公開されている。

## アイデア

- プライバシー情報の「復元タイミング」をセキュリティ変数として設計する発想は、エージェントの各処理ステージを信頼境界として定義するゼロトラスト的なアーキテクチャ思想と整合する
- typed placeholders / semantic abstraction / secure symbolic mappingという三層のルーティング戦略により、文脈保持率（AC=0.94）とタスク成功率（TSR=0.92）を高水準で維持しながらプライバシー漏洩を抑制できる点は、実用性と安全性のトレードオフを具体的な数値で示している
- LangGraphやMCPのようなマルチステージエージェントフレームワークにおいて、ノード間データフローのプライバシー制御レイヤーとして本フレームワークを組み込む設計パターンは、内部監査エージェントの機密情報取り扱い要件に直接対応できる
