## 要約

現行のLLMチャットボットはMMLU・HumanEval・MATHなどのベンチマークで急速に改善されているが、それらは単一ターンの非インタラクティブな評価であり、実際のユーザー体験の向上とは必ずしも比例しない。本稿は「目的意識のある対話（Purposeful Dialogue）」という概念を提唱し、LLMチャットボットが抱える構造的限界を整理する。

目的意識のある対話とは、「旅行プランニング」「カスタマーサポート」「心理療法士」のような具体的なゴールを中心に展開する複数ターンの会話を指す。交渉理論の反復的バーゲニングと同様に、一度に全情報を渡すより、複数回のやり取りを通じて重要情報だけを選択的に交換する方が良い結果を生む。SWE-benchのようなGitHub issue解決タスクも、単一アクションでは達成困難であり、人間とAIの往復コミュニケーションが不可欠である。

対話システムの歴史的背景として、1970年代のRoger Schankによる「レストランスクリプト」、ELIZAやPARRYのようなルールベースシステムから始まり、現在のLLMベースシステムに至る変遷を整理。LLMは（1）次トークン予測による事前学習、（2）`tokenizer.apply_chat_template`によるチャットフォーマット導入、（3）RLHFによる微調整、の3段階で構築されるが、RLHFはLeCunの「ケーキのチェリー」に過ぎず、訓練データ量は事前学習と比較して桁違いに小さい。

一貫性の定量評価として、著者らはLLMエージェント2体を長時間対話させ、任意の時点で「脱線プローブ」（話題を変えようとする介入）を挿入して元の役割維持能力を測定する環境を構築。8ラウンドの対話中に役割逸脱が蓄積することを確認している。また、コンテキストウィンドウに占めるシステムプロンプトの割合が低下するにつれ、指示追従能力が劣化する「Lost in the Middle」問題も引用している。

解決策として、強化学習（RL）を対話レベルで適用するCoH（Chain-of-Hindsight）やRRHF、PPOベースの対話最適化手法を検討。また、目的達成に向けた対話戦略として「情報収集」「提案」「確認」などの発話行為（Speech Act）を明示的にモデル化することが示唆される。監査エージェント開発への示唆として、監査調書作成や証跡収集の場面では、単一プロンプトではなく複数ターンの目的指向対話設計が不可欠であり、対話の一貫性維持機構（長期メモリ、役割固定プロンプト設計）の実装が求められる。

## アイデア

- 2体のLLMエージェントを長時間対話させ任意時点で「脱線プローブ」を挿入することで、システムプロンプト追従能力の経時劣化を定量評価できる手法は、監査エージェントの堅牢性テストに直接応用可能
- 交渉理論の反復的バーゲニング（iterative bargaining）をLLM対話設計に援用する視点：一括情報渡しより複数ターンの選択的情報交換がアウトカムを改善するという原則は、要件定義や証跡収集エージェントの設計指針になる
- RLHFが「事前学習の巨大な氷山に対するチェリー」に過ぎないという構造的限界の指摘：対話の一貫性を根本的に改善するには対話レベルのRLや長期メモリ機構が必要であり、fine-tuningだけでは不十分
