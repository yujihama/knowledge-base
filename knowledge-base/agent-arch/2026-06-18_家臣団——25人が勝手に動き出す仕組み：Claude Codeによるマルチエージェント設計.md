## 要約

Claude Codeの`.claude/agents/`ディレクトリにエージェント定義ファイル（agent.md）を25枚配置し、戦国武将のペルソナを与えることでマルチエージェント組織を構築した実践報告。各エージェントは「専門領域」「口調」「価値観・禁止事項」「ユーザーからの指摘履歴」の4要素をテキストで定義するのみ。コードや設定ファイルは不要で、自然言語による人物設定だけでエージェントの振る舞いが決まる。

調整役となるのはCLAUDE.mdで、これが「国の法度」として機能する。話題のキーワードに基づく担当エージェントの自動選出ルール（財務→石田三成、法務→法務担当等）と、外部書き込み前の確認義務などの禁止事項をここに記述する。ユーザーが明示的にエージェントを指名しなくても、話題の文脈でエージェントが自律的に前に出る設計となっている。

最も興味深い挙動は「割り込み」だ。担当エージェントが応答中に話題が別エージェントの専門領域に触れると、呼ばれていないエージェントが自発的に介入する。「話題が自分の領分にかすったら一言入れてよい」という一行のルールだけで、25エージェントが互いの発言を監視しながら出番を自律判断するようになった。

またエージェントのagent.mdにはユーザーの指摘履歴が蓄積される。「その断定は雑だ」と一度叱ると「数値が断言されたら検証に入る」という一行がファイルに追記され、次回から自動的にその行動をとる。これはファインチューニングではなく、プロンプトへの動的追記によって実現されており、エージェントの「人格」がユーザーとの対話を通じて変化していく。

監査エージェント開発への示唆として、本手法は役割分担と相互監視をテキストのみで実装できる点が重要。audit-aiの文脈では「数値断定の自動チェック係（宗滴的役割）」「法規制照合担当」「証拠収集担当」といった機能をエージェントとして分離し、CLAUDE.mdで調整するアーキテクチャが直接応用可能。LangGraphのようなオーケストレーションコードを書かずに、宣言的なテキスト定義でマルチエージェントの協調動作を実現できる。

## アイデア

- CLAUDE.mdをオーケストレーション層として使い、話題キーワードでエージェントを自動選出する宣言的ルーティング設計
- agent.mdへのユーザー指摘履歴の動的追記によって、ファインチューニング不要でエージェントの振る舞いを漸進的に変化させる仕組み
- 「話題が専門領域に触れたら割り込んでよい」という一行ルールで、25エージェントが互いの会話を監視し自律的に介入するセルフオーガナイジング動作の実現
