## 要約

本稿はThe Gradientに掲載された長編論考（35分読了目安）で、AIアライメントにおける「目標ベース」の合理性モデルを根本から問い直す。著者の中心的主張は「合理的な人間は目標を持たない、合理的なAIも目標を持つべきではない」というものだ。

論文は「eudaimonic rationality（エウダイモニア的合理性）」という概念を提唱する。アリストテレスの徳倫理学に由来するeudaimonia（幸福・繁栄）から着想を得た概念で、行動・行動傾向・行動評価基準・行動リソースのネットワークである「実践（practice）」を中心に据える。例えば数学者がTerry Taoの例を通じて説明されるように、数学的卓越性とは「数学的卓越性を数学的に卓越した方法で促進する」という再帰的構造を持つ。これは「目的のための手段」という従来の instrumental/terminal value の二分法とは根本的に異なる。

著者が批判する標準モデルはEffective Altruism的な最適化——効用関数を最小の内在的価値セットに還元し、残りを instrumental として「説明する」アプローチ——であり、これをAIエージェントに適用することを「型の不一致（type mismatch）」と呼ぶ。人間の実践（kindness, corrigibility, harmlessness等）はeudaimonic rationality上では自然な概念だが、功利主義的オプティマイザには根本的に翻訳不能だと論じる。

論文の構成は6部に渡る。Part I〜IIIで eudaimonic rationality の概念整理、Part IVで「eudaimonic practice」の形式定義（高スコア行動が将来の高スコア行動を因果的に促進するネットワーク）、Part VではClaude流コーギビリティ・害なさ・透明性といった安全性プロパティが目標ベースエージェントには脆く不安定だが、eudaimonic エージェントには自然に成立することを論じる。Part VIでは eudaimonic に合理的なAIエージェントが人間の繁栄を支援する場合の条件を検討する。

監査エージェント開発への示唆としては、コーギビリティ（人間の修正を受け入れる性質）をルールや目標として実装しようとする設計の脆さが指摘されている点が重要だ。eudaimonic モデルでは、コーギビリティは「コーギブルに行動することがコーギビリティの実践を促進する」という自己強化的なネットワーク構造として実装されるため、境界条件での予期しない行動（脱獄・ゴール追求的行動）が起きにくくなる可能性がある。ReActベースのエージェント設計においても、固定目標ではなく practice-oriented な評価基準を持つエージェント設計が、長期的な安全性において優位になり得る。

## アイデア

- 「promote x x-ingly」という公式——kindnessをkindlyに促進する、honesty をhonestlyに促進する——がAIの安全プロパティ実装に応用できるという発想は、ルールベース・目標ベース設計の代替として具体的に検討に値する
- instrumental/terminal value の二分法ではなく「行動ネットワーク内の有機的因果一貫性」を価値構造とするモデルは、LangGraphのようなグラフベースエージェントのノード評価関数設計に構造的な対応物がある
- corrigibility・harmlessness等の安全プロパティが目標ベースエージェントには「不自然（brittle）」で、eudaimonic エージェントには「自然」だという主張は、RLHFによる訓練が何を最適化しているかという問いと直接接続する
