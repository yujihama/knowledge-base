## 要約

個人開発者が公開運用中の「AIが自走する物語サイト」の設計・実装を紹介した記事。システムは京都の霊脈世界を舞台に、5体の妖（あやかし）をClaudeが2時間ごとに1日分動かすシミュレータである。各妖は利他心・独立心・信頼といった異なるコアを持ち相互作用する。世界には30日の期限があり、30日目に「大禍」が来るとループリセットされるローグライク構造を採用。条件が揃ってループを断ち切ると物語が完結し、サーバー側ワーカーが自動停止する一回きりの設計となっている。

アーキテクチャ上の最大の特徴は「判断はLLM、計算は決定論的エンジン」という責務分離。キャラの行動選択・台詞・日記・関係性ラベルはClaudeに委ね、食糧の実り・霊力バランス・日次負荷・パラメータのクランプ・死判定・ステージ遷移はTypeScriptエンジンが担保する。これによりLLMの出力が不安定でもシミュレーション状態が壊れない設計を実現している。

LLM呼び出しにはClaude Code CLIのヘッドレスモード（claude -p）を使用し、従量課金APIへの誤転落を防ぐためANTHROPIC_API_KEYを子プロセス起動時に明示的に除去。本番はCLAUDE_CODE_OAUTH_TOKENによるサブスク認証で運用している。使用上限エラーはUsageLimitErrorとして専用分類し、tickを丸ごと中断してDBの最終スナップショットから同日を再試行するバックオフ機構を実装。

公開APIはGETのみで、POST/PUT/DELETEエンドポイントは一切定義しない「watch-only」設計。外部からは世界を止められず、ワーカーのみが状態を変更できる。デプロイはRailwayでoven/bunベースのDockerfile上にバージョン固定したClaude Code CLIをインストール、SQLiteをRailway Volumeに永続化している。DBはDrizzleのdrizzle-kit pushで同期し、「完結まで放置」前提のためマイグレーション管理を省略している。開発規模は約1週間（2026/05/30〜）、コミット119+、TypeScript/TSX 55ファイル・約13,000行の1人開発。監査エージェント開発への示唆としては、LLMを「判断層」に特化させ決定論的エンジンと組み合わせることで長期自律運用の安定性を確保するアーキテクチャパターンが参考になる。

## アイデア

- LLMを「判断」、決定論的エンジンを「計算」に分離することで、LLMの出力の揺れがシミュレーション状態の破壊につながらない長期自律運用アーキテクチャを実現している点
- Claude Code CLIのヘッドレスモード（claude -p）をバックエンドのLLM呼び出しに活用し、ANTHROPIC_API_KEYを子プロセスから除去してサブスク認証に強制する課金制御の実装テクニック
- 物語の完結をトリガーにサーバーワーカーが自動停止し、APIはGETのみ定義する「watch-only」設計で外部干渉を設計レベルで排除した一回性イベントの実装パターン
