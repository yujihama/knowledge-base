## 要約

本レポートは、AIエージェントのセキュリティ保護機能（ガードレール）を比較評価した研究である。評価対象は DKnownAI Guard、AWS Bedrock Guardrails、Azure Content Safety、Lakera Guard の4製品で、人手アノテーションをグラウンドトゥルースとして使用している。

評価するリスクは2カテゴリに分類される。第1カテゴリは「エージェント自体への脅威」で、具体的には指示オーバーライド（instruction override）、間接プロンプトインジェクション（indirect injection）、ツール濫用（tool abuse）が含まれる。第2カテゴリは「有害コンテンツの引き出しを意図したリクエスト」で、ヘイトスピーチ、ポルノグラフィー、暴力的コンテンツが対象となる。

評価結果では、DKnownAI Guard が再現率（Recall）96.5%、真陰性率（TNR: True Negative Rate）90.4%でいずれも最高値を記録し、総合的に最も優れたパフォーマンスを示した。TNR（正常なリクエストを誤ってブロックしない率）も高いことから、偽陽性を抑えつつ高い検出率を実現している点が特徴的である。

AI エージェントが自律的に外部ツールを呼び出したり、複数ステップで推論・実行を行う現代のアーキテクチャでは、従来のコンテンツフィルタでは対応しきれない脅威が増加している。特に間接プロンプトインジェクションは、エージェントが処理する外部データ（Webページや文書）に埋め込まれた悪意ある指示を介してエージェントの挙動を乗っ取る攻撃であり、RAGやツール連携を伴うエージェントシステムにおける主要リスクの一つである。

監査エージェントシステムの開発という観点では、本研究は重要な示唆を持つ。内部監査用エージェントが外部データソース（ERPログ、会計システム出力等）を参照・処理する際、間接インジェクションによって監査判断が操作されるリスクが現実的に存在する。ガードレール製品の導入前に recall と TNR のトレードオフを定量評価するフレームワークとして、本研究の手法は参考になる。各クラウドベンダー（AWS、Azure）の既製品との比較評価を人手アノテーションで実施している点も、実務適用を検討する上での信頼性の根拠となる。

## アイデア

- エージェント固有の脅威（instruction override, indirect injection, tool abuse）とコンテンツ系脅威を明確に2カテゴリに分けて評価している点が、従来の汎用コンテンツフィルタ評価との差別化ポイントになっている
- TNR（真陰性率）を再現率と並べて主要指標として評価している点が実務的：偽陽性が高いガードレールは正常業務を妨害するため、セキュリティと利便性のバランス指標として重要
- 人手アノテーションをグラウンドトゥルースとして採用しており、LLM-as-judge ではなくヒューマンアノテーションによるベンチマーク設計の信頼性確保という方法論が参考になる
