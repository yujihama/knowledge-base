## 要約

AGENTS.mdやCI設定などのエージェント制御ハーネスは、一度書いたら終わりではなく、継続的に改善しなければならないという問題意識から出発する記事。静的なハーネスはモデルアップデートやプロジェクト構成の変化に追従できず、エージェントが同じ失敗を繰り返す原因となる。

Self-Evolving Agentとは、「エージェント実行→自動評価(Grader)→メタプロンプト書き換え」の3ステップフィードバックループによってハーネス自体を自動改善するアーキテクチャパターン。OpenAI公式Cookbookが製薬企業の規制文書作成タスクを題材に3戦略を体系化した：①Platform Optimizer（人間のthumb up/downで高速改善）、②Static Metaprompt（4種のGraderで自動反復改善）、③GEPA Optimization（定量スコア＋テキストFBで構造的探索）。人間関与度を段階的に下げられる点が特徴。

実装パターンは3段階。パターン1は失敗ログの構造化記録（failures.jsonl）で、どのハーネスファイルがどんなエラーを引き起こすか1週間計測するだけで改善箇所が可視化される。パターン2はCI失敗ログをLLMに渡してAGENTS.mdへの追加ルール案を自動生成し、人間がレビューしてマージするセミオート方式。自動適用ではなく自動提案に留める点が2026年時点の実用的な落とし所とされる。パターン3はClaude Code hooksを活用したPostToolUseトリガーで、ファイル編集後に自動lint実行などをプロンプトではなくコードで決定論的に強制する。

Meta ResearchがICLR 2026で発表したHyperAgents（arXiv: 2603.19461）は、タスクエージェントとメタエージェントを統合し、メタエージェント自身の改善手順も書き換える自己参照ループを実現。paper reviewタスクで精度0.0→0.710を達成し、エージェントが自律的に「知見蓄積ストレージ機構」を自コードに追加するなど設計者の想定外の構造が自然発生した。

限界として、戦略転換・品質基準定義・倫理判断の3点は人間が担うべきとされる。Graderの設計が誤れば誤った方向に最適化が進む。週次運用サイクルとして「月曜:ログ分析→改善提案生成、火曜:人間レビュー、水曜:マージ、木金:改善ハーネスで作業、土日:自動ログ蓄積」を提示。人間の関与は週1日に絞りつつ改善を累積させる設計。監査エージェント開発においても、同じ判断ミスの繰り返し防止にfailures.jsonlパターンとhooksによる制約強制の組み合わせは直接応用可能。

## アイデア

- HyperAgentsでエージェントが自律的に「知見蓄積ストレージ機構」を自コードに追加したという実験結果は、設計者が意図しないインフラが自己組織化される可能性を示しており、エージェントの創発的設計能力の証左として興味深い
- プロンプトによる制約は解釈のブレやhallucinationが生じるが、Claude Code hooksはシェルコマンドとして決定論的に動作するため、品質基準をコードに落とすことで再現性を担保できるという観点は、監査エージェントの制御設計に直接応用できる
- 週次サイクルで「書いてあるのに効いていない制約」をログ分析で発見できるという副産物は、ハーネスの問題が記述不足だけでなく有効性検証の欠如にもあることを示しており、ルールの実効性監査という新しい視点を提供する
