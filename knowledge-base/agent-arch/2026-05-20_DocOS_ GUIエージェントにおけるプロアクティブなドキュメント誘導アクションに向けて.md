## 要約

GUIエージェント（Graphical User Interface Agent）は、デバイス操作の自動化において一定の成果を上げているが、その能力は事前学習や命令チューニングで得た静的なパラメトリック知識に依存している。このため、モデルのパラメータに存在しない明示的な手続き的知識を必要とするロングテールタスク（頻出度の低い稀なタスク）において、エージェントは試行錯誤に頼らざるを得ず、非効率で脆弱な動作となる問題があった。本論文は、この根本的な制限を解消するために「Proactive Document-Guided Action（能動的ドキュメント誘導アクション）」という新しいパラダイムを提案する。このパラダイムでは、エージェントが自律的に関連ドキュメントを検索し、ロングテールタスクを解決できるよう設計されており、人間の問題解決プロセス（わからなければマニュアルを調べる）を模倣している。このパラダイムの評価基準として、完全インタラクティブ環境でのドキュメント誘導型問題解決を測定するベンチマーク「DocOS」を新たに提案した。DocOSでは、エージェントはウェブブラウザを自律的に操作し、関連する公式ドキュメントを探索・取得し、その手続き的指示を理解し、実際のGUI操作へと変換（グラウンディング）することが求められる。広範な実験を通じて、現状のGUIエージェントは「デュアルボトルネック」と呼ぶべき2つの課題に厳しく制約されていることが明らかになった。第一に、プロアクティブ検索中に関連情報を確実に特定するのが困難であること（検索精度の問題）。第二に、取得したドキュメントの指示を正確なGUIアクションへと忠実にグラウンディングするのに頻繁に失敗すること（指示実行精度の問題）。これらの知見から、ドキュメント誘導型のインタラクションが、動的な環境で自己進化するGUIエージェントを実現するための重要な研究方向性であることが示唆されている。監査エージェント開発への示唆としては、長尾の手続き的タスク（例：監査基準・内部規定の参照が必要な判断）を自律的にドキュメント参照で補完するアーキテクチャは、監査エージェントのロバスト性向上に直結する可能性がある。

## アイデア

- 静的な事前学習知識に依存せず、実行時に自律的にドキュメントを検索・参照する『プロアクティブ検索』という設計思想は、知識のカバレッジ問題を根本から回避するアプローチである
- 「検索精度」と「グラウンディング精度」という2つの独立したボトルネックを実験的に特定した点は、GUIエージェントの性能改善の研究ロードマップを明確化する
- ドキュメント誘導型アクションを監査エージェントに応用すると、監査基準（ISA、IIA等）や社内規程を動的に参照しながら判断するエージェントの設計に直接転用できる
