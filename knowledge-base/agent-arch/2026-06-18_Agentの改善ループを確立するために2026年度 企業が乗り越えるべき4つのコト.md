## 要約

本記事は、企業がAIエージェントの改善ループ（人とAgentの好循環）を軌道に乗せるために取り組むべき4つの課題を実務視点で整理したもの。

**1. AgentとシステムのMCP連携**
Agentが企業内システムにアクセスできなければ、汎用コメンテーターに留まる。MCP（Model Context Protocol）の普及とSkillsの標準化により技術的な口は開いてきたが、セキュリティ・ネットワーク・権限制御がボトルネックとなっており、DX推進部隊が主導すべき課題として位置づけられる。

**2. コンテキスト整備（RAGのモダナイズ）**
RAGを単なるベクトル検索として放置している企業が多いが、現代のRAGはAgentic化（繰り返し検索クエリを生成して情報を探索する手法）が主流になりつつある。o3等のReasoning Modelと組み合わせることで検索精度が飛躍的に向上する。また、チャンク単体ではなくファイル名・章構造・更新日時・利用文脈などの背景情報を付与する「コンテキストレイヤー」の構築が不可欠。参照権限問題については、Azure AI SearchのACL引き継ぎ機能や、Microsoft 365のWork IQ APIのようにベンダ側のAgentに検索を委任する2つのアプローチが有効。

**3. Agentの業務適用とFDE人材**
Agent適用には業務暗黙知の言語化が前提となるが、この整理作業は人間がボトルネックになりやすい。適用を推進する人材には、業務理解力・コミュニケーション能力・ITシステム設計力・LLM技術力・Coding Agent活用力が同時に求められ、これがFDE（Forward Deployed Engineer）と呼ばれる。コンサル企業がFDEチーム構成を取るケースが現実的な解として想定される。

**4. 改善ループの確立**
Agentのメモリは3種類に分類される：①パーソナライズ用メモリ（ユーザ背景情報の蓄積）、②作業状態管理メモリ（タスク進捗管理）、③失敗分析に基づく改善メモリ（最重要）。AnthropicはこのメモリからAgentを自律改善する「Dreaming」機能をリサーチプレビュー中。改善されたAgentが劣化していないかを評価するEval設計が最難関であり、コーディングAgentはテストによる明確な評価基準を持つため改善ループが組みやすいが、資料作成Agentのような一般業務では評価指標の設計自体が大きなテーマとなる。監査エージェント開発への示唆として、内部監査業務のAgent化においても「評価指標の定義」と「ループ設計」が先行課題となり、特に監査判断の正誤をどうEvalするかがAgent品質の鍵を握る。

## アイデア

- AnthropicのDreaming機能：失敗ログを睡眠中の記憶整理に喩え、Agentが自律的に改善施策を検討・整理する仕組みのリサーチプレビューが進行中であり、Agent自己改善ループの実装パターンとして注目に値する
- Coding Agentがローカルで改善ループを組みやすい根本理由がテストによる明確な評価基準の存在にあるという指摘は、他業務へのAgent適用における「Eval設計こそがループ品質の律速段階」という普遍的な原則を示している
- 参照権限問題の2パターン解決策（Search側でACL引き継ぎ vs ベンダAgent側に委任）は、企業のデータガバナンス設計において重要な意思決定分岐点であり、Microsoft Work IQ APIのようなベンダAPIへの権限管理の外部化という新しいアーキテクチャパターンを示す
