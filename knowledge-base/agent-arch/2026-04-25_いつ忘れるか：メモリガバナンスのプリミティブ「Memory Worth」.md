## 要約

エージェントのメモリシステムは経験を蓄積し続けるが、タスク分布が変化するにつれてどのメモリを信頼・抑制・廃棄すべきかを決定する原理的な指標が存在しなかった。本論文はこの問題に対し「Memory Worth（MW）」という軽量なシグナルを提案する。

MWはメモリ1件ごとに「成功時の共起カウンター」と「失敗時の共起カウンター」の2つのスカラー値を保持する。エージェントがあるメモリを検索して使用し、エピソードが成功（y=+1）または失敗（y=-1）で終わるたびに対応するカウンターをインクリメントする。この比率が条件付き成功確率 p+(m) = Pr[y_t = +1 | m ∈ M_t]、すなわち「メモリmが検索された条件のもとでタスクが成功する確率」に確率1で収束することを理論的に証明している。収束条件は「定常な検索レジーム」と「最小限の探索条件」であり、実装コストは極めて低い。

重要な留意点として、p+(m)は因果的な量ではなく連想的な量である。メモリmがタスク成功を引き起こしたかどうかではなく、成功エピソードと共起した頻度を測定している。それでも運用上のガバナンス指標として有用であると著者らは主張し、実験でも検証している。

合成環境での10,000エピソードの実験では、MWと真のユーティリティとのSpearman順位相関が ρ = 0.89 ± 0.02（20シード平均）に達した。対して評価を更新しないシステムは ρ = 0.00 であり、MWの有効性が明確に示された。

さらに実テキストとニューラル埋め込み（all-MiniLM-L6-v2）を用いたリアルな検索実験（3,000エピソード）では、陳腐化したメモリのMWが閾値 0.17 を下回る一方、専門メモリは 0.77 と高い値を維持した。この閾値ベースのアプローチにより、検索抑制や廃棄のトリガーを自動化できる。

実装上の要件は「検索ログ」と「エピソード結果ログ」のみ。既存アーキテクチャへの追加が容易であり、LLMによる判断やヒューリスティックに依存せずにメモリ品質をデータドリブンで管理できる点が実用上の強みである。監査エージェント開発においては、判断根拠となる過去事例（メモリ）の信頼性を定量的に管理する仕組みとして直接応用可能。

## アイデア

- 成功/失敗の2カウンターのみでメモリの信頼性を定量化できるという極限まで削ぎ落とされた設計思想：追加のLLM呼び出しもベクトル計算も不要で、既存のログから後付けで計算できる
- p+(m)を「因果」ではなく「連想」として明示的に定義することで、理論的な誠実さを保ちつつ実用的なガバナンス指標として使う割り切り：因果推論が難しい非実験的な設定での現実的な妥協点
- MW閾値（0.17 / 0.77）によりメモリの「廃棄・抑制・信頼」を自動分類できるため、監査エージェントの判断根拠メモリが時間経過で陳腐化するリスクをデータドリブンで管理できる
