## 要約

LLMを用いた知識グラフ質問応答（KGQA）では、グラフトラバーサルを言語モデルに委譲し、各質問をビームと複数ホップにわたるローカル関係選択の連続として処理する。従来の一般的なアプローチでは、ルーティングプロンプトに完全な部分パスをシリアライズして渡していたが、コントローラーはすでにそのパスを正確なシンボリック状態として保持しているため、これは冗長な二重管理となっていた。

本論文が提案するBounded Path Context（BPC）は、この二つの役割を分離する。コントローラーはフルパスをシンボリックメモリに保持して回答抽出や監査に使用し、関係選択プロンプトには「質問」「現在のエンティティ」「出力関係候補」「直近Kホップのみ」を渡す設計とする。Kの値を固定したグラフ近傍・ビーム予算・深さ・デコーディング・回答抽出フォーマットのもとで制御スイープ実験を実施した。

実験はWebQSP・CWQの完全テストセットにQwen3.5-9B-AWQを用いて行われ、K=1がWebQSPで回答集合F1=0.487（フル履歴=0.472）、K=0がCWQで0.287（フル履歴=0.274）を達成し、入力トークン数はそれぞれ9.7%・12.1%削減された。4Bスケールモデルでも両ベンチマークでK=1が最強設定であった。

例ごとの分析では、71〜84%の事例は履歴長の影響を受けず、影響を受けた事例において「過去ホップが曖昧性を解消するケース」と「逆に気を散らすケース」の両方が明らかになった。この結果は、パスシリアライズ長はデフォルト仮定ではなく、チューニング可能なインターフェース変数として扱うべきであることを示す。監査エージェント開発においても、LLMへの入力コンテキスト設計（何を渡し何を省くか）は推論精度とトークン効率の両面で重要な設計判断であり、本研究の知見は直接応用可能である。

## アイデア

- パスの完全履歴をLLMに渡すことが必ずしも精度向上につながらず、直近1〜2ホップのみで同等以上の性能が出るという反直感的な結果は、LLMのコンテキスト活用方法の再考を促す
- コントローラーとLLMの役割分離（シンボリック記憶 vs. 関係選択）という設計原則は、LangGraphなどのエージェントフレームワークにおける状態管理設計に直接転用できる
- 71〜84%の事例が履歴長に無影響という知見は、ほとんどのKGQAタスクがローカル情報のみで解決可能であることを示唆し、グラフ推論の局所性に関する理論的示唆を持つ
