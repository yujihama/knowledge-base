## 要約

本書はZennで公開された有料書籍（900円）で、50代のシステムエンジニアである著者saitokoが、プログラミングスキルなしにClaude Codeを活用してマルチエージェント構成の「AIチーム」を構築・運用するまでの実践記録。全8章・約120,497字で構成される。

内容の中心は「AIに役割を持たせる」という発想転換にある。従来のチャットボット的なLLM利用から脱却し、各AIエージェントに明確な役割（ペルソナ）を与えることで、チームとして機能させるアプローチを解説している。具体的には、まず2体構成から始め（例：企画担当エージェントと実装担当エージェント）、必要に応じて役割を追加していく段階的な設計方針を採用している。

Claude Codeを中心ツールとして使い、コードを書かずにエージェントのプロンプト設計・役割定義・チーム構成を行う手法を詳述。エージェントが間違える・作業が散らかるなどの失敗事例も第5部で正直に記載しており、ノーコード実践者視点のリアルな運用知見が含まれる。付録にはプロンプト集・設定サンプル・FAQも収録されている。

著者は「コードを書けないこと」を弱点ではなく、エージェントへの指示設計に集中できる強みとして再解釈しており、プロンプトエンジニアリングとエージェント設計を主武器とするアプローチを提唱している。監査エージェント開発への示唆としては、LangGraph等のフレームワークを使わずにClaude Code単体でマルチエージェント的な役割分担を実現する低コスト構成の参考になる。特に、各エージェントへの役割定義（システムプロンプト設計）と、2体構成から始めるスモールスタートの思想は、監査ワークフローへのエージェント導入設計に直接応用可能。

## アイデア

- 「コードを書けない」制約がプロンプト設計への集中を強制し、エージェント役割定義の質を高めるという逆説的優位性の主張
- 2体構成（企画・実装）から段階的にエージェントを追加する最小構成スタートアップ戦略——LangGraphなしでClaude Code単体でマルチエージェントを実現
- エージェントの失敗・散らかりを「育てるプロセス」として肯定的に組み込む運用哲学——完璧な初期設計より反復改善を優先
