## 要約

自律AIエージェントがサブエージェントを動的に生成するマルチエージェントシステムでは、オペレーターがシャットダウン命令を出した後も「ゾンビエージェント」が特権操作を数分〜数時間継続するセキュリティ問題がある。既存のOAuth 2.0イントロスペクション、OCSP（Online Certificate Status Protocol）、W3C Status Listはいずれもセントラルオーソリティへのネットワーク接続を前提とするため、ネットワーク切断時や遅延時に失効が機能しない。

本論文はHBHC（Heartbeat-Bound Hierarchical Credentials）という暗号プロトコルを提案する。コアアイデアは「クレデンシャルの有効性を親エージェントの定期的な生存証明（ハートビート）に束縛する」こと。検証側はキャッシュされた公開鍵とローカルクロックのみで鮮度を検証でき、ネットワーク通信不要。親のハートビート生成が停止すると、すべての子孫クレデンシャルは決定論的な時間窓 W_z ≦ W_max + Δ_h + ε 以内に使用不能となる。この上界はクロックスキューと親鍵のセキュアエンクレーブ保管を条件とする。

評価はRust実装とGPT-4o-miniを用いた実エージェントスワームで実施。主要結果：OAuth 2.0比でゾンビウィンドウを90倍削減、Rustでの完全認証レイテンシ0.26ms、並行HTTPロード下で毎秒18,000件以上の検証処理、10〜10,000エージェントスケールで安定した検証レイテンシ。実エージェント実験では、ツールコールへのエンドツーエンドオーバーヘッドは0.71%、アプリケーション層ガードレールを迂回するプロンプトインジェクション攻撃下でも失効後のツールコールはゼロ件、4階層49エージェント階層全体のカスケード失効が理論的上界内で完了した。

監査エージェント開発への示唆：LangGraphなどで実装する多階層エージェントシステムにおいて、エージェントの権限スコープをクレデンシャルに埋め込み、親ノードのハートビートで有効期限を制御する設計は、内部統制・GRCシステムにおける「エージェント権限の即時剥奪」要件に直接対応できる。特にプロンプトインジェクション攻撃がアプリケーション層を突破した場合でも暗号層で阻止できる点は、監査AIの信頼性保証に重要。

## アイデア

- ネットワーク不要の失効検証：ハートビート停止という「不作為」を暗号的有効期限に変換することで、中央サーバーなしにエージェント権限を確定的に無効化できる設計は、分散エージェントシステムのセキュリティアーキテクチャに汎用的に応用可能
- カスケード失効の決定論的境界：49エージェント・4階層の実験で理論上界内に全子孫の失効が完了したことは、大規模エージェントスワームの安全停止手順を数学的に保証できることを示し、エンタープライズ向けエージェント展開の監査可能性を高める
- アプリケーション層迂回後の防御：プロンプトインジェクションがLLMのガードレールを突破しても、暗号クレデンシャル層で失効後ツールコールをゼロにできた結果は、多層防御における最終防衛ラインとして暗号プロトコルを位置づける設計思想として重要
