## 要約

本稿はThe Gradientに掲載された約35分読みの長編論考で、AIアライメントにおける目標指向型（goal-directed）エージェント設計の根本的な問題を哲学的に再検討する。著者の中心的主張は「合理的な人間は目標を持たない、合理的なAIも目標を持つべきではない」というものだ。

従来のAIアライメント研究、特にEffective Altruism（EA）コミュニティが前提とする「直交性テーゼ（Orthogonality Thesis）」——知性と目標は独立しており、いかなる目標もいかなる知性と組み合わせ可能——に対し、著者はアリストテレス的な「エウダイモニア（eudaimonia：能動的・合理的な人間的繁栄）」概念を援用して対案を提示する。

提案する枠組みは「エウダイモニア的合理性（eudaimonic rationality）」と呼ばれ、行動・行動傾向・行動評価基準・行動リソースのネットワークである「実践（practice）」を基盤とする。数学者がテリー・タオの例で示されるように、優れた数学者は「数学的卓越性を数学的に促進する」——つまり手段と目的の区別がなく、行動はメロディにおける音符のように実践の要素として評価される。

著者はこの構造を「x をx的に促進する（promote x x-ingly）」という公式で定式化する。芸術的であるとは芸術を芸術的に促進することであり、誠実であるとは誠実さを誠実に促進することだ。これはEAスタイルの功利計算（最小の手段で最大の目的を）とは本質的に異なる型（type）を持つ。

AIアライメントへの含意として、「無害性（harmlessness）」「修正可能性（corrigibility）」「透明性」といったセーフティ特性は、目標・ルール・効用関数として表現すると脆弱で恣意的になるが、エウダイモニア的実践のダイナミクスとして解釈すると自然で安定するという。つまり人間の価値と合理性の間の「型不一致（type mismatch）」を解消することが、アライメントの本質的な問いだと論じる。

論文は6部構成で、Part V でEA型オプティマイザーが前提する場合に生じるアライメントの「パラドックス」群（コントロール問題、価値学習の困難等）がこの型不一致に起因すると論じ、Part VI ではエウダイモニア的に合理的なAIエージェントが人間の実践に従事するとはどういうことかを検討する。

監査エージェント開発への示唆：「修正可能性（corrigibility）」を効用関数の制約として設計するのではなく、監査実践のダイナミクス（監査を監査的に促進する）として実装するアプローチは、LangGraphのステートフルなエージェント設計や、ReActループにおける自己評価基準の構造化に直接応用できる可能性がある。

## アイデア

- 「promote x x-ingly」公式：エージェントの目標を効用関数ではなく実践のダイナミクスとして定義することで、修正可能性・透明性・無害性をより自然に表現できる可能性がある
- 手段と目的の区別の消滅：エウダイモニア的合理性では instrumental/terminal values の区別が不自然とされ、これはLLMのCoT推論が「中間ステップ」を道具的ではなく内在的に評価している挙動と共鳴する
- 型不一致（type mismatch）フレーム：人間の価値を効用関数に翻訳しようとすること自体が情報損失を生むという視点は、RLHF/RLAIFにおける報酬ハッキング問題の哲学的説明として有用
