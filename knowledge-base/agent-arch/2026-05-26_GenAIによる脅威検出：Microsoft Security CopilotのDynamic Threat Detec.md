## 要約

本論文は、Microsoftが開発したDynamic Threat Detection Agent（DTDA）を紹介する。DTDAはMicrosoft Defender上でセキュリティインシデントを継続的に調査し、見落とされた脅威を検出して説明可能なアラートを自動生成する自律型エージェントである。

従来のサイバー脅威検出は、セキュリティアナリストが攻撃者の手口の変化を常に追いながら検出ロジックを手動更新する必要があり、防御側が常に後手に回る構造的問題を抱えていた。DTDAはこの課題を、4つの主要コンポーネントによって解決する。

第一に「統合アクティビティタイムライン」：アラート、イベント、ユーザー・エンティティ行動分析（UEBA）、脅威インテリジェンスを一元的に統合した時系列ビューを構築する。第二に「バージョン管理されたLLMプロンプトコントラクト」：スキーマバリデーション、グラウンディング要件、再試行上限、フェイルクローズ型のサプレッション機構を持ち、LLMの出力品質と安全性を担保する。第三に「プランナー・エグゼキューター調査ループ」：攻撃固有の仮説を生成し、支持・反証証拠を収集するReActスタイルのループ構造。第四に「動的アラート生成」：文脈に応じたタイトル、深刻度、MITREマッピング、修復ガイダンス、関連エンティティ、自然言語による攻撃説明を含むアラートを出力する。

評価結果は定量的に優れている。120日間のオンライン評価では顧客フィードバックに基づく適合率80.1%を達成し、調査対象インシデントの約15%で従来未検出の新規アラートを生成した。オフライン評価ではGPT-5.4使用時にF1スコア0.78を記録し、GPT-4.1比で+0.12、ベースライン比で+0.26の改善を示した。運用面では、1インシデントの調査をメディアン28分、トークンコストメディアン2.04ドルで処理し、ジョブレベルの失敗率は0.38%と低水準に抑えられている。

DTDAはすでにMicrosoft Security Copilotに統合され、数万のDefenderカスタマーに対して継続稼働中である。監査エージェント開発への示唆として、プランナー・エグゼキュータループによる仮説駆動型証拠収集パターンは、監査証拠の自動収集・異常検知にそのまま転用可能である。また、LLMプロンプトコントラクトのスキーマバリデーションとフェイルクローズ設計は、監査の信頼性要件（虚偽アラートの抑制）に直接対応するアーキテクチャパターンである。

## アイデア

- LLMプロンプトコントラクトにスキーマバリデーション・グラウンディング要件・フェイルクローズ機構を組み込むことで、本番環境でのLLM出力の信頼性を担保する設計パターンは、監査AIにおける虚偽アラート抑制にも直接応用できる
- 仮説生成→証拠収集（支持・反証）→アラート生成というプランナー・エグゼキューターループは、ReActパターンをセキュリティドメインに特化させたもので、調査型エージェントの汎用アーキテクチャとして参照価値が高い
- GPT-4.1からGPT-5.4へのモデル更新でF1が+0.12向上した事実は、基盤モデルの世代差がエージェント性能に直接反映されることを示しており、監査エージェントのモデル選定指針となる
