## 要約

エンジニアが「設計書を書く時間 > 設計を考える時間」という問題意識から開発したSaaS「ArchitectAI」の技術的詳細と設計判断を解説した記事。仕様書・RFPを入力とし、要件定義書・アーキテクチャ設計書・AWS構成図（draw.io XML）・コスト見積・提案書を自動生成する。対象は要件定義〜アーキテクチャ設計フェーズで、SIerの基本設計・詳細設計書は対象外。

最大の技術的課題は出力品質の安定性だった。初期プロトタイプではClaude単体で生成していたが、同じ入力でも「可用性設計がしっかりした設計書」と「シングルAZ・冗長性ゼロの設計書」が混在する品質ブレが発生。自己レビューでは評価が甘くなる問題も確認された。

これを解決するために「Multi-LLM品質ゲート」を実装した。Claude・GPT・Geminiの3社が独立してスコアリングし、全社が合格点（≥8.0/10）を出すまで再生成するループ機構。ソフトウェアレビューの「複数レビュアー」に相当するアプローチで、品質の安定性が大幅に向上した。

設計フローは企画フェーズから移行方針まで15ステップに整理。重要な設計判断として「どのStepからでも開始できる」柔軟性を持たせた。Step9（アーキテクチャ設計書）のみの利用も、Step1〜14の全フロー実行も同一サービスで対応可能。

draw.io構成図の自動生成が最難関だった。テキスト系設計書（DOCX）と異なり、XMLの「見た目」が重要なため、ノード重複・凡例欠損・AWSアイコン非使用などの問題が頻出。解決策として「2-passレビューフロー」を実装：①初稿生成 → ②3社AIが6軸でレビュー・フィードバック収集 → ③フィードバックを反映した改善版を合否に関わらず必ず生成 → ④最終チェック記録。「合格でも改善版を生成する」設計により、図の平均品質スコアが6.3から7.8に向上した。

料金設計はクレジット制（月次配布＋追加購入）＋プラン別機能制限の組み合わせ。FREEプランは10クレジット使い切り型（月次リセットなし・クレジットカード不要）で、企画フェーズ全体（Step0〜3、計7cr）を無料で完走可能。PROプランは9,800円/月・125クレジットで、フルプロジェクト設計（30cr）を月4件相当こなせる想定。ITコンサル・SIer向けに設計されている。

監査エージェント開発への示唆：Multi-LLM品質ゲート（複数モデルによる独立スコアリング＋閾値判定ループ）は、監査エージェントにおける判断結果の信頼性検証に直接応用できるパターン。また2-passレビューフロー（生成→フィードバック→強制改善）は、監査調書の品質保証プロセスに組み込める設計手法として参考になる。

## アイデア

- Claude・GPT・Gemini 3社が独立してスコアリングし全社≥8.0/10を要求するMulti-LLM品質ゲートは、LLM-as-judgeの実用的な実装パターンとして監査エージェントの判断検証にも転用できる
- 「合格でも改善版を必ず生成する」2-passレビューフローは、初回合格時の出力を最低品質ラインではなくベースラインとして扱う設計思想で、構造化アウトプット（XML/JSON）の品質向上に特に有効
- 15ステップの設計フローを「どのステップからでも開始できる」設計にすることで、フルフロー利用とピンポイント利用を同一サービスで吸収するアーキテクチャは、エージェントパイプラインのモジュール設計原則としても参考になる
