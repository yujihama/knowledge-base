## 要約

本書はClaude Codeを用いた個人開発の完全自動化・無人化を目的とした実践的な運用ガイド（全8章、約55,860字、1,200円）。単なる使い方の紹介ではなく、エージェントを継続的に稼働させるための「運用設計」に焦点を当てる。

主要トピックは以下の通り。①CLAUDE.mdを「エージェントの憲法」として設計し、振る舞い・制約・目的を明文化する方法。②権限スコープ・サンドボックス・安全設計による意図しない副作用の防止。③スキルとスラッシュコマンドによるタスクの構造化と再利用性向上。④フック・cron・スケジュールエージェントを組み合わせたイベント駆動・時刻駆動の自動実行パイプライン構築。⑤サブエージェントの並列起動による複数タスクの同時処理とワークフロー設計。⑥Claude OpusやSonnet等モデル選択の判断基準とトークンコスト最適化。⑦実運用での失敗パターン（ループ・暴走・誤操作等）の分類と回復手順。

アーキテクチャ上の特徴として、Claude Codeのフック機能（tool_call前後に任意シェルコマンドを実行）とcronベースのスケジューラを組み合わせることで、人間の介在なしに定期タスクを完結させる設計を解説する。サブエージェント並列化では、独立したworktreeを持つ複数エージェントが並行して異なるタスクを処理するパターンを扱う。

監査エージェント開発への示唆：CLAUDE.mdによる制約明文化はLangGraphのノード定義における守備範囲設計と対応する。フック機構はReActループの各ステップに検証・ロギングを挟む実装パターンに応用可能。また失敗パターン集は本番エージェントのガードレール設計に直接活用できる。

## アイデア

- CLAUDE.mdを『エージェントの憲法』として捉え、振る舞い・権限・目的を宣言的に記述することで、プロンプト外からエージェントの行動空間を制御するアーキテクチャパターン
- フック（tool_call前後のシェル実行）＋cronスケジューラの組み合わせにより、LLMエージェントをデーモンプロセスとして常駐・自律稼働させる設計思想
- worktreeベースの並列サブエージェント起動により、コンテキスト汚染なしに独立タスクを同時処理する並列ワークフロー構成
