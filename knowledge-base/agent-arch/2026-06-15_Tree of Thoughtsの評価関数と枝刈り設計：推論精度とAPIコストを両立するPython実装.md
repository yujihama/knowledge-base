## 要約

Tree of Thoughts（ToT）は木探索を用いてLLMの推論精度を高める手法で、Yao et al.（2023）の原論文ではGame of 24タスクでCoTの4%に対し74%の正答率を達成している。一方でノードあたり3-5回のAPI呼び出しが必要となり、CoT比10-100倍のトークン消費が発生するコスト問題がある。本記事はその評価関数と枝刈り戦略の設計パターンを体系化したもの。

評価関数は3パターンに分類される。①LLM自己評価（分類方式）：LLMにsure/likely/impossibleの3値分類をさせ、n=3のサンプリングで確率的にスコアを算出。temperature=0.7が必須で、temperature=0では分散が失われる。②スコアリング方式：1-10の数値を直接出力させる方式。分類方式より粒度は細かいが出力が不安定なため、パースエラー時のフォールバック（0.5）が必要。③ドメイン特化ヒューリスティック：プログラムで評価関数を実装しAPI呼び出しをゼロにする方式。Game of 24ではJaccard距離と数値近傍性を組み合わせたスコアを使用。

枝刈り戦略は4パターン。①閾値ベース：スコア0.3未満を除外するシンプルな方式。②ビームカットオフ：各深さで上位k件を残す方式で、最大ノード数はN_max = k×b×d（b:分岐数、d:深さ）で予測可能。③多様性考慮：スコアとJaccard距離を(1-w)×score + w×distanceで統合し、意味的に類似した思考の集中を防ぐ。diversity_weight=0.2-0.3が推奨。④適応的枝刈り：スコアの分散と最大値から問題難易度を推定し、ビーム幅をmin_beam～max_beamの間で動的調整する2026年最新手法（DST・LiteSearch統合）。

APIコスト削減の主要手法は評価用モデルの分離で、生成をGPT-4o、評価をGPT-4o-miniに分担することで精度劣化2%以内に抑えながら約50%のコスト削減を達成。DST（Nie et al., 2026）は軽量な教師付きモデルを評価器として用い26-75%削減、LiteSearch（Wang et al., AAAI 2025）は動的探索バジェットでgreedy比10倍未満に抑制する。非同期実装（asyncio）により評価の並列化も可能で、同一深さのノードを並行評価することでレイテンシを削減できる。

監査エージェントへの応用観点では、複数の監査仮説を並列探索しながら低品質な推論パスをコスト効率よく刈り込む設計に直接転用可能。特にドメイン特化ヒューリスティックと適応的枝刈りの組み合わせは、監査ルールベースの評価器と親和性が高い。

## アイデア

- 評価モデルと生成モデルを分離（GPT-4o生成 + GPT-4o-mini評価）することで精度劣化2%以内でコストを50%削減できるという具体的な数値が示されており、実務設計の直接的な判断基準になる
- スコア分散と最大値から問題難易度をリアルタイム推定し、ビーム幅を動的調整する適応的枝刈りは、難易度の事前見積もりが困難な監査タスクで特に有効な設計パターン
- 多様性枝刈りでJaccard距離を用いてスコアと探索の多様性をトレードオフする手法は、類似監査仮説への過集中を防ぐための実装アイデアとして転用可能
