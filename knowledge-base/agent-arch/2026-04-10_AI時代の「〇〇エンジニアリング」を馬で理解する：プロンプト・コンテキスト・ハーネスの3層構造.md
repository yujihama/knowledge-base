## 要約

ChatGPT登場以降に生まれた「プロンプトエンジニアリング」「コンテキストエンジニアリング」「ハーネスエンジニアリング」の3概念を、「賢い馬に1分でグラウンドを一周させる」という比喩で整理した記事。

**プロンプトエンジニアリング（2022年〜）**は「何をさせるか」の設計で、成功条件を曖昧にしない指示文の工夫が本質。「グラウンドを一周」という曖昧な指示では馬が往復を「一周」と解釈するように、AIへの指示は「白線に沿って・ショートカットせず・スタート地点に戻る形で」と明示する必要がある。

**コンテキストエンジニアリング（2025年〜）**は「何を見せるか」の設計で、AIに渡す情報全体のマネジメント。AnthropicとOpenAIはともに「コンテキストは有限資源」「巨大な説明書は重要情報を埋もれさせる」と指摘しており、必要な情報を必要なタイミングで必要量だけ渡すことが重要。事前のシステムプロンプト・知識ベースだけでなく、走行中の偵察情報＝RAGやツール実行結果のリアルタイムフィードバックまで含む。

**ハーネスエンジニアリング（2026年〜）**は「どう動かし、確かめ、直すか」の設計で、モデル外側の実行系全体を指す。4要素として①Tools（蹄鉄：ファイルI/O・Web検索・コード実行等のツール定義）、②Guardrails（柵：パーミッション制御・サンドボックス・操作制約）、③Observation（センサー：フック・ログ・モニタリング）、④Verification（ラップタイム：テスト・CI・eval・改善サイクル）を挙げる。

3つは「どれか1つを選ぶ」ものではなく積み重なるレイヤーとして機能する。単純なChatGPT利用はプロンプトのみで足りるが、RAG活用はコンテキストが加わり、「1時間かけてコードベース全体をリファクタリング」のような自律エージェント用途ではハーネスまで含めた3層すべてが必要になる。参考文献としてMartin Fowlerの「Harness engineering for coding agent users」、AnthropicのEffective context engineering、OpenAIのHarness engineeringが挙げられている。

## アイデア

- コンテキストは「事前のメモ（システムプロンプト）」と「走行中のリアルタイム情報（RAG・ツール結果）」の両方を含むという整理は、RAGをコンテキスト設計の一部として統一的に捉えるフレームとして有用
- ハーネスの4要素（Tools・Guardrails・Observation・Verification）は、自律エージェントの信頼性評価軸として使えるチェックリストになっており、実装レビューの観点として実践的
- 「複雑さのレベルに応じて必要なレイヤーが増える」という段階的設計思想は、過剰設計を避けながらエージェント化を進める際の判断基準として機能する
