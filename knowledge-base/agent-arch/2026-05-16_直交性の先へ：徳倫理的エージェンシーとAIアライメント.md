## 要約

本論文はThe Gradientに掲載された長編エッセイで、AIアライメントの主流パラダイムである「目標最適化エージェント（Effective Altruism型）」への根本的な批判を展開する。著者の中心的主張は「合理的な人間は目標（goals）を持たず、合理的なAIも目標を持つべきではない」というものだ。

人間の合理的行為は目標への最適化ではなく「プラクティス」——行為・行為傾向・評価基準・行為リソースが相互に促進し合うネットワーク——に依拠する。数学者は抽象的な「人類幸福最大化」を目標に数学をするのではなく、「数学的卓越性を数学的卓越性をもって促進する」という構造で活動する。これを著者は「ユーダイモニック合理性（eudaimonic rationality）」と呼ぶ。

アリストテレスの幸福論（eudaimonia）に基づくこのフレームワークでは、手段と目的、道具的価値と終端的価値の区別が自然に溶解する。音符がメロディの要素であるように、個々の行為は実践ネットワークの要素として意味を持つ。

AIアライメントへの含意として、著者は以下を論じる。まず「タイプミスマッチ」問題：人間の価値（ユーダイモニック構造を持つ）をEffective Altruism型最適化AIが解釈しようとすると、harmlessness・corrigibility・transparencyといった概念が不自然で脆弱なものになる。しかしユーダイモニック的に解釈すれば、これらは実践ネットワークの自然な動態として安定する。

次に「promote x x-ingly」の公式：真の誠実さとは誠実に誠実さを促進すること、真の親切さとは親切に親切さを促進すること。この構造がAIの安全性プロパティにも適用できると著者は主張する。たとえばcorrigibleなAIとは「修正可能性をcorrigibleな方法で促進する」エージェントであり、これは単なるルール（「訂正を受け入れよ」）より安定したアーキテクチャ的特性になりうる。

論文はさらに、ユーダイモニック合理性がMLトレーニングで学習可能かという問いにも踏み込む。人間の文化的進化が収束してきたこの形の合理性は、アルゴリズム複雑度が低く、ニューラルネットワーク型AIとの収束進化の可能性もあると論じる。

監査エージェント開発への示唆：監査AIにおけるharmlessness（誤判定の回避）やtransparency（説明責任）を終端目標として設定すると、エッジケースで脆弱になりやすい。ユーダイモニック的アプローチでは、「透明性のある監査を透明性をもって促進する」という実践ネットワークとして設計することで、より頑健な監査エージェントアーキテクチャが構築できる可能性がある。

## アイデア

- 「promote x x-ingly」公式：harmlessnessやcorrigibilityをルールや目標関数ではなく実践ネットワークの動態として設計することで、エッジケースでの安定性が向上するという設計原則
- 手段・目的の区別が消える実践ネットワーク構造：LangGraphのような実行グラフにおいて、ノード間の遷移を「道具的→終端的」ではなく「プラクティス内の相互促進」として設計するアーキテクチャ的含意
- ユーダイモニック合理性の学習可能性：人間文化で収束進化してきたこの合理性パターンはアルゴリズム複雑度が低く、RLHFやRLAIFのような強化学習ベースの訓練で直接ターゲットにできる可能性
