## 要約

ADEMAは、LLMエージェントが長時間にわたるタスク（Long-Horizon Task）を実行する際に生じる「知識状態のドリフト」「暗黙的な中間コミットメント」「中断による証拠チェーンの断絶」という3つの失敗モードに対処するために設計された知識状態オーケストレーションアーキテクチャである。

従来のマルチエージェントランタイムとは異なり、ADEMAは汎用的なエージェント実行基盤ではなく、知識合成プロセスの認識論的状態管理に特化している。主要な設計コミットメントは以下の通り：

1. **明示的な認識論的簿記（Explicit Epistemic Bookkeeping）**：各ラウンドで確立された知識状態を明示的にトラッキングし、暗黙的な前提が次ラウンドに引き継がれる問題を防ぐ。

2. **異種デュアル評価ガバナンス（Heterogeneous Dual-Evaluator Governance）**：2種類の異なる評価器を組み合わせてタスクの品質と進捗を監視し、エージェントの軌跡規律（Trajectory Discipline）を維持する。

3. **適応的タスクモード切替（Adaptive Task-Mode Switching）**：タスクの性質や進捗状況に応じて動的に処理モードを切り替えるガバナンス機構。

4. **チェックポイント・再開可能な永続化（Checkpoint-Resumable Persistence）**：中断が発生しても証拠チェーンを維持したまま処理を再開できる。固定60ランのメカニズムマトリクス実験において、チェックポイント/再開機能の除去が唯一の無効実行を引き起こした（中断感受性条件下）。

5. **セグメントレベルのメモリ圧縮（Segment-Level Memory Condensation）**：長期タスクでのコンテキスト肥大化を防ぐため、処理済みセグメントのメモリを圧縮・要約して保持。

6. **アーティファクトファースト組み立て（Artifact-First Assembly）**：最終回答ではなく中間成果物（アーティファクト）を明示的に進行させることで、証拠を持つアーティファクト進行（Evidence-Bearing Artifact Progression）を実現。

7. **評判ベースのリソース配分（Reputation-Shaped Resource Allocation）**：エージェントの過去のパフォーマンスに基づいてリソースを動的に割り当てる。

実験は4シナリオのショーケースパッケージと固定60ランのメカニズムマトリクスで構成され、マイクロアブレーション実験も実施された。デュアル評価・セグメント合成・動的ガバナンスは完了の必須前提条件というより、コスト品質トレードオフを最適化する支援制御機構として機能することが示された。コード指向の評価が最も品質感受性の高いメカニズムブロックであることも確認されている。

監査エージェント開発への示唆として、長時間にわたる内部統制評価や監査証跡の収集プロセスにおいて、中断・再開可能な証拠チェーン管理とデュアル評価によるガバナンスは直接応用可能な設計パターンである。特に複数ラウンドにわたる監査手続きの知識状態管理に有効と考えられる。

## アイデア

- 「認識論的簿記（Epistemic Bookkeeping）」という概念：エージェントが何を知っており、何を確定させたかを明示的に記録することで、長期タスクでの知識ドリフトを防ぐ設計思想は、監査エージェントの証拠管理に直接応用できる
- チェックポイント/再開が唯一の必須メカニズムであり、デュアル評価やセグメント合成は「あると品質が上がるが必須ではない」という実証的な階層化は、エージェント設計の優先順位付けに実用的な指針を与える
- 「アーティファクトファースト」設計：最終回答ではなく中間成果物の明示的な進行を追跡することで、長期タスクの進捗可視化と品質管理を両立させるアプローチは、LangGraphのステート管理パターンと親和性が高い
