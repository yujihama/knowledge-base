## 要約

DS-STARはGoogleが開発したデータサイエンス自動化エージェントで、CSV以外のJSON・非構造化テキスト・Markdownなど異種データ形式を横断して統計分析・可視化・データ加工タスクを自律実行する。DABStep・KramaBench・DA-Codeの3ベンチマークで既存のAutoGen・DA-Agentを上回り、DABStepの公開リーダーボードで首位（2025年9月18日時点）を獲得した。

アーキテクチャは2フェーズで構成される。第1フェーズでは「Data File Analyzerエージェント」がディレクトリ内の全ファイルを自動解析してテキスト形式のコンテキストサマリを生成する。このサマリがなければDABStardの難問正解率は45.2%から26.98%に急落することがアブレーション実験で示されており、データ文脈理解の重要性が定量的に裏付けられている。

第2フェーズは「Planner → Coder → Verifier → Router」の4エージェントによる反復ループである。Plannerが高レベル計画を立案し、Coderがコードに変換、VerifierがLLMベースの判定でその十分性を評価する。不十分と判定された場合、Routerが「ステップ追加」か「既存ステップ修正」かを判断して計画を改訂し、再度Coderが実装する。このサイクルを最大10ラウンド繰り返す仕組みで、Google Colabでエキスパートが中間結果を確認しながら段階的に作業を進める手順を模倣している。

性能比較では、DABStepで先行最良手法から41.0%→45.2%、KramaBenchで39.8%→44.7%、DA-Codeで37.0%→38.5%へ改善した。難問タスク（複数ファイル横断）での改善幅が特に大きく、異種データ統合処理における差別化が確認される。反復回数分析では、難問は平均5.6ラウンド、易問は平均3.0ラウンドを要し、易問の50%超は初回ラウンドで完結する。

LLM互換性についても検証されており、Gemini-2.5-ProだけでなくGPT-5でもフレームワークが有効に動作した。GPT-5版は易問で優位、Gemini版は難問で優位という傾向が観測されている。

## アイデア

- Verifier+Routerによる『ステップ追加 vs ステップ修正』の分岐判断が精度向上の鍵であり、単純なステップ積み上げより誤りの修正を優先するループ設計が有効
- Data File Analyzerで異種フォーマットを事前にテキストサマリ化してコンテキスト注入する手法は、構造化・非構造化が混在する実務データへのエージェント適用の汎用パターンとして応用可能
- 難問ほど反復回数が増える自然な収束特性は、コスト制御（max_rounds=10）と品質のトレードオフをベンチマークで可視化できる点で評価設計の参考になる

## Yujiの取り組みへの示唆

監査エージェントでも財務データ・契約書・ログなど異種フォーマットが混在するため、DS-STARのData File Analyzerによる事前コンテキスト抽出パターンはLangGraphのノードとして組み込める実装参考になる。Verifier（LLM-as-judge）が計画の十分性を評価してRouterが修正・追加を判断する構造は、監査手続の妥当性検証ループとして直接対応づけられる。さらにPydanticで計画オブジェクトをスキーマ化し、各エージェント間の入出力を型安全に管理することで、LangGraphの状態遷移をより堅牢に設計できる。
