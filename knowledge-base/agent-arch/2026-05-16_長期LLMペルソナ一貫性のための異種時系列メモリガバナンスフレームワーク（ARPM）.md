## 要約

大規模言語モデル（LLM）は長期対話において、事実の損失・タイムライン混乱・ペルソナドリフト・安定性低下という4つの問題を抱える。特に高ノイズ知識ベース・コンテキストクリア・クロスモデル転送の条件下でこれらが顕著になる。本論文ではARPM（Adaptive Retrieval-augmented Persona Memory）という外部時系列メモリガバナンスフレームワークを提案する。ARPMの核心的な設計思想は「ペルソナ一貫性をモデル重みに埋め込むのではなく、追跡・監査・転送可能なガバナンス問題として扱う」という点にある。

技術的な構成として、ARPMは静的知識メモリと動的対話経験メモリを分離管理する。検索には①ベクトル検索（意味的類似性）、②BM25（語彙的マッチング）、③RRF（Reciprocal Rank Fusion）による融合ランキングを組み合わせる。さらに、デュアル時系列リランキングにより時間軸上の証拠の優先度を調整し、時系列順の証拠読み取りと制御分析プロトコルにより証拠検証と回答バインディングを実施する。

3つの実験が行われた。第1実験は50ラウンドのQ&A設定でノイズ比1:5と1:200+を比較。CSV自動判定ではノイズ1:5でリコール精度54.0%、1:200+で44.0%だったが、手動レビューではそれぞれ100.0%・80.0%に向上。自動ルールはプロンプト内に支持証拠が存在する場合にリコールを過小評価することが示された。第2実験（アブレーション）では対話履歴検索を無効化すると厳密精度が100%から66.7%に低下、BM25無効化では80.0%に低下し、純粋な意味的検索だけでは訂正・追跡が不十分であることが確認された。第3実験では510万文字のノイズ基盤・定期的コンテキストクリア・マルチモデルハンドオフという極限条件下でARPMが意味的継続性・境界継続性・ペルソナ一貫性を維持したが、プロトコル準拠が弱い場合の限界も露呈した。

監査エージェント開発への示唆として、ARPMの「ホワイトボックス評価可能なガバナンス構造」という設計哲学は、監査エージェントにおける証跡管理・根拠の追跡可能性・複数エージェント間での知識転送に直接応用できる。特にコンテキストウィンドウをまたいだ長期セッションでのエージェント一貫性維持と、静的ルール知識と動的対話履歴の分離管理は、監査AIの信頼性確保に有効なアーキテクチャパターンである。

## アイデア

- ペルソナ一貫性をモデル重みやロングコンテキストではなく『追跡・監査・転送可能なガバナンス問題』として外部化する設計思想は、監査エージェントの根拠管理アーキテクチャとして直接応用可能
- 静的知識メモリと動的対話経験メモリの分離＋BM25・ベクトル検索・RRFの融合により、意味的検索単独では拾えない時系列的な訂正・修正履歴を追跡できる点が新規性
- 510万文字ノイズ基盤・定期コンテキストクリア・マルチモデルハンドオフという実験設定は、実運用LLMシステムの評価プロトコルとして参考になり、CSV自動判定の限界（証拠がプロンプト内にある場合のリコール過小評価）の発見は実用的な知見
