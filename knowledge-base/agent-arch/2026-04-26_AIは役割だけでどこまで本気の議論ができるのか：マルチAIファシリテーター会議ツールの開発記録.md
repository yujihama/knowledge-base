## 要約

複数AIエージェントが多角的に議論するプロセスを可視化するWebUIツールを個人開発した約1ヶ月の記録。構成はNext.js + nginx + MySQL + Docker で、CLAUDE.md（約300行）を「AIの脳」として会議進行ロジックを自然言語で管理する。ストレージは三層設計：会議横断の正データはMySQL、会議内ベクトル検索はSQLite + sqlite-vec（multilingual-e5-small 384次元）、リアルタイムログはブラウザ側のsql.js + IndexedDB。AIはMCP経由でのみSQLiteに触れる設計とし、MySQL直アクセスは禁止している。

技術判断の変遷として、まず自前SSEオーケストレーター（Node.js 800行）で基本動作を実現したが、コードで会議ロジックを書くほど想定外への対応が困難になることを確認。その後、Python製Swarmsを数日試したが日本語リアルタイム進行制御で重複発言・ハングアップが発生し不採用。Claude Agent Teamsもlead自律性の調整困難・トークン消費の重さから却下。最終的にclaude -p --agentsを使ったサブエージェント方式（ファシリテーターが必要時にパネリストをサブエージェントとして呼び出す構造）に着地した。

コスト最適化の実測値も詳述されており、ベクトル検索導入当初はコストが35%悪化したが、再設計により最大49.5%削減を達成。APIモード（安定運用）とCLIモード（claude -p、開発反復用）の二モード構成を採用し、CLIモード最適化後は$1〜3/回に抑制（最適化前は数回の会議で$15相当を消費）。

会議介入機構としてstatus.jsonファイルへの書き込み（pause/continue/feedbackの3状態）を実装し、シンプルなファイル書き換えのみでファシリテーターの挙動を変更できる設計とした。過去会議の引き継ぎはminutes.mdやnotes/をファイルベースで保全し、新会議のdata/prev/に配置してembeddingを新SessionDBに投入する方式を採用。監査エージェント開発への示唆として、自然言語指示書（CLAUDE.md）でロジックを管理するアプローチはReActエージェントの判断ルール定義にも応用可能であり、MCPによるツールスコープ制限（AIが触れる範囲を人間が明示的に制御）はエージェント安全設計の具体例として参考になる。

## アイデア

- 会議進行ロジックをコードではなくCLAUDE.md（自然言語指示書）で管理することで、条件分岐の複雑化を回避しつつAIに柔軟な判断を委譲するアーキテクチャ
- status.jsonへの単純なファイル書き込みでAIプロセスに介入できる軽量な人間-AI協調インターフェース（pause/continue/feedback）
- MySQL・SQLite・IndexedDBの三層ストレージ設計により、AIがアクセスできるデータ範囲を人間がprev/ディレクトリへの配置で明示的にコントロールする構造
