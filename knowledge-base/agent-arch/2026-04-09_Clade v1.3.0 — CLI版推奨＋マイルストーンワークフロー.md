## 要約

CladeはClaude Codeを活用したマルチエージェント開発ワークフローツール。v1.3.0では主に2つの変更が加わった。

1つ目はCLIを推奨実行環境にした点。VS Code拡張では、複数エージェントをバックグラウンドで並列実行する際に確認ダイアログが表示できないバグが存在し、処理が無言で止まったり承認フローがスキップされる問題があった。CLI（ターミナル）ではこの問題が発生しない。v1.3.0からはセッション開始時に実行環境を自動検出し、VS Code拡張と判断した場合は警告メッセージを表示して「逐次実行で続けるか（yes）」「CLI切り替え案内を表示して終了するか（no）」をユーザーに選択させる。yesを選んだ場合は並列バックグラウンド実行を無効化した状態でセッションを継続し、noを選んだ場合はCLI移行手順（統合ターミナル起動→claudeコマンド→/terminal-setup→/init-session）を案内する。

2つ目はマイルストーンワークフローの追加。数時間以上かかる機能開発や複数セッションにまたがる開発では、「実際に動かして確認できる状態」を単位とした自然な区切りが必要になる。プランナーがマイルストーンを含むplan-reportを作成する際、承認前に挙動モードを質問する。「confirm」モードは各マイルストーン完了・コミット後に続行確認を表示し、途中停止と再開が可能。「autoモード」は確認なしで自動的に次のマイルストーンへ進む。選択結果はplan-reportのメタ情報セクションに記録され、デベロッパーが作業開始時に参照する。

その他の修正として、sandbox設定のフォーマットが真偽値（true）からオブジェクト形式（enabled/autoAllowBashIfSandboxed/allowedPathsを持つ）に変更され、CLIでのパースエラーによりtool実行のたびに確認プロンプトが出る問題が解消された。また、httpProxyPort: nullとsocksProxyPort: nullがバリデーションエラーを引き起こしていた問題も、未使用時はフィールドを省略する形で修正された。アップグレードはsetup.ps1（Windows）またはsetup.sh（macOS/Linux）の再実行で対応できる。

## アイデア

- VS Code拡張のUIとCLIでエージェント並列実行の挙動が異なるという環境差異を実行時に自動検出し、ユーザーに選択肢を提示する設計は、エージェントシステムのデプロイ環境多様性への現実的な対処パターン
- 「タスク完了」ではなく「実際に動かして確認できる状態」をマイルストーンの粒度として定義する考え方は、エージェントによる長時間タスク実行における人間のオーバーサイト設計として参考になる
- plan-reportのメタ情報にmilestone_modeを明示的に記録することで、エージェントがセッション跨ぎで動作方針を引き継ぐ実装パターンは、マルチセッション対応エージェントの状態管理の一例

## Yujiの取り組みへの示唆

LangGraphで監査エージェントを構築する際、長時間・複数セッションにわたる監査タスクの実行制御にCladeのマイルストーンワークフロー設計が直接参考になる。特に「人間が介入して結果を確認してから次ステップに進むconfirmモード」は、監査における承認・レビューゲートをエージェントフローに組み込む際の設計パターンとして活用できる。また、並列エージェント実行時の確認ダイアログ問題への対処（環境検出→フォールバック提示）は、監査エージェントを複数の実行環境（CI、ローカル、VS Code等）で動かす際の堅牢性設計の参考になる。
