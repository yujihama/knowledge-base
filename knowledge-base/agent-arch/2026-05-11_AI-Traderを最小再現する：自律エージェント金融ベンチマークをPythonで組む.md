## 要約

論文「AI-Trader: Benchmarking Autonomous Agents in Real-Time Financial Markets」（arXiv:2512.10971、2025年12月公開）の中核アイデアを、標準ライブラリのみで動く最小ベンチマークとして再現した実装記事。

AI-Traderの問題意識は、静的なQAやバックテストでは動的環境における意思決定能力を測れないという点にある。論文はリアルタイム金融市場を舞台に、LLMエージェントをライブ評価するベンチマークを提案しており、①ライブ評価、②最小情報パラダイム（保有状況・価格・ツールのみ提供）、③複数市場（U.S.株式・A株・暗号資産）での汎化性評価が主な特徴。

今回の再現はLevel 1〜2で、ライブ市場データの代わりに3種類の合成市場を使用。U.S.風（上昇トレンド＋テック売りショック）、A株風（政策・センチメント変動）、暗号資産風（高ボラティリティ＋ドローダウン局面）を設計。エージェントは4種類：均等保有のbaseline_hold、直近モメンタム重視のmomentum_agent、リスクニュース検知時にキャッシュへ逃げるrisk_aware_agent、全力集中投資のreckless_agent。エージェントの出力は{asset: weight}形式の目標ウェイトに統一し、評価器を差し替えなしに再利用できる設計。

評価指標はCR（累積リターン）、Sortino Ratio、Volatility、MDD（最大ドローダウン）の4指標。合成市場での結果では、暗号資産風市場でbaseline_holdが−17.54%まで落ち込む一方、momentum_agentは+0.23%に抑制。A株風ではrisk_aware_agentが5.50% CRを記録し、リスク回避の効果が顕著。CRだけでは高いreckless_agentもMDDが大きく、複数指標での評価の重要性が確認できる。

監査エージェント開発への示唆として、「最小情報パラダイム」と「目標ウェイトによる意思決定インターフェースの正規化」は、監査エージェントにおける証拠収集→判断→アクション出力のパイプライン設計に直接応用可能。特に、エージェントの出力を正規化された形式に統一することで評価器を共通化するアプローチは、複数LLMを比較評価する監査AI基盤の設計に参考になる。次の拡張としてLLMにDecisionをJSONで返させるアダプタ追加が示されており、実際のLLM統合への橋渡しも明確。

## アイデア

- エージェント出力を{asset: weight}の目標ウェイトに正規化することで、LLM差し替えと評価器の分離を実現する設計パターンは、監査エージェントの判断出力標準化にも応用できる
- CR・SR・Vol・MDDの4指標セットで戦略を多面評価する手法は、単一指標では見えないリスク特性（高CRでも高MDD）を可視化し、エージェント品質評価の枠組みとして汎用性が高い
- 「再現レベルの明示（Level 1〜2）」と「動いた≠再現できた」の区別を設けるアプローチは、LLMベンチマーク研究の再現性問題に対する実践的な誠実さのモデルケース
