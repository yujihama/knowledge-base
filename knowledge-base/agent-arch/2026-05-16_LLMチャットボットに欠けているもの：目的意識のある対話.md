## 要約

現在のLLMチャットボットはMMLU・HumanEval・MATHといったベンチマークで急速に性能向上しているが、これらは非インタラクティブな一回限りの評価であり、実際のユーザー体験の向上と比例していない。本稿は「目的意識のある対話（Purposeful Dialogue）」の欠如を問題提起する。目的意識のある対話とは、旅行計画・コード生成・心理療法といった特定ゴールを中心に展開する多ターン会話を指す。交渉理論の反復的交渉と同様に、一度に全情報を渡すより往復のやり取りの方が重要情報だけを選択的に交換でき、より良い結果が得られる。Terry Winogradの言葉を借りると、発話は相手の世界モデルを変える意図的な行為であり、対話はゴール達成のための協調ゲームとして定式化できる。

対話システムの構築は1970年代のSchankの「レストランスクリプト」から始まり、ELIZAやPARRYを経て、現代のLLMベースシステムに至る。現代のシステムは(1)大規模コーパスでの次トークン予測事前学習、(2)chat_templateによる対話フォーマット変換、(3)RLHFによる微調整、という3段階で構築される。しかしRLHFは事前学習に比べて極めて小規模なファインチューニングに過ぎず（LeCunのケーキのチェリー比喩）、多ターン対話での一貫性維持は依然として困難。

著者らの実験では、2つのシステムプロンプトを持つLMエージェント同士を長時間対話させるDailogue Injectionという評価フレームワークを構築。8ターンの対話途中にインジェクション発話を1回挿入し、その後モデルが元の役割に戻れるかを測定。結果、GPT-4oやClaude-3.5-Sonnetを含む主要モデルでも長期対話での役割一貫性は著しく低下することが判明した。

解決策として論文が提案するのは、(1)系統的なデータ収集：Chatbot Arena等のプラットフォームから多ターン対話データを収集、(2)RL的アプローチ：各発話を行動、対話全体を軌跡として定式化し報酬でモデルを訓練、(3)ゲーム理論的定式化：例えばDiplomacyゲームを使った自然言語交渉訓練——の3方向。監査エージェント開発への示唆として、複数ターンにわたる要件確認・証跡収集・ギャップ指摘というエージェント的対話フローの設計に際し、長期ロール一貫性の担保が重要な未解決課題であることが確認できる。SWE-benchが示すように、複雑タスクは単発アクションでは解決できず、人間との往復対話が不可欠であり、エージェント型監査システムにもこの知見は直結する。

## アイデア

- 対話をゲーム理論的に定式化する発想：各発話を行動空間の選択、対話軌跡全体を最適化対象とするRL的枠組みは、監査エージェントの多ターン証跡収集フローの設計指針として直接応用できる
- Dialogue Injectionという評価手法：対話途中に外部発話を挿入して役割一貫性を測定するフレームワークは、エージェントのロバスト性テストとして既存のMT-BenchやAlpaca-Evalでは測定できない次元を評価している
- DiplomacyゲームやNegotiation理論を訓練データとして活用する方向性：自然言語での交渉・説得・情報開示戦略を持つゲームを使ったエージェント訓練は、監査インタビューシミュレーションへの応用可能性がある
