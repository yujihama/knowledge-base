## 要約

現在のLLMチャットボットはMMLC・HumanEval・MATHなどのベンチマークでスコアが向上し続けているが、これらは非インタラクティブな一回限りの評価であり、実際の人間-AI協調に必要な「目的意識のある対話（purposeful dialogue）」を測れていない。本記事では、複数ターンの対話を通じて目標達成を目指す「purposeful dialogue」の概念を整理し、現状のLLMダイアログシステムの限界と今後の方向性を論じている。

目的意識のある対話とは、旅行計画・コード生成・カスタマーサポートなど具体的なゴールを中心に据えた複数ターンの会話を指す。交渉理論の「反復的バーゲニング」と同様に、情報を一括ではなく段階的に交換することで効率的なゴール達成が可能になる。SWE-benchのようなGitHub Issue解決タスクも、単一パスでは解決困難であり、AIが要件確認・不足情報収集・人間への作業依頼を繰り返す対話が必要とされる。

現在のLLMは、事前学習（次トークン予測）→チャットフォーマット導入（apply_chat_template）→RLHF（KLペナルティ・LoRA等）という3段階で構築される。RLHFはLeCunの「ケーキのチェリー」比喩通り、事前学習データと比べごく小規模なファインチューニングに過ぎない。この構造上、システムプロンプトは対話の一貫性維持の主要手段だが、数ターン経過後にロールや指示から逸脱する「コンシステンシー崩壊」が実証的に確認されている。

著者らの研究では、2つのシステムプロンプト付きLMエージェントを長期間互いに会話させ、特定ターンで片方のエージェントへ話題逸脱を誘発する「distracter」を挿入してモデルの一貫性をストレステストした。GPT-4・Llama-3等の主要モデルで実験した結果、ターン数が増えるほど元の指示への追従率が低下することが確認された。

解決策として、(1)構造化された状態管理（dialogue state tracking）、(2)ゴール指向の報酬設計（goal-oriented RL）、(3)長期記憶の更新機構、(4)マルチエージェント協調（debate・role-playing）が提示されている。監査エージェント開発への示唆としては、LangGraphのようなグラフ型フレームワークでステート管理を明示的に行い、ReActループでの対話一貫性を担保する設計が有効である。複数ターンにわたる監査手続きの追跡・要件確認フローでも、distracterへの耐性設計が重要となる。

## アイデア

- 2つのLMエージェントを対話させ途中にdistracterを挿入してコンシステンシーをストレステストする手法は、監査エージェントの長期セッション耐性評価にそのまま流用できる
- 交渉理論の反復バーゲニングをHuman-AI情報交換に適用する視点は、要件定義フェーズを対話的に設計するAIシステムの理論的根拠として有用
- RLHFが事前学習データ量に対して極めて小規模なファインチューニングであることを踏まえると、長期対話一貫性はシステムプロンプト設計だけでなくアーキテクチャレベルの状態管理で補完する必要がある
