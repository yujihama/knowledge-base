## 要約

MCPThreatHive は、MCP（Model Context Protocol）ベースのエージェントシステムに特化した脅威インテリジェンスを自動化するオープンソースプラットフォームである。MCP の急速な普及に伴い、既存のセキュリティフレームワークが対応できていない新たな脅威カテゴリが出現しており、本研究はその対処を目的としている。

プラットフォームの主要機能は、脅威インテリジェンスのエンドツーエンドライフサイクルの自動化にある。具体的には、①複数ソースからの継続的データ収集、②AI駆動による脅威の抽出・分類、③構造化知識グラフへの格納、④インタラクティブな可視化、の4段階で構成される。

本研究の中核となるのが「MCP-38脅威タクソノミー」であり、MCP 固有の38種類の脅威パターンを定義・整理したものである。これらのパターンは STRIDE（なりすまし・改ざん・否認・情報漏洩・サービス妨害・権限昇格）、OWASP Top 10 for LLM Applications、OWASP Top 10 for Agentic Applications の3つのフレームワークにマッピングされており、脅威の多角的な分類を可能にする。また、複合リスクスコアリングモデルにより、脅威の定量的な優先度付けが実施できる。

既存の代表的な MCP セキュリティツールとの比較分析から、MCPThreatHive が対処する3つの重大なカバレッジギャップが明らかになった。第一に「コンポジショナル攻撃モデリングの不完全性」——複数のMCPツールや権限が組み合わされた際に生じる複合攻撃を既存ツールが十分にモデル化できていない点。第二に「継続的脅威インテリジェンスの欠如」——脅威情報がリアルタイムで更新されないため新興攻撃への対応が遅れる点。第三に「統合マルチフレームワーク分類の不在」——単一フレームワークのみに依拠した分類では見落としが生じる点である。

監査エージェント開発への示唆として、MCPを利用したエージェントシステムを本番環境に展開する際、LangGraph や ReAct ベースのエージェントが MCP 経由でツールを呼び出す構造は MCP-38 が定義するプロンプトインジェクション・ツール毒性（Tool Poisoning）・認証情報漏洩などのリスクにさらされる。監査エージェントが扱うデータの機密性を考慮すると、MCPThreatHive のような脅威タクソノミーを設計段階から参照し、STRIDE マッピングに基づくリスク評価を組み込むことが重要である。

## アイデア

- MCP固有の38脅威パターン（MCP-38タクソノミー）をSTRIDE・OWASP LLM・OWASP Agenticの3フレームワークに同時マッピングすることで、単一フレームワークでは見えないリスクを可視化できる設計
- 脅威情報の収集から分類・知識グラフ格納・可視化までを一貫して自動化するパイプライン構造——LLMによる脅威抽出をグラフDBと組み合わせた点は監査証跡管理システムにも応用可能
- コンポジショナル攻撃（複数MCPツールの組み合わせによる権限昇格など）を独立したカバレッジギャップとして明示した点——マルチエージェント環境では単体ツールの安全性だけでなく相互作用の安全性評価が必要という設計上の視点
