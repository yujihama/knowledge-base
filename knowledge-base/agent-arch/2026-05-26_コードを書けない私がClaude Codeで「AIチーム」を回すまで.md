## 要約

50代のシステムエンジニアである著者が、プログラミングスキルなしでClaude Codeを活用し、複数のAIエージェントを「チーム」として組織的に運用するワークフローを構築するまでの実践記録。全5章・約75,000字の有料書籍（900円）。

第1部「チームが動く組織設計」では、AIエージェントに対して役割・責任・権限を明示的に定義する「組織設計」の考え方を導入。コードを書く代わりに、プロンプトによる指示書・役割定義・エスカレーション規則を整備することで、Claude Codeが自律的にタスクを処理できる環境を構築した。

第2部「走り出すと何が起きるか」では、実際に複数エージェントを並走させた際に発生する問題—指示の解釈ずれ、コンテキスト汚染、タスクの重複実行—を記録。エージェント間のハンドオフ設計や、明示的なチェックポイント設置の重要性が示される。

第3部「事故と気づきから育つ仕組み」では、エージェントが予期しない挙動をした「事故」事例を分析し、ガードレール・レビュープロセス・ログ記録による再発防止策を整備するプロセスを解説。失敗から組織ルールを帰納的に育てる手法として、CLAUDE.mdへのルール追記やSubagent構成の見直しが具体的に語られる。

著者の特徴は、技術実装ではなく「マネジメント」視点でAIチームを設計している点。人間チームの組織論（役割分担、ナレッジ共有、インシデント管理）をAIエージェント運用に転用し、コードなしでも高度なマルチエージェントワークフローを実現した。監査エージェント開発への示唆として、エージェントへの権限・責任の明示的付与と、インシデントログによる継続的改善サイクルの設計は直接応用可能。特に内部統制的な観点から「エージェントの暴走を止めるガードレール設計」の章は参考になる。

## アイデア

- コードを書く代わりに「組織設計書」としてプロンプト・役割定義・エスカレーションルールを整備することで、非エンジニアでもマルチエージェントチームを運用できる
- 人間チームのインシデント管理手法（事後分析・ガードレール追加・ルール文書化）をAIエージェント運用に転用し、CLAUDE.mdを継続的に育てる帰納的改善サイクル
- エージェント間のハンドオフ設計とチェックポイント設置により、コンテキスト汚染やタスク重複を防ぐマルチエージェント協調の実践パターン
