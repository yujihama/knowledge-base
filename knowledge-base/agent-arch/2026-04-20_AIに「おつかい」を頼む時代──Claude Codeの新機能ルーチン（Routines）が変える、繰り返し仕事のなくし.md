## 要約

2026年4月14日、AnthropicはClaude Codeに「ルーチン（Routines）」機能をリサーチプレビューとして追加した。ルーチンとは、AIへの指示（プロンプト）を一度定義しておくと、スケジュール（時刻）またはGitHubイベント（PRの作成など）をトリガーとして自動実行される仕組みである。従来のGitHub ActionsやCronジョブと異なる最大の特徴は、「AIが内容を読み、文脈を理解した上で判断・実行する」点にある。スクリプトによる固定処理ではなく、例えばPRの差分を意味的に解釈してレビューコメントを生成したり、Linearから優先度の高いバグを取得して修正のドラフトPRを作成したりといった、状況依存の知的作業を自律的にこなせる。また、Anthropicのクラウド上で動作するためユーザーのPCを起動しておく必要がなく、深夜・休日でも稼働し続ける。ユースケースとしては、毎朝9時にSlackの#dailyチャンネルへ前日のマージ内容を投稿、PRトリガーで自動コードレビュー、毎週金曜に週次進捗レポートをSlackへ送信などが挙げられる。設定はWebUI（claude.ai/code/routines）、CLIの/scheduleコマンド、デスクトップアプリの「ルーチン」メニューの3経路から行える。プロンプト設計では「何をするか」「対象範囲（リポジトリ、過去24時間など）」「出力先（Slackチャンネル名、PRコメントなど）」の3要素を自己完結する形で明記することが推奨される。利用制限はプランごとに異なり、Proで1日5回、Maxで15回、Team/Enterpriseで25回。上限超過分は従量課金で継続利用可能だが割高。監査エージェント開発への示唆としては、監査チェックリストの自動実行（毎週月曜に特定GitHubリポジトリへの変更を走査してコンプライアンス懸念点をIssueとして起票するなど）に直接応用できるアーキテクチャパターンである。LangGraphの定期実行グラフをReActパターンで設計する際、ルーチンのトリガー＋プロンプト自己完結という設計思想はエージェントのスケジューリング設計の参考になる。

## アイデア

- GitHub ActionsやCronジョブとの本質的差異は「固定スクリプト実行」vs「AIによる文脈理解と判断」にあり、同じスケジューリング基盤でも出力の柔軟性が根本的に異なる
- プロンプトを『いつ・何を・どこに』の3要素で自己完結させる設計思想は、マルチエージェントシステムのタスク定義フォーマットとして汎用的に転用できる
- ユーザーのPCに依存しないクラウド実行という設計は、監査エージェントのような24時間稼働が求められる業務自動化において、インフラコストと信頼性のバランスを取る参考モデルになる
