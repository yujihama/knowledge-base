## 要約

本稿は2024年9月にThe Gradientに掲載された論考で、現行のLLMチャットボットが「目的意識のある多ターン対話（purposeful dialogue）」を欠いているという問題を体系的に分析する。

著者はまず、MMlu・HumanEval・MATHなどのベンチマークがワンパス生成を前提としており、インタラクティブな対話品質を測定できていない点を指摘する。旅行計画や対話型コード生成（SWE-bench）を例に挙げ、現実の複雑なタスクは複数ターンの情報交換なしには解決できないと論じる。交渉理論の「反復的バーゲニングは一括提示より良い結果をもたらす」という知見を援用し、マルチターン対話が人間-AI協調の本質であるとする。

対話システムの歴史的変遷として、1970年代のSchankの「レストランスクリプト」やELIZA・PARRYといった明示的スクリプト型から、現代のLLMベース（次トークン予測）への移行を概説する。現代の構築フローは(1)大規模コーパスでの事前学習（Llamaの技術報告にあるようにニュース・書籍・GitHubコード主体、Reddit等の対話的データは少量）、(2)chat templateによる対話フォーマットの導入（<system>/<INST>タグ等）、(3)RLHFによるファインチューニング（KLペナルティ・LoRA）の三段階で構成されるが、RLHFはLeCunの「ケーキのさくらんぼ」比喩通り学習データ規模が小さい。

一貫性（consistency）の定量評価については、著者らが独自に構築した環境を用い、二つのLMエージェントを最大8ターン対話させる手法を採用。対話途中の任意ターンに「侵入文（intrusion utterance）」を挿入し、モデルが本来の役割を維持するか測定するコンフォーミティ評価（conformity score）を定義した。実験結果として、現行のLLMはターン数が増えるにつれシステムプロンプトへの追従率が低下し、敵対的な文脈では役割を放棄するケースが確認された。

今後の課題として、(1)目標・意図・信念を明示的に推論するシステム設計（BDI: Belief-Desire-Intentionモデルの活用）、(2)長期記憶によるユーザプロファイルの動的更新、(3)対話履歴をゲーム理論的フレームワーク（協調ゲーム）として定式化することが提案される。監査エージェント開発の観点では、複数ターンにわたって調査意図を保持し、不足情報を能動的に質問しながら証拠を収集するエージェント設計に直結する知見であり、RLHFだけでなくBDI型の目標管理モジュールを組み合わせるアーキテクチャが有効と考えられる。

## アイデア

- 二つのLMエージェントを相互対話させ、途中に侵入文を挿入することで役割一貫性を定量評価するフレームワークは、監査エージェントのロール維持テストにそのまま応用できる
- 交渉理論の反復的バーゲニング（iterative bargaining）を人間-AIの情報交換モデルとして捉える視点は、要件定義フェーズでエージェントが人間から仕様を段階的に引き出すインタビュープロトコル設計に転用可能
- BDI（Belief-Desire-Intention）モデルを明示的に組み込むことで、長期対話でのシステムプロンプト忘却問題を構造的に解決できる可能性があり、LangGraphの状態管理と組み合わせた実装が考えられる
