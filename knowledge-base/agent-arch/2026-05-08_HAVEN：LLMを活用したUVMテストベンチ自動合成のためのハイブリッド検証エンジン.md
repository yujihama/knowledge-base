## 要約

IC（集積回路）検証はIC開発サイクルの約70%を占める高コスト工程であり、LLMを用いたテストベンチ自動生成への期待が高まっている。しかし、HDL（Hardware Description Language）はLLMの学習データに極めて少なく、LLMが直接HDLコードを生成すると誤りが多発する問題があった。本論文では、この課題を克服するHAVEN（Hybrid Automated Verification ENgine）を提案する。

HAVENの核心的な設計思想は「LLMにHDLを直接書かせない」ことである。UVMテストベンチ生成においては、LLMエージェントが設計仕様書を解析して構造化されたアーキテクチャ計画を生成し、HAVENのTemplate Engineがそれをプロトコル固有のJinja2テンプレートと組み合わせることで、バスハンドシェイクタイミングが正確な全UVMコンポーネントを生成する。LLMの出力は自然言語レベルの設計意図にとどめ、実際のHDL生成はルールベースのテンプレートエンジンが担う。

UVMシーケンス生成では、Protocol-Aware Sequence DSL（ドメイン固有言語）を新たに導入し、シーケンスを細粒度のステップ型に分解する。まず事前定義済みのDSLパターン集合によってLLMを介さずに高カバレッジを達成し、その後LLMエージェントがカバレッジギャップレポートを反復的に解析して追加のターゲットDSLシーケンスを生成することでカバレッジを漸進的に向上させる。

評価実験はDirect、Wishbone、AXI4-Liteの3種のインターフェースプロトコルにまたがる19のオープンソースIP設計で実施。結果はコンパイル成功率100%、コードカバレッジ90.6%、機能カバレッジ87.9%（平均）を達成し、LLM支援テストベンチ生成システムの中で最高水準（SOTA）を記録した。プロトコル固有のJinja2テンプレートで全UVMコンポーネントを生成し、Protocol-Aware DSLとルールベースコードジェネレータを組み合わせた初のシステムとして位置づけられる。

監査エージェント開発への示唆として、「LLMに誤りやすい形式のコードを直接生成させず、DSL＋テンプレートエンジンの組み合わせで制御する」というアーキテクチャパターンは、監査ルールや規制文書からの構造化アウトプット生成にも応用可能である。LLMをオーケストレーター兼アナリストとして使い、実際の成果物生成はルールベースエンジンに委ねるハイブリッド設計は、高精度が求められる監査AIシステムの信頼性向上に直接活用できる考え方である。

## アイデア

- LLMに誤りやすいドメイン固有言語（HDL）を直接生成させず、LLMの役割を「設計意図の解析・計画立案」に限定し、実コード生成はルールベースのテンプレートエンジンに委ねるというハイブリッドアーキテクチャは、LLMの幻覚問題を構造的に回避する汎用的なパターンである
- カバレッジギャップレポートをLLMエージェントに反復的に解析させてターゲットシーケンスを追加生成する「フィードバックループ型カバレッジ改善」は、テスト自動化だけでなく監査手続きの網羅性向上にも応用できる発想である
- Protocol-Aware DSLという中間表現層を導入することで、LLMの出力を検証可能・再利用可能な形式に制約しつつ、プロトコル知識をDSLパターンとして形式化して蓄積できる点は、知識ベース駆動型エージェント設計の一形態として注目に値する
