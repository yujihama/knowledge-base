## 要約

ハーネスエンジニアリングとは、LLMエージェントを「正しい方向に、安全に、再現性をもって動かす仕組み」の設計技術であり、2026年現在、エージェント開発の競争優位の源泉として注目されている。

OpenAI Codexチームは3〜7人で5ヶ月間に100万行以上のコードを生成・1,500件超のPRをマージした実績を持つ。その鍵は「自由度を絞ること」——依存関係をTypes → Config → Repo → Service → Runtime → UIという厳密な階層で制約し、エージェントのアーキテクチャ選択肢を事前に限定した。

AnthropicはClaude Codeにデフォルト読み取り専用の権限モデルを採用し、書き込みには明示的許可を要求する。さらにGANにインスパイアされた3エージェント循環——Planner（タスク分解・計画）、Generator（コード生成）、Evaluator（検証・フィードバック）——を組み合わせることで単一エージェントを超える品質を実現する。

Google CloudはAgent Development Kit（ADK）をベースに8つのマルチエージェント・デザインパターンを体系化。Sequential Pipeline（線形・決定論的で可デバッグ）、Review and Critique（品質閾値に達するまで生成→批評→改善をループ）、Swarm（全エージェントが相互通信）の3パターンが実践的に直結する。エージェントをマイクロサービスのAI版として捉え、A2A（Agent2Agent）プロトコルで異なるフレームワーク間の連携も可能にする。

LangChainは「Agent = Model + Harness」と定式化し、状態管理・ツール実行・フィードバックループ・実行制約をすべてプラガブルなMiddlewareとして実装するDeep Agents SDKを提供。LangSmithによるトレーシングで各ステップの入出力を可視化し可観測性を担保する。

ThoughtWorksのBirgitta Böckeler氏はハーネスを3カテゴリに整理する：①Context Engineering（必要な情報を必要なタイミングで適切な粒度で渡す技術）、②Architectural Constraints（触れてよい範囲を構造的に定義）、③Garbage Collection（長時間稼働で蓄積する不整合の定期的解消。AnthropicはContext Resetで「汚れた部屋を片付ける」のではなく「きれいな部屋に移る」方式を採用）。

実践指針は5点：最小権限からの段階的拡張、確定的ルールとLLM判断の明確な分離、Review and Critiqueパターンの活用、LangSmithなどによる可観測性の組み込み、段階的自律性（Graduated Autonomy）に基づく制約の調整。

## アイデア

- GANの敵対的学習構造をエージェントアーキテクチャに転用したPlanner/Generator/Evaluatorの3エージェント循環は、LLM-as-judgeの継続的フィードバックループとして機能し、単一エージェントでは収束しない品質問題を反復的に解決できる
- Anthropicの「Context Reset」——コンテキストを圧縮・要約するのではなく、構造化された引き継ぎ情報とともに新鮮なコンテキストウィンドウを与える手法——は、長時間稼働エージェントの状態劣化問題に対するアーキテクチャレベルの解として実装コストが低い
- OpenAI Codexの依存層階層制約（Types→Config→Repo→Service→Runtime→UI）は、エージェントの行動空間を事前に制限することで品質の再現性を高める設計パターンであり、コード生成以外のエージェントタスク（文書作成・データ変換）にも一般化できる
