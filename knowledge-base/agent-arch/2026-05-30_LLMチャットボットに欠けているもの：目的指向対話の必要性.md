## 要約

本記事は2024年9月にThe Gradientで公開された論考で、LLMベースのチャットボットが「目的指向対話（Purposeful Dialogue）」を欠いているという問題を指摘する。MMLU・HumanEval・MATHなどのベンチマークスコアが向上しても、実際のユーザー体験が比例して改善されていない現状を問題提起する。

目的指向対話とは、特定のゴール（旅行計画、心理療法、カスタマーサービスなど）を中心に据えた複数ターンの会話を指す。交渉理論における「反復交渉は単一提案より良い結果をもたらす」という知見を援用し、情報の断片的な伝達ではなく双方向のやり取りによって重要情報を選択的に交換できる点を強調する。Terry Winogradの言葉「すべての言語使用は相手の世界モデルを変更する手続きの起動」を引用し、対話を協調ゲームとして定式化する。

対話システムの構築プロセスとして、(1)次トークン予測による事前学習、(2)chat_templateによる対話フォーマット導入、(3)RLHFによる微調整という3段階を解説する。特にRLHFはLeCunの「ケーキのチェリー」比喩通り、事前学習コーパスに比べてデータ量が桁違いに少ない微調整段階にすぎない点を指摘する。

既存システムの一貫性評価については、著者らの研究で「2つのシステムプロンプト付きLMエージェントを相互に対話させる」環境を構築し、対話の途中時点でロールプレイプローブを注入してチャットボットの役割維持を検証する手法を提案。8ターンの対話中に途中から割り込ませた場合の応答を測定し、対話が長くなるにつれてシステムプロンプトへの遵守率が低下することを定量的に示した。ChatGPT・Claude等で複数ターン後に役割逸脱が起きるという日常的体験を裏付ける。

改善策として、(1)より長い対話データでの事前学習、(2)長期対話を考慮したRLHF・RLHF-V、(3)ユーザープロファイルの動的更新、(4)外部メモリの活用を挙げる。SWE-benchに代表されるGitHub Issue自動解決のような実務課題においても、単一パス生成ではなく人間との往復コミュニケーションが不可欠であり、ペアプログラミング的な協調モデルがコード品質向上に寄与すると論じる。監査エージェント開発への示唆として、監査手続きの実行においても証拠収集・質問・確認という複数ターンの往復が本質的であり、単一プロンプト型エージェントの限界と多ターン型エージェント設計の重要性を本論文は裏付ける。

## アイデア

- 2つのLMエージェントを相互対話させ途中にプローブを注入することで、長期対話での役割一貫性を定量評価する手法は、監査エージェントの堅牢性テストにそのまま応用できる
- 対話を「双方が世界モデルを更新し合う協調ゲーム」として定式化することで、エージェントの行動設計をゲーム理論・交渉理論の枠組みで最適化できる可能性がある
- ユーザープロファイルの動的更新と外部メモリの組み合わせは、監査エージェントが過去の監査知見・クライアント固有情報を蓄積し継続的に精度向上する仕組みの設計指針になる
