## 要約

本記事は、現行のLLMベースチャットボットが「目的志向型対話（Purposeful Dialogue）」を欠いているという問題提起を行い、その原因と改善方向を論じる。

現在のLLMはMMLU・HumanEval・MATHなどのベンチマーク性能が向上し続けているが、これらは単発応答を評価するものであり、複数ターンにわたるインタラクティブなユーザー体験の質とは乖離している。目的志向型対話とは、旅行計画・コード生成・カスタマーサポートのように、特定のゴール達成を中心に展開するマルチターン会話を指す。交渉理論における「反復的バーゲニング」が単発のtake-it-or-leave-itより優れた結果をもたらすのと同様に、複数ターンの情報交換により重要情報が選択的にやり取りされる。

LLMの構築過程を見ると、事前学習は大規模インターネットテキストでの次トークン予測であり、対話形式データは小比率。その後`tokenizer.apply_chat_template`等でシステムプロンプト/過去交換履歴を構造化し、RLHFで対話形式への適応を図る。RLHFはYann LeCunの「ケーキのチェリー」比喩通り、事前学習コーパスに比べ極めて小規模な微調整であり、LoRAやKLペナルティで既存の重みを大きく変えない。

問題の核心は対話の一貫性（Consistency）だ。著者らの研究では、2つのLMエージェントを互いに会話させる環境を構築し、[a1,b1,...,a8,b8]のような長期対話を合成してシステムプロンプト遵守を評価した。任意のターン後にプローブを挿入し、「今どんな役割を担っているか」を問う手法で、GPT-4・Claude・LLaMA-2-70B等を8ターンでテストしたところ、軒並み性能が劣化した。人間もトピックドリフトするが、LLMはより顕著に目的を忘れる。

対策として著者らはsummary-augmented dialogueを提案：各ターン後にLLMが対話要約を生成し、それをコンテキストに常時注入することで、長期対話での役割遵守精度が大幅改善した。ただしこれは「記憶」の代替であり根本解決ではない。

より根本的なアプローチとして、対話を「信念状態（Belief State）の逐次更新」として定式化するBelief State Transformer等の研究も紹介されている。ユーザーの隠れた目的・嗜好を確率的に追跡し、情報要求行動（Clarification Request）やゴール達成行動を最適化する枠組みであり、タスク指向型対話の古典的アーキテクチャ（DST: Dialog State Tracking）をLLM時代に再解釈したものだ。

監査エージェント開発への示唆として、監査シナリオはまさに目的志向型対話の典型例である。エビデンス収集・リスク評価・追加質問という反復的プロセスにおいて、LangGraphベースのエージェントが複数ターンにわたって監査目的を一貫して保持する仕組み（サマリー注入、Belief State管理）は直接応用可能。MT-Bench的な単発評価ではなく、長期一貫性を測るベンチマーク設計も検討に値する。

## アイデア

- 2つのLMエージェントを互いに会話させて長期対話を合成しシステムプロンプト遵守をストレステストする手法は、監査エージェントの役割維持能力の自動評価フレームワークとして応用できる
- 各ターン後にLLMが対話要約を生成してコンテキストに注入する「summary-augmented dialogue」は、長いコンテキストウィンドウが使えない環境でも目的一貫性を維持する実用的な軽量解
- 対話を信念状態の逐次ベイズ更新として定式化するBelief State Transformerは、ユーザーの隠れた意図推定という問題をDSTの現代版として再解釈しており、RAGと組み合わせたエージェント設計に示唆がある
