## 要約

パーソナライズドエージェントが長期間ユーザーと対話するには、セッションをまたいだ永続的な記憶の維持と、状況変化に応じた記憶の更新が不可欠である。しかし既存のベンチマークは「過去会話からの事実検索」として長期記憶を評価する枠組みに偏っており、記憶の統合能力や頻繁な知識更新への対応能力を十分に測定できていない。本論文はこの課題に対し、数週間〜数ヶ月にわたるユーザー会話をカバーする長期記憶ベンチマーク「Memora」を提案する。Memoraは記憶に根ざした3種類のタスク——記憶（Remembering）、推論（Reasoning）、推薦（Recommending）——を評価軸として設定する。データ品質確保のため、自動化された記憶根拠チェックと人手評価を組み合わせている。さらに本研究の核心として、「忘却考慮型記憶精度（FAMA: Forgetting-Aware Memory Accuracy）」という新指標を導入する。FAMAは無効化・陳腐化した記憶に依存した応答にペナルティを与えることで、単純な事実想起精度では捉えられない「古い記憶を適切に棄却できるか」という能力を評価する。4種のLLMと6種の記憶エージェントを対象とした評価実験では、無効化された記憶の頻繁な再利用と、進化する記憶の整合性確保の失敗が広く観察された。記憶エージェントの導入は改善をもたらすものの、その効果は限定的であり、パーソナライズドエージェントにおける長期記憶の根本的な弱点が露わになっている。監査エージェント開発への示唆として、内部監査プロセスでも「以前の監査所見が現在も有効か」「規制や基準の変更で過去の判断が無効化されていないか」という類似の問題が存在する。FAMAのような「陳腐化した情報への依存をペナルティとして検出する」評価手法は、監査エージェントの記憶モジュール設計に直接応用できる観点である。

## アイデア

- 「忘却できること」を精度指標として定量化したFAMAの発想——正しく記憶するだけでなく、無効化された情報を捨てられるかを評価する観点は、エージェントの記憶設計を根本から見直す契機になる
- Memoraが数週間〜数ヶ月スパンの会話を対象とする点——短期セッション前提の既存ベンチマークでは見えなかった「記憶の経時劣化」問題をタスク設計に組み込んでいる
- 4 LLM × 6 記憶エージェントの比較で記憶エージェントの効果が限定的だったこと——現行の記憶エージェント実装がいずれも陳腐化した記憶の整合的更新に本質的に対応できていないことを示唆する
