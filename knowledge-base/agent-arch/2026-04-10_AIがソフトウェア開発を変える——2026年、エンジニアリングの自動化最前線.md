## 要約

2026年4月時点における、Claude Codeを中心としたAIエージェントによるソフトウェア開発工程の自動化実態をまとめた実践レポート。Anthropicは2026年に入り74件のリリースを数週間単位で続けており、その恩恵が特にエンジニアリング現場に集中している。

【アーキテクチャ設計】Claude Codeに自然言語で要件を伝えると、構成案とDraw.io互換の図面を5分以内に出力。Agent Skillsとの組み合わせで高品質な成果物が得られるとの実践報告が相次いでいる。

【IaC（インフラのコード化）】「このマイクロサービス用のAWS環境をTerraformで作って」という指示一つでTerraform設定ファイル一式を生成。「Claude CodeでTerraform生成→Gitでレビュー→適用」のフローが標準化しつつある。

【CI/CD】Claude Code Action（GitHub Actions連携）やHeadless Modeを使い、パイプライン設定YAMLをAIが自動生成・更新。コードプッシュ時点でテスト・セキュリティチェックまで完結する事例も報告されている。

【テスト設計・実装】GitHub上のdiffやIssue内容を読み取り、ユニットテスト・統合テストを網羅的に自動生成。Agent Skillsによるエッジケース提案も可能になった。

【ドキュメント整備】コードや会話履歴からREADME・API仕様書・ADR・運用プレイブックを自動生成し、NotionやConfluenceへの反映、MkDocs静的サイト生成もAgent Skillsで一気通貫。

【PRレビュー】2026年3月リリースのClaude Code ReviewをCI/CDに組み込むと、PR作成と同時にスタイル・セキュリティ・ベストプラクティス違反を自動指摘。レビュー時間が従来比1/3〜1/2に短縮。

【デプロイ】AWS Agent PluginsとComputer Use機能でClaudeがクラウドコンソール操作やTerraform applyを実行。「PRマージ→自動デプロイ→監視開始」まで完全自動化が実現可能。

現場では「開発速度3〜5倍」「エンジニアはコードを書く時間より判断する時間が増えた」という声が主流になっている。一方、要件定義・インシデント対応・プロダクト方向性の判断は依然として人間の領域として残る。エンジニアの役割は「コードを書くこと」から「正しいゴールを設定しAI出力に責任を持つこと」へシフトしていると結論付けている。

## アイデア

- Claude Code + Agent Skillsの組み合わせにより、アーキテクチャ設計からデプロイまでの全工程をエージェントが主体的に担う「縦断型自動化」が実現しつつある点
- CLAUDE.mdにプロジェクトルールを記述しておくことでドキュメント生成精度が劇的に向上するという知見——エージェントへのコンテキスト注入手法としてのファイルベース設定の有効性
- AIが得意なのは『決まったゴールへ速く正確に進む』こと、人間が必要なのは『正しいゴールを設定する』ことという役割分離の明確化——エージェントシステム設計の責任境界を考える上での実践的フレームワーク
