## 要約

現行のLLMチャットボット評価はMMLU・HumanEval・MATHなどのベンチマークで行われているが、これらは非インタラクティブな一発生成を測定するに過ぎない。本稿は「目的意識のある対話（Purposeful Dialogue）」という概念を提唱し、マルチターン対話における現行システムの限界を体系的に分析する。

目的意識のある対話とは、旅行計画・心理療法・カスタマーサービスといった具体的なゴールを軸に展開する複数ターンの会話を指す。交渉理論でいう「反復的バーゲニング」がモデルになっており、一回限りのやり取りより反復的な情報交換の方が優れた結果をもたらすという前提に立つ。SWE-benchのようなGithueイシュー解決タスクも、単一アクションでは解決できず、AI・人間間の往復通信が不可欠であることが示されている。

対話システムの構築フローとして、①事前学習（次トークン予測）、②対話フォーマット導入（<system>/<INST>タグによるチャットテンプレート適用）、③RLHF（報酬・ペナルティによるファインチューニング）の3段階を整理する。RLHFはLecunの「ケーキのチェリー」に例えられ、事前学習コーパスに比べてデータ規模は極めて小さい。

現行システムの一貫性問題については、著者らが構築した実験環境で定量評価を実施。2つのシステムプロンプト付きLMエージェントを互いに長期間対話させ、任意の中間ターンでプローブ質問（「あなたのキャラは？」など）を挿入して役割維持率を測定した。具体的には8ターン対話で中間的プローブを挿入し、ChatGPT・GPT-4・Claude 2・Llama-2-70Bなど主要モデルで評価した結果、対話が長くなるにつれてほぼすべてのモデルが役割から逸脱することが確認された。

評価手法の難しさも論じられている。モデル出力が確率的なため、事前に返答を用意してスクリプト型テストを行うことができない。そのため2エージェント間の合成対話を自動生成し、任意スケールで一貫性をストレス試験する手法が採用された。

将来の展望として、長期記憶を通じたユーザープロファイルの動的更新、メールの自動下書きと編集学習、Twitter・arxiv・Slackなどのリソースからの朝のニュース要約生成など、パーソナルアシスタント的機能の実現が示されている（IVA・Siriを例示）。監査エージェント開発への示唆として、監査手続きは複数ステップの情報収集・確認・判断を伴うため、目的意識のある対話フレームワークはReActループや人間-AIの往復確認プロセスの設計に直接応用可能。特に「ゴールを持つ対話をゲームとして定式化する」視点は、LangGraphでの状態遷移設計と相性が良い。

## アイデア

- 2つのLMエージェントを互いに対話させ、任意ターンにプローブ質問を挿入することで役割一貫性をストレス試験する手法は、監査エージェントの品質保証テストにそのまま転用できる
- 対話を「両者がそれぞれのゴールを持つ協調ゲーム」として定式化する視点は、LangGraphの状態グラフ設計やマルチエージェント間のプロトコル設計に理論的根拠を与える
- RLHFのデータが対話フォーマットを初めて含む段階であるにもかかわらず、事前学習コーパスとの規模差が桁違いであるため、長期一貫性維持には構造的な限界があるという分析は、ファインチューニング戦略の再考を促す
