## 要約

本記事はThe Gradient掲載（2024年9月）の論考で、LLMチャットボットのMMLU・HumanEval等のベンチマーク性能が向上しても、実際のユーザー体験が比例して改善しない理由を「目的指向対話（purposeful dialogue）」の欠如から説明する。目的指向対話とは、「旅行計画エージェント」「カスタマーサポートボット」のような特定ゴールを中心に据えた多ターン会話を指す。交渉理論の反復交渉モデルになぞらえ、情報の一括転送より逐次的なやり取りの方が質の高い成果をもたらすと論じる。

LLM対話システムの構成を3段階で整理する：(1)次トークン予測による事前学習、(2)chat_template等によるダイアログフォーマットの導入、(3)RLHFによる対話品質の調整。RLHFは事前学習コーパスに比べてデータ量が極めて少なく、LeCunの「ケーキのチェリー」比喩で表現されるように、対話能力の基盤として限定的な役割しか果たしていない。

一貫性の評価手法として、著者らは2つのシステムプロンプト付きLMエージェントを相互に会話させ、任意の長さの合成対話を生成する環境を構築した。8ラウンド対話の任意時点tにおいて、「役割A」が実際に役割Aの発言をしているかをLM-as-judgeで採点し、ターン経過に伴う指示追従率の劣化を定量化。結果として、ほぼすべての主要モデルでターンが進むほど一貫性スコアが低下することを確認した。

対話戦略の観点では、Grice の協調原則（量・質・関係・様態の格率）を理論的支柱として取り上げ、特に「関係性（relevance）」格率の維持が長期対話で困難になる点を指摘する。また目標の明示度によって対話を「明示的目標」と「隠れた目標」に分類し、後者は情報引き出しや嗜好適応など実アプリで重要なシナリオを形成するとする。

今後の展望として、(1)長期記憶を通じたユーザープロファイルの逐次更新、(2)arXiv・Slack等の外部ソースから個人化ニュースを生成するパーソナルアシスタント、(3)SWE-benchのようなコード生成ベンチマークを超えた実GitHub Issue解決への適用を挙げる。監査エージェント開発への示唆としては、複数エージェントが証拠収集・質問・確認を繰り返す監査インタビューシナリオにおいて、本研究の長期一貫性評価フレームワークを直接転用できる。エージェントがロール（監査人・被監査人）を維持しながら多ターン対話を継続する能力は、LangGraphベースのReActエージェント設計で特に重要な評価軸となる。

## アイデア

- 2つのシステムプロンプト付きLMを相互対話させることで任意長の合成対話を自動生成し、LM-as-judgeでターンごとの一貫性を定量評価する手法は、監査エージェントのロール維持能力のベンチマーク設計に直接応用できる
- Grice の協調原則を対話評価の理論的枠組みとして使うことで、単なる正解率ではなく「会話の目的に対する貢献度」という軸で対話品質を測定できる可能性がある
- 目標を『明示的』と『隠れた目標』に二分する分類は、RAGやツール呼び出しを伴うエージェント設計において、ユーザーの真の意図をいつ・どのターンで確認すべきかのトリガー設計に活用できる
