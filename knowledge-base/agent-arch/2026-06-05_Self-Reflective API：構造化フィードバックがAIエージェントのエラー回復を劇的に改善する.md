## 要約

AIエージェントがAPIを呼び出してバリデーションエラーに遭遇した場合、単なるエラーメッセージ（何が悪かったか）ではなく、修正方法を示す構造化フィードバックが必要だという研究。本論文では「Self-Reflective API」という概念を提唱し、バリデーション失敗時に machine-readable な `recovery_feedback.suggestions[]` ペイロードを返すことで、エージェントが外部推論なしにリクエストを修正・リトライできるようにする仕組みを提案している。

実験は leak-audited なパイロット試験として設計され、セルあたりN=30、3種類のLLM（Anthropic Claude系2モデルとgpt-4o-mini）、10種類の敵対的タスクを使用。主な結果として、構造化サジェスチョンはAnthropicモデルに対してタスク完了率を+36.7〜40.0パーセントポイント向上させた（Fisherの正確検定 p≤0.0022）。さらに成功あたりのトークン効率も1.8〜2.2倍改善した。一方、gpt-4o-miniでは有意差なし（p=0.435）というモデル依存性も示された。

この差異は、Anthropicモデルが構造化された指示に対してより素直に従う傾向があるのに対し、gpt-4o-miniは平文の英語診断メッセージからでも十分に推論できる可能性を示唆している。また、billing API という第2ドメインへの再現実験でもパターンが確認されており、特定ドメインへの過学習ではないことが示された。

方法論的な貢献として、LLMベンチマークにおける2種類の未文書化「答えリーク」クラスを発見・修正し、その監査ツール `audit_prompt_leakage.py` をCI基盤として公開している。これはベンチマーク信頼性向上に寄与する実用的な成果物である。

監査エージェント開発への示唆：監査エージェントがAPIや外部システムとやり取りする際、エラー応答の設計自体がエージェントの自律回復能力を左右する。現行の多くのAPIは人間向けのエラーメッセージしか返さないが、Self-Reflective API設計を採用することで、LangGraphベースのエージェントのリトライループを大幅に効率化できる可能性がある。特にPydanticバリデーションエラーを構造化フィードバックに変換する機構はReActエージェントと相性が良い。

## アイデア

- APIのエラーレスポンス設計そのものをエージェント最適化の対象として捉える視点：従来は人間の開発者向けに設計されていたエラーメッセージを、LLMが機械的に解釈できる構造化ペイロードに変換するだけでタスク完了率が40pp近く向上するという発見は、エージェント時代のAPI設計パラダイムシフトを示唆している
- モデルによる構造化フィードバックへの感受性の差異：AnthropicモデルとOpenAIのgpt-4o-miniで効果が大きく異なる（有意差あり vs なし）という結果は、各LLMが持つ instruction-following 特性の違いを実用的な観点から明らかにしており、エージェント構築時のモデル選定基準になりうる
- LLMベンチマークにおける答えリークの体系的監査：2種類の未文書化リーククラスを発見してCI用ツールとして公開したこと自体が、AI研究の再現性・信頼性問題に対する実践的な貢献であり、今後の評価実験設計に応用できる
