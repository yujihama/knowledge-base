## 要約

本記事は、LLMの出力挙動を安定させるために構築されたプロンプト設計の構造と設計思想を解説したものである。著者が実運用しているプロンプトは、[PURPOSE]・[CORE]・[STRUCTURE]・[PROCESS]・[OUTPUT]・[CONSISTENCY]・[INTERACTION]・[EXPLORATION]・[RESOLUTION]の9セクションで構成されている。

設計の核心は、入力情報を「FACT（明示された事実）」「UNKNOWN（未確定・欠損情報）」「CONTEXT（依存・同値・共起の関係性）」の3要素に分解し、それらを統合した「DRIVE（統合的解釈）」として内部推論を行う4層構造にある。ただしこれらの内部構造は出力には露出させず、自然文のみを提示する設計となっている。

アーキテクチャは4層に整理される。①インターフェース層：会話としての自然さを最優先し、機械的な構造表示を排除する。②認知構造層：FACT/UNKNOWN/CONTEXTによる入力分解とDRIVEによる統合推論を内部で実施する。③検証層：未確認情報の事実化を禁止し、不足情報がある場合は推測より確認を優先する。④表現層：内部構造を隠蔽し、中立トーンの自然文のみを出力する。

タイムスタンプ設計も特徴的で、すべての出力の先頭にシステム提供の現在時刻から毎回再計算したタイムスタンプ（YYYY-MM-DD形式）を強制表示する。目的は単なるログ整理ではなく、LLMが本質的に欠如する時間概念を外部フレームで補完し、過去文脈の過剰継承・意図しない連続解釈・状態混線を防ぐことにある。

Before/Afterの比較では、設計前は「推測と事実の混同」「冗長な補足説明」「文脈過剰継承」が課題だったのに対し、設計後はFACT/UNKNOWNの分離、不確実性の明示、出力圧縮が実現されたとしている。

著者はこの設計を「小規模な対話制御アーキテクチャ」と位置づけており、完成形ではなく調整段階としながらも、思考構造の分離・出力安定化・長期対話制御に効果があると評価している。監査エージェント開発への示唆としては、エージェントの内部推論をFACT/UNKNOWN/CONTEXTに分離して管理するパターンはLangGraphのステート設計やReActの観察・思考・行動分離と構造的に親和性が高く、エージェントの誤推論抑制やトレーサビリティ向上に直接応用可能な考え方である。

## アイデア

- 入力を FACT/UNKNOWN/CONTEXT/DRIVE の4要素に分解し内部推論を構造化しながら出力には露出させない「内部隠蔽型アーキテクチャ」は、LangGraph の隠れステートや ReAct の scratchpad と同様の発想で、エージェント設計に直接転用できる
- タイムスタンプを「LLMの時間認識欠如を補う外部フレーム」として位置づける視点は、長期対話・マルチターンエージェントにおける文脈汚染対策として実装上有効な知見
- OUTPUT ルールが DRIVE を含む他のすべてのセクションをオーバーライドするという優先順位の明示（[CONSISTENCY]セクション）は、エージェントのガードレール設計における「出力層の最終権限」パターンとして参考になる
