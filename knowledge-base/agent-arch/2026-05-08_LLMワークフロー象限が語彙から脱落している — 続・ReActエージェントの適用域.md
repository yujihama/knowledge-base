## 要約

業務AIを4象限（①スクリプト象限、②Algorithmic Search象限、③LLMワークフロー象限、④Autonomous Agentic Loop象限）に分類するフレームワークの続編。横軸「決定論/意味判断」×縦軸「ワークフロー定義可能/探索的」の4象限のうち、③LLMワークフロー象限（意味判断×定義可能）を肯定形で指示する業界語彙が存在しないことを主題とする。

業界の言説は「決定論で書ける部分は従来技術で、それ以外は自律エージェントで」という二分法に収束しており、③と④の間に切り分けが生じない。その結果、④Autonomous Agentic LoopのアーキテクチャであるReActループが③LLMワークフロー象限の業務にも無差別に適用される「カテゴリエラー」が常態化している。

このカテゴリエラーが引き起こす4つの症候を具体的に指摘する。(a) RPAの例外処理ボトルネックの再発：LLMワークフロー象限の業務が自律エージェントに丸投げされ、結局人間の監視担当が増える。(b) サンドボックス強度の過剰要請：ReActループは実行内容が事前予測不能なため、請求書マッチング程度の業務にもFirecrackerやmicroVMレベルの隔離が要請される。(c) Human-in-the-Loopの構造的歪み：HITLが「AI改善のフィードバックループ」ではなく「エージェントが処理できない判断を人間が常時補完する構造」として機能する。(d) 人為的redirect不能性：LLMワークフロー象限を適切に設計すれば「関数fの精度問題→モデル選定担当へ」のように責任をredirectできるが、ReActループでは判断連鎖がblendされて事後に分離できず、Elish(2019)のいうmoral crumple zone（自律システムの責任が制御能力の限られた人間オペレータに集中する構造）が発動する。

さらに④Autonomous Agentic Loop象限固有のアカウンタビリティ問題を③の混乱と峻別する。③の人為的redirect不能性は設計変更で解消可能だが、④で発生するattribution gap（自律的判断のどの要素が結果に寄与したかを遡及的に分離できないこと）は自律性の採用そのものに内在する原理的問題であり、技術成熟で解消されない。法律案件での和解条件提示を例に、「なぜ不利な合意に至ったか」をログから切り出せない構造を示す。

現状の議論は③の混乱を繕うサンドボックス・HITL論に集中し、④固有のattribution gapの議論が相対的に薄い「倒錯」が生じている。解決の起点として、LLMワークフロー象限を肯定形で呼ぶ語彙を現場から立ち上げることを提言する。監査エージェント開発への示唆として、監査手続の各ステップを③LLMワークフロー象限の単機能LLM関数として明示的に分離して設計することで、失敗時の責任redirect（「どの判断工程に問題があったか」の特定）が可能になり、四眼原則・職務分掌との整合も取りやすくなる。

## アイデア

- 業務AIを4象限で分類し、③LLMワークフロー象限（意味判断×定義可能）を独立した設計単位として命名することで、ReActの過剰適用を防ぐ語彙論的アプローチ
- ③の人為的redirect不能性（設計変更で解消可能）と④のattribution gap（自律性に内在する原理的問題）を峻別することで、アカウンタビリティ議論の論点を整理できる
- Elish(2019)のmoral crumple zoneをReActエージェントの運用責任に適用し、責任がエージェントにも設計者にも帰属できず運用責任者に集中する構造を可視化する概念装置として有用
