## 要約

Claude Opus 4.8 が 2026年5月28日に公開された。前モデル Opus 4.7（4月16日公開）からわずか41日でのリリースで、料金は据え置きのまま複数の改善が加えられた。

ベンチマーク面では、Terminal-Bench 2.1 で 92.3%、OSWorld-Verified（PC操作エージェント）で 84.2%、Finance Agent v2（ツール併用の専門推論）で 76.4% を記録。これらはコード補完精度よりも「タスクを最後まで自律的に遂行する」能力を測る指標であり、長時間の自律エージェント運用への適性が高まっていることを示す。

品質面の最大の変化は「正直さ」の改善で、自分が書いたコードの欠陥を見逃す確率が前モデル比で約1/4（4倍少ない）に低下した。「適切な問いを立てる」「自分のミスに気づく」「作業の不確実性を申告する」といった挙動が改善し、アラインメント評価でもユーザーの自律性を尊重する prosocial 指標で新水準を達成。AI生成コードをレビュー前提で運用している現場では、この変化がレビューコストの削減に直結する。

Claude Code は同日 v2.1.154（7つの新機能含む大型アップデート）で Dynamic Workflows（research preview）を搭載。ユーザーがワークフローの作成を依頼すると、Claude 自身が段取りを組み、背後で数十〜数百のサブエージェントを並列調整して大規模タスクに対処する。1セッションあたり最大1,000サブエージェントの上限があり、実行状況は /workflows コマンドで確認可能。従来の「手動でサブエージェントを設計・配置する」アプローチから、「設計そのものをモデルに委ねる」段階への移行を意味する。

料金面では、通常モードが入力 $5 / 出力 $25（100万トークンあたり）で据え置き。Fast mode は 2.5倍高速化しつつ従来比3倍安価（入力 $10 / 出力 $50）となり、レイテンシ重視の用途でのコスト障壁が下がった。Claude Code 上では Opus 4.8 は high effort がデフォルトで、特に難しいタスク向けに xhigh も用意されている。effort 設定が高いほど深く頻繁に推論するため、品質とトークン消費のトレードオフ管理が必要になる。

監査エージェント開発への示唆として、欠陥見逃し率の低下と不確実性申告の改善は、監査判断を補助するエージェントの信頼性確保において重要な進展。Dynamic Workflows の並列サブエージェント調整は、多工程にわたる監査手続きの自動化アーキテクチャ設計の参考になる。ただし Dynamic Workflows は research preview 段階のため、本番ワークフローへの組み込み前に挙動検証が必要。

## アイデア

- Dynamic Workflows によりサブエージェントの設計・配置をモデル自身に委ねる段階に移行した点は、エージェントアーキテクチャの抽象レベルが一段上がったことを意味し、ユーザーはタスク記述に集中できる
- コード欠陥見逃し率を前モデル比1/4に削減した「正直さ」の改善は、AI生成コードのレビューコスト削減に直結し、CI/CDパイプラインへのAI統合の信頼性閾値を引き下げる可能性がある
- 41日という短いリリースサイクルは、ベンチマーク・料金・機能の前提を月次で更新する運用を利用者側に求めており、モデル選定ポリシーの継続的な見直しが必要になる
