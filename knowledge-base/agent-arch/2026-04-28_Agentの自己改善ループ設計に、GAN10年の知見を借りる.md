## 要約

LLMの自己改善ループ（Self-Refine / Reflexion / Constitutional AI等）は、同一モデルがworkerとevaluatorを兼任する構造を持ちやすい。この構造はGAN（Generative Adversarial Networks）がGeneratorとDiscriminatorで直面してきた失敗モードと構造的に類似しており、GANの10年の知見をAgent設計に転用できるという試論。

**失敗モード1: pass/fail二値化と勾配消失**
オリジナルGANではDiscriminatorが偽物を完全判別するとJS divergenceに基づく勾配が消え学習が停止する。WGAN（Wasserstein GAN）はEarth Mover's Distanceによる連続シグナルで解決した。LLMにおける対策は評価シグナルの解像度向上で、垂直方向（5段階→連続値→pairwise Elo）と水平方向（単一軸→多項目rubric＋checklist）の2軸がある。LLM judgeは構造化自然言語を返せるため水平展開が有利。JSON形式でcorrectness/clarity/completenessをスコアリングし根拠も付記する設計を推奨。

**失敗モード2: Mode Collapse**
Generatorが「確実にDを騙せる1パターン」に収束し多様性を失う現象。Salimansらのminibatch discriminationはbatch内サンプル間距離を評価軸に追加することで対処。LLMループでの対策は、ループ内で生成履歴を参照した「過去N件との差分強制」プロンプト、複数seedによるサンプル多様化、ループ外でembeddingベースの多様性モニタリングを組み合わせる設計。workerとevaluatorの分離強化（異なるmodel/temperatureの使用）も有効。

**失敗モード3: 評価者の硬直化（Discriminator overfitting）**
Discriminatorが特定のアーティファクトに過剰適応し、Generatorの本質的改善を促せなくなる問題。Historical Averagingやspectral normalizationで対処。LLMでは評価rubricのローテーション（定期的な項目差し替え）、複数judge（Judge Panel）による投票、Adversarial Examplesの定期注入が対応策となる。

**失敗モード4: 評価者の崩壊（Discriminator collapse）**
Discriminatorが機能不全に陥りGeneratorへのシグナルが消える問題。LLMでは同一モデルがself-evaluateする際に「自分の出力を高く評価する」バイアスが類似現象として現れる。外部oracle（human feedback）の定期介入、judge品質の自動モニタリング（meta-judge）、judge自体への定期fine-tuningが対策。

監査エージェント開発への示唆として、監査プロセス自体が「実行者と評価者の独立性」を会計監査の独立性原則として要求している点が重要。同一LLMがaudit handoverの生成と品質評価を兼任する設計は構造的リスクを内包する。rubricを監査基準（IIA Standards等）の項目に対応させ、多項目checklist化することで評価シグナルの解像度を上げ、かつ外部judgeまたはhuman-in-the-loopを組み込む設計が望ましい。

## アイデア

- GANの失敗モード（勾配消失・mode collapse・discriminator硬直化）とLLM自己改善ループの失敗パターンが構造的に対応しており、数学的解法ではなく「設計目的」レベルで知見を転用できるという視点は、異分野からの設計パターン借用の方法論として汎用性が高い
- 評価シグナルの解像度向上を「垂直（スカラー精緻化）」と「水平（多項目rubric展開）」の2軸で整理した点が実践的で、LLMはテキスト出力できるため水平展開に固有の優位性があるという指摘は、GAN→LLM転用時の非自明な差分を正確に捉えている
- 監査AIの文脈で「会計監査の独立性原則」をself-improvement loopの設計上の問題と接続した点が重要で、evaluatorとworkerの分離はエンジニアリング上の効率だけでなく信頼性・監査証跡の観点からも必須要件として位置づけられる
