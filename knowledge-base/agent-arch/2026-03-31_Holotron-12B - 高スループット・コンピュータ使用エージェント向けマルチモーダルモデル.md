## 要約

H CompanyがNVIDIA Nemotron-Nano-2 VLをベースにポストトレーニングしたマルチモーダルモデルHolotron-12Bをリリースした。本モデルの主な設計目標は、静的なビジョン理解や汎用的な指示追従ではなく、コンピュータ操作エージェント（Computer Use Agent）のポリシーモデルとして本番環境で高スループット・低メモリフットプリントで動作することにある。

アーキテクチャ上の最大の特徴は、Nemotronが採用するハイブリッドSSM（State-Space Model）＋Attentionアーキテクチャである。純粋なTransformerベースモデルと異なり、SSMはシーケンス長に依存しない定数サイズの状態（state）のみを保持するため、KVキャッシュが系列長×層数×バッチサイズに比例して膨張する従来Attentionの問題を回避できる。これにより長コンテキスト・複数画像を含むエージェント的ワークロードで特に効果的なVRAM利用効率を実現している。

推論性能の検証としてWebVoyagerベンチマーク（同時100ワーカー）をH100 GPU 1枚＋vLLM v0.14.1で実行した結果、Holotron-12Bは最大同時実行数100時点で8,900トークン/秒のスループットを達成し、先代Holo2-8Bの5,100トークン/秒（同条件でプラトー）に対して2倍超の改善を示した。バッチサイズが大きくなってもスループットが線形に近い形でスケールする点が確認されており、データ生成・アノテーション・オンライン強化学習など並列処理が必要なワークロードに有利とされている。

トレーニングは2段階構成で実施された。まずNemotron-Nano-12B-v2-VL-BF16をベースに、H Company独自のローカライゼーション・ナビゲーションデータ混合でSFT（Supervised Fine-Tuning）を行い、スクリーン理解・グラウンディング・UIレベルの操作に特化した能力を強化した。最終チェックポイントは約140億トークンで学習されている。

エージェントベンチマークでは、WebVoyager正解率がNemotronベースの35.1%から80.5%に向上し、Holo2-8Bの性能を上回った。ローカライゼーション・グラウンディングベンチマーク（OS-World-G、GroundUI、WebClick）でもベースモデルを大幅に上回る結果を示している。

モデルはNVIDIA Open Model LicenseのもとHugging Faceで公開されており、次世代としてNemotron 3 OmniベースのHolotronシリーズへの発展も予告されている（MoE＋強化されたハイブリッドSSM-Attentionを採用予定）。

## アイデア

- SSMのKVキャッシュ不要化によるメモリ効率：長い操作履歴や多数のスクリーンショットを含むエージェントの軌跡データを処理する際、Transformerの二次計算コストを回避できるSSMは本質的にエージェント用途に適合しており、単純にモデルを小型化するのとは異なる次元のスケーラビリティを提供する
- オンライン強化学習との親和性：高スループット推論はデータ生成・アノテーションに有利なだけでなく、GRPO/RLAIFのようなオンラインRL手法でロールアウトを大量生成するボトルネック解消にも直結する。推論効率がRL学習速度を規定する場面でSSMアーキテクチャ選択が戦略的意義を持つ
- GUI操作の自動化に特化したSFTデータ設計：スクリーン理解・UI要素のグラウンディング・ナビゲーションに特化したデータ混合でファインチューニングすることで、汎用VLMでは困難な座標指定・クリック・入力操作の精度を大幅向上させており、エージェント専用データセット構築の重要性を示している

## Yujiの取り組みへの示唆

監査エージェントがポータルへのログイン・画面操作・証跡スクリーンショット取得を自動化するComputer Useユースケースにおいて、Holotron-12BのSSMベース高スループット推論は複数の監査ワークフローを並列実行する際のコスト効率に直接寄与する。また、GRPOやRLAIFでエージェントをオンライン学習させる際、ロールアウト生成速度が学習反復速度のボトルネックになるが、2倍超のスループット改善はその制約を緩和する。LangGraphで構築した監査エージェントのポリシーモデルとして12B規模のオープンモデルを使う選択肢として評価対象に値する。
