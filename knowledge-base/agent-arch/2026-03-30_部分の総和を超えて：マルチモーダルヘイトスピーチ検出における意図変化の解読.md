## 要約

ソーシャルメディア上のヘイトスピーチは、テキスト単体から画像とテキストを組み合わせたマルチモーダル表現へと移行しており、暗黙的な攻撃の自動検出が困難になっている。既存システムの課題は、各モダリティ単体では無害に見えるが、組み合わさることで有害な意味が生まれる「創発的意味」を捉えられない点にある。本論文はこの問題を二値分類（ヘイト/非ヘイト）ではなく、モダリティ間の相互作用による「意味的意図変化」として定式化する。具体的には、（1）無害なテキストと画像が組み合わさって暗黙的ヘイトを形成するケース、（2）テキストの毒性が画像の文脈によって意味論的に反転・中和されるケース、の2パターンを区別する。この定式化に基づき、H-VLI（Hate via Vision-Language Interplay）ベンチマークを構築。このデータセットは、明示的な視覚的・テキスト的スラーではなく、モダリティの複雑な相互作用によって真の意図が決まる事例を集めている。検出フレームワークとしてARCADE（Asymmetric Reasoning via Courtroom Agent DEbate）を提案。司法プロセスを模倣し、「起訴側」と「弁護側」の2エージェントが対話的に議論を展開することで、モデルが表層的な手がかりに頼らず深い意味的解析を行うよう強制する。最終判定は、この対審的なディベートを経て下される。H-VLIベンチマーク上でのARCADEの実験結果は、既存のSOTAベースラインを有意に上回り、特に暗黙的なケースでの改善が顕著であった。既存ベンチマークでも競争力のある性能を維持している。コードとデータはGitHub（Sayur1n/H-VLI）で公開済み。

## アイデア

- 「起訴側 vs 弁護側」という対審的なエージェント構造が、単一エージェントよりも深い意味的推論を引き出す設計原則—監査判断のような二項対立判断に転用できる
- 二値分類を超えて「意図変化のタイプ」を細かく分類する定式化は、監査でいえば『不正なし/あり』ではなく『リスクの性質と経路』を分類する粒度設計に対応する
- モダリティ単体では無害・複合で有害という創発的意味の問題は、監査エージェントが複数データソースを統合する際の「文脈依存的リスク評価」と構造的に同型

## Yujiの取り組みへの示唆

ARCADEの対審的マルチエージェント設計（起訴側・弁護側が議論し最終判定を下す）は、LangGraphで実装する監査エージェントの「リスク有り/無し」二項判断に直接応用できる。特に、単一エージェントが見落としやすい「文脈依存的なリスク」を、役割の異なるエージェントの議論で顕在化させるアーキテクチャは、LLM-as-judgeの精度向上にも有効な示唆を含む。H-VLIのような意図変化の細粒度分類手法は、監査証跡の異常を「異常のタイプ」として分類する際のラベル設計にも参考になる。
