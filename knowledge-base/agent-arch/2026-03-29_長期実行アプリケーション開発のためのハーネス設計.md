## 要約

AnthropicのエンジニアがClaude向けに構築した、フロントエンドデザイン生成と長期自律コーディングの2領域における多エージェントハーネス設計の詳細報告。GANにインスパイアされたgenerator/evaluatorの分離アーキテクチャを採用。主な課題として①コンテキストウィンドウ充填による coherence 低下・「コンテキスト不安」、②自己評価の甘さ（自分の出力を過大評価する傾向）を特定。対策として、同一エージェントでの要約（compaction）ではなく完全なコンテキストリセット＋構造化ハンドオフを採用。フロントエンドデザインでは「デザイン品質・独自性・クラフト・機能性」の4基準を定義し、evaluatorエージェントにPlaywright MCPで実ページを操作させてスコアリング。最終的にplanner/generator/evaluatorの3エージェント構成で、数時間にわたる自律フルスタックアプリ構築を実現。Claude Agent SDKをオーケストレーション基盤として使用。

## 要点

- 自己評価の甘さはLLMの本質的傾向であり、generator/evaluatorを分離してevaluatorを「懐疑的」に調整する方が、generatorを自己批判的にするより遥かに実効性が高い
- 長期タスクではcompactionよりもコンテキストリセット＋構造化ハンドオフが有効——クリーンスレートを与えることでコンテキスト不安を根本解消できる
- 主観的品質（デザインの良し悪し等）も、具体的な採点基準（4基準＋重み付け＋few-shot例）に落とし込むことでエージェントが安定して評価・改善できる
