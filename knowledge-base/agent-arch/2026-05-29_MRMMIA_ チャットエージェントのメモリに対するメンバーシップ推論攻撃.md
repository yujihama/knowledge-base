## 要約

メンバーシップ推論攻撃（MIA）とは、ターゲットデータレコードがシステムのプライベートデータに含まれるかどうかを判定する手法であり、機械学習システムのプライバシー漏洩を測定するための標準的なツールとして確立されている。従来研究では学習コーパスや検索データベースへのMIAが主な対象だったが、チャットエージェントのメモリストアに対するMIAは十分に研究されていなかった。エージェントメモリにはユーザーとエージェントのインタラクション履歴、検索により取得したファクト、ユーザーの嗜好情報などが含まれており、これらはセンシティブな情報を多分に含む。本論文では、チャットエージェントのメモリを対象とするMIAとして「Multi-Recall Memory MIA（MRMMIA）」を提案する。MRMMIAは複数のリコールプローブをエージェントに送信し、候補メモリユニットがエージェントのメモリストアに存在するかどうかを推定するための統一的な攻撃フレームワークである。攻撃設定としてブラックボックス・グレーボックス・ホワイトボックスの3種類をカバーしており、各設定でベースライン手法を一貫して上回る性能を示した。実験結果はエージェントメモリにおけるプライバシーリスクを定量的に明らかにし、チャットエージェントメモリシステムにおけるメンバーシップ漏洩評価のための初期フレームワークを提供するものとなっている。監査エージェント開発への示唆として、エージェントが蓄積するメモリ（会話履歴、内部統制関連ファクト、審査対象情報）は推論攻撃によって外部から漏洩するリスクがあり、セキュアなメモリ設計（アクセス制御、メモリ分離、差分プライバシー適用等）が不可欠となる点は重要である。

## アイデア

- エージェントのメモリストアはRAGの検索DBと異なり動的に更新・蓄積されるため、静的データセット向けのMIAをそのまま転用できない点が新規性の核心
- 複数の『リコールプローブ』を組み合わせてメンバーシップシグナルを抽出するアプローチは、単一クエリより頑健で、エージェントのAPI応答のみから情報を引き出せるブラックボックス設定でも有効
- 監査AIシステムでは被監査情報がエージェントメモリに蓄積されるケースが多く、本攻撃フレームワークはそのリスク評価ツールとして直接転用可能
