## 要約

本記事は、OpenAI の Codex（コーディングエージェント）に対して「Fable 5」スタイルのワークフローガイドを適用するための軽量スキル層「Codex Fable5 Skill」の紹介である。Fable 5 とは、構造化・検証重視のエージェント動作フレームワークの一種で、本スキルはそれをコピーやアンロックするものではなく、Codex の動作をより慎重・体系的にするためのワークフローレイヤーとして設計されている。

主要コンポーネントは以下の4つ。①「ツール優先の Agent ループ」：Codex がコードを推測で生成する前にツール呼び出しを優先させることで、根拠のない出力を抑制する。②「ゴール台帳（Goal Ledger）」：タスクの目標を明示的に記録し、エージェントが途中で目的を逸脱しないよう管理する。③「エビデンスチェックポイント（Evidence Checkpoint）」：各処理ステップにおいて、出力が証拠に基づいていることを確認するための中間検証を挿入する。④「最終検証ゲート（Final Verification Gate）」：タスク完了前に最終的な品質確認フェーズを強制する。オプションとして「Provider Bridge ドキュメント」も用意されており、異なるモデルプロバイダー間のインタフェース差異を吸収するための記述を提供する。

この取り組みは、LLMエージェントが「確信を持った誤り」を高速に出力するという既知の問題に対処するアプローチとして注目できる。特にエビデンスチェックポイントとゴール台帳の組み合わせは、監査エージェントシステムにおいて「エージェントが何を根拠に判断したか」を追跡する監査証跡（Audit Trail）設計に直接応用可能な考え方である。LangGraph での ReAct ループに同様のチェックポイントノードを挿入し、各ステップの根拠を Pydantic モデルで構造化記録する設計との親和性が高い。

GitHub（https://github.com/baskduf/FableCodex）にて公開されており、スター数は記事執筆時点では少数だが、コーディングエージェントの信頼性向上という方向性は実用的である。

## アイデア

- エビデンスチェックポイントを Agent ループの中間ノードとして挿入する設計は、LangGraph の条件分岐ノードと組み合わせることで、監査エージェントの根拠追跡に直接流用できる
- ゴール台帳（Goal Ledger）という概念は、長いタスクでエージェントが目的を逸脱する問題への対処として、Pydantic モデルで構造化された目標状態管理として実装できる
- ツール優先ループは「推測より検索・実行を先に行う」原則を強制するもので、コーディングエージェントの幻覚（hallucination）を抑制する実用的なヒューリスティックとなっている
