## 要約

multi-agent-shogunは、Claude Code + tmuxを使い複数AIエージェントを並列運用するOSSフレームワーク。Lord→Shogun→Karo→Ashigaru/Gunshiという戦国時代の軍制をモチーフにした階層構造を持ち、YAMLファイルベースのメールボックスによるイベント駆動通信（inotifywait使用、ポーリング禁止）、エスカレーション機構（2分放置でEscapeキー送信、4分でclearリセット）、Gunshiによる品質チェックフローを備える。

筆者はこのフレームワークをprivate forkし、アイドルマスター シンデレラガールズのキャラクターをエージェントに割り当てた。Shogunに依田芳乃（戦略・指揮）、Karoに大石泉（タスク分解）、Gunshiに八神マキノ（QC・分析）、Ashigaruに黒埼ちとせ/若林智香/荒木比奈/新田美波（BE・FE・ライター・バックエンド専門分業）を配置。

主要な独自追加・拡張は7点。①ペルソナ3層設計：CLAUDE.md（自動読込）→persona-core.md（詳細ルール）→personas.yaml（870行のキャラ定義）で、コンテキストリセット後もペルソナを復元可能にした。②AI間通信の「ロッキーペルソナ」：エージェント間メッセージは簡潔3行のロッキーペルソナを使用し、トークン消費を削減。③QCフライホイール：違反パターンをYAMLに蓄積→チェックリスト化→Ashigaru指示書に昇格するサイクルを構築。axiosのbaseURL二重プレフィックス、git add -Aによる未追跡ファイル混入等の再発を防止。④安全ルールの独立ファイル化：upstream push禁止やWSL2環境固有ルール（/mnt/c/配下の破壊的操作禁止）を追加。⑤スケジュール実行：crontab連携で平日朝8時・休日深夜2時にバッチタスクを自動投入。⑥Production Portal：FastAPI + Vue 3 PWAでダッシュボードをブラウザから確認でき、Tailscale経由で外出先アクセスも可能。⑦依存マップ：3ファイルに分散したペルソナ定義の暗黙依存関係を依存マップとして明文化。

設計思想として「AI自律型から人間主導型へ」の転換も行い、定型処理はシェルスクリプトに移譲してLLMには判断・生成のみを担当させるコスト意識の高いアーキテクチャを採用している。

## アイデア

- AI間通信と人間向け出力でペルソナを分離する「ロッキーペルソナ」パターン：エージェント間は簡潔3行ペルソナ、人間向けのみキャラ口調を使うことでトークンコストを構造的に削減する設計
- QCフライホイール：品質チェックで検出された違反パターンをYAMLに蓄積→チェックリスト化→エージェント指示書に昇格させる自己改善サイクルで、同一ミスの再発を防ぐ学習機構
- コンテキストリセット耐性のある3層ペルソナ設計：Claude Codeの自動読込（CLAUDE.md）→詳細ルール（rules/）→完全定義（config/yaml）の階層化により、リセット後も最低限の一貫性を担保する

## Yujiの取り組みへの示唆

監査エージェントシステム（LangGraph + ReAct）開発において、QCフライホイールの設計は直接参考になる。Gunshi（QC担当エージェント）が違反パターンをYAMLに蓄積してチェックリスト化するサイクルは、監査エージェントの判断ルール・例外パターン蓄積機構として転用可能。またペルソナ3層設計やAI間通信の簡潔ペルソナ分離は、マルチエージェント監査システムでの役割分担設計とトークンコスト管理の実践例として参考になる。人間主導型への転換（定型処理のスクリプト移譲）は、LLM-as-judgeの適用範囲を絞りコストを抑える設計指針にも合致する。
