## 要約

プログラミング未経験の公務員が2025年5月からVibe Coding（自然言語でAIに指示してコードを生成させる手法）を実践し、1年間でGitHubに20以上のプロジェクトを公開した記録。使用技術スタックはPython・TypeScript・GAS・React・FastAPIなど15種以上。テストコードは4,798件（pytest）、月間LLM処理量は27.4億トークンに達する。

代表プロジェクトとして、10以上の専門エージェント（要件定義・計画・実装・テスト・レビュー）を6フェーズのパイプラインで協調動作させるマルチエージェントフレームワーク「NexusCore」、LINE Messaging API + Stripe決済 + Cloudflare Workersで構築した予約管理システム「Reserve Optimizer」（Stripe Webhookの署名検証・冪等性チェック・タイミング攻撃対策を日本語指示で実装）、Flask + Playwright + 価格最適化エンジンの物販管理システム等がある。

品質管理はコードを読めない前提で仕組み化。①AIにテストコードを生成させNexusCoreで全4,798テストをパス、②アーキテクチャ・セキュリティ・テスタビリティ等7軸でAIが生成したコードを別AIに★1〜5評価させる自作レビュープロンプト（3プロジェクト30ファイルでセキュリティとアーキテクチャ全A評価達成）、③mutmutによるMutation Testing（543変異中325を検出）、④GLM・MiniMax・Claude Opusの3LLMクロスチェックで単一バイアスを軽減。

コスト最適化はLLMルーティングで実現。Layer 1: GLM-5.1（$160/月、処理量85%）、Layer 2: MiniMax（$20/月、14%）、Layer 3: Claude API（従量、1%未満）の3層構成で合計$180/月（約28,000円）。429レートリミット対策のローカルプロキシも自作し、使用率に応じてモデルを自動切替。

得られた知見として、Vibe Codingの核心は「ドメイン知識を正確に言語化する設計力」であり、アーキテクチャ指示（Handler/Service層分離、Agent/Orchestratorパターン）やセキュリティ要件はAIへの明示的な指示で高品質に実装できる一方、テスタビリティ（DI・コンストラクタ注入の明示が必要）と複雑バグのデバッグは工夫が必要。全プロジェクト横断の意思決定記録323件をObsidian SSOTで管理し、なぜその設計にしたかを全てトレース可能にしている。限界として、コード読解・チーム開発経験・深いデバッグの困難さを著者自身が明示している。監査エージェント開発への示唆：品質管理の仕組み化（多層レビュー・Mutation Testing）とLLMルーティングによるコスト最適化は、監査エージェントの本番運用設計に直接応用できる。

## アイデア

- AIが生成したコードを別AIに7軸評価させる自作レビュープロンプトで、コードを読めない状態でも定量的な品質担保が可能になる点
- GLM・MiniMax・Claude APIの3層LLMルーティングで月$180・27.4億トークン処理を実現し、Claude Code Max（$220/月）より安価に構成できる点
- mutmutによるMutation Testingを非エンジニアが導入し、543変異中325検出・残218を等価変異と確認することでテスト品質を客観測定した点
