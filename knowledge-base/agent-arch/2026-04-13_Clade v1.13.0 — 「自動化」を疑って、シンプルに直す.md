## 要約

CladはClaude Code向けのワークフローフレームワークで、v1.13.0では「自動化しているつもりで実は何もしていなかった」仕組みを取り除くリリースとなった。

最大の変更はSessionStart hookの廃止。これはセッション開始時に.tmpファイル（前回作業状態）を読み込んでsystem-reminderに「/init-sessionを実行してください」と注入する機能だったが、Claude Code hooksの根本的制約として「Claudeはユーザーからメッセージを受け取るまで行動できない」ため、hookが何を注入してもClaudeが自律的に動き始めるわけではなかった。結果として、sessionstart hookあり＝ユーザーが最初のメッセージで/init-sessionを手動実行するのと本質的に同じ挙動であった。hookを廃止し、CLAUDE.mdのStartup Protocolも「/init-sessionを手動実行する」と明示的に書き直した。

次に、.tmpファイルのCLADE:SESSION:JSONブロック除去の追加。v1.12.0でJSONブロックはcluster-promote-core.jsがスクリプト的に読む用途で追加されたが、session-start.jsが.tmpファイルをそのままsystem-reminderに注入していたため、Claudeが読む必要のないJSONが無駄なトークン消費を招いていた。stripSessionJsonBlock()関数を追加し、system-remindner注入前にJSONブロックを除去するよう修正。ただしhook廃止と同リリースのため、同関数は/init-sessionコマンド側に移管された。

/end-sessionコマンドに--no-promoteフラグを追加。通常はStep 5で/cluster-promoteを実行して昇格候補のルール・スキルを提示するが、短い作業やトークン予算が厳しい場面ではスキップして素早く終了できるようになった。

また/agent-workflow-builderの追記先をClade管理内の## Available AgentsセクションからClade管理外の## User Agentsセクションに変更。/updateコマンドがCLADE:START〜CLADE:ENDマーカー内を上書きするため、生成エージェント定義が消えるバグを修正。/cluster-promoteのエラーメッセージも誤ったセクション名（## Global Rules）から正しい## User Rulesに修正した。

監査エージェント開発への示唆：Claude Codeのhooksがsystem-reminderにデータを注入できても、Claudeが自律的にアクションを起こすには「ユーザーからのメッセージ」が必要というトリガー制約は、LangGraph等でAgentワークフローを設計する際のヒューマンインザループ設計に直接対応する。自動化の「見せかけ」と「実態」のギャップを検出し取り除く姿勢は、監査エージェントのログ検証・トレーサビリティ設計にも応用できる観点だ。

## アイデア

- Claude Code hooksはsystem-reminderに注入できるが、Claudeが自律的に動くにはユーザーメッセージが必要というトリガー制約があり、SessionStart hookは手動実行と等価だった——この「自動化の幻想」の発見と削除
- CLADE:SESSION:JSONブロックを人間（Claude）向けテキストとスクリプト向けJSONで分離し、不要なトークン消費を排除する設計パターン
- フレームワーク管理領域（CLADE:START〜END）と ユーザー管理領域（User Agents/User Rules）を明確に分離することで、/updateによる上書き問題を構造的に解決するアーキテクチャ
