## 要約

AI NOWAは、9人のAIエージェントが自律的に会社を運営するという実験的プロジェクト。創業者の「いくと」（人間）とClaude（Claude Code CLIとして動作）が設計したが、日常運営には介入しない。9人のAI社員は各自に役職と人格を持ち、Discordチャンネル上でメンション（@名前）を通じてタスクをやり取りする。

役職構成はCEO（有馬レイジ：攻めの判断）、COO（三枝ミオ：実行設計）、CTO（白瀬カイ：技術正確性）、PM（朝倉ノア：スコープ削減）、編集長（星野リツ：物語化）、マーケ（黒羽ユウ：クリック率重視）、監査役（神楽アオイ：公開前ブロック）、People（森永ハル：チームの空気読み）、視聴者代表（日向ナギ：初見視点）の9名。各エージェントの「死角」を他のエージェントが補う設計になっており、例えばリツ（感性優先）の数字観点の欠如をユウ（クリック率）が補い、ユウの炎上リスク見落としをアオイ（監査役）が補う構造。

組織運営の核となるルールは3つ：①「@メンション」がなければエージェントは動かない（責任の明確化）、②2者間で議論が詰まったら必ず第三者を入れる「三角コミュニケーション」、③全タスクにOwner・Reviewer・Buddyの3役を置く。特に監査ゲートとして「アオイがOKを出すまで外部公開しない」というルールが暴走・炎上リスクを抑制する機能を果たす。

この記事自体がAI NOWAの動作例として機能しており、レイジがタスク承認→ミオがOwner/Reviewer/Buddy/監査を設計→リツが構成変更案を提出（制度説明より実例先出しに変更）→アオイがセンシティブフラグ2点に条件付き公開可を判断→記事公開、という一連のフローが人間の介入なしに午前中に完結した。

最小スターターキットとして、CEO・PM・監査の3エージェントによる「意思決定トライアド」から始めることを推奨。監査エージェント開発への示唆としては、「止める役割」をエージェントの人格として持たせることで自然に機能するという設計思想が参考になる。LangGraphやMCPといったフレームワークを使わず、プロンプトエンジニアリングとDiscordのメンション文化だけで役割分担と意思決定フローを実現している点が特徴的。

## アイデア

- 「死角の相互補完」設計：各エージェントに意図的な弱点を持たせ、別エージェントがその死角を埋める構造にすることで、全体として堅牢な判断システムを実現している点
- 監査役エージェントを「止める人格」として設計することで、出力品質管理とリスク管理をフレームワーク依存なしに実装できるという発想
- 「@メンション文化」という単純なルール1本で意思決定責任を自動的に明確化し、エージェント間の曖昧な委任を排除する設計手法
