## 要約

IBM ResearchがオープンソースPythonライブラリ「Mellea 0.4.0」と3つの「Granite Libraries」（granitelib-rag-r1.0、granitelib-core-r1.0、granitelib-guardian-r1.0）を同時リリースした。

Melliaは汎用オーケストレーションフレームワークとは異なり、確率的なプロンプト挙動を「構造化・保守可能なAIワークフロー」に置き換えることを目的とした設計思想を持つ。中核的な仕組みは「制約付きデコーディング（constrained decoding）」であり、LLMの出力をスキーマ準拠に強制することで予測可能性を担保する。バージョン0.4.0での主な追加機能は以下の3点：(1) Granite Librariesとのネイティブ統合（制約付きデコーディングによるスキーマ正確性保証のための標準化API）、(2) rejection samplingによる「instruct-validate-repair」パターン（生成→検証→修復のループ）、(3) イベント駆動コールバックによる観測性フック（ワークフローの監視・トレーシング）。

Granite Librariesは、granite-4.0-microモデル向けに設計されたLoRAアダプタ群であり、それぞれ特定タスクに特化したファインチューニングが施されている。granitelib-core-r1.0はinstruct-validate-repairループの要件検証ステップを担当し、granitelib-rag-r1.0はアジェンティックRAGパイプラインの事前検索・事後検索・生成後処理をカバーする。granitelib-guardian-r1.0は安全性・ファクトチェック・ポリシーコンプライアンスの確認に特化する。汎用プロンプティングではなく専用アダプタを用いることで、ベースモデルの能力を損なわずに各タスクの精度を向上させる設計である。

全体として、このリリースは「LLM出力の非決定性をソフトウェア工学的に制御する」アプローチの具現化であり、エンタープライズ向けの信頼性・監査可能性が求められるユースケースへの適用を想定していると考えられる。

## アイデア

- 制約付きデコーディングによるスキーマ準拠保証：LLMの出力を確率的挙動ではなくスキーマに強制適合させる手法は、監査エージェントが生成する構造化レポート（Pydanticモデル等）の信頼性確保に直結する
- instruct-validate-repairループ：生成→rejection samplingによる検証→修復という3ステップのループ設計は、ReActエージェントのself-correction機構と組み合わせ可能な汎用パターンとして注目に値する
- granitelib-guardian-r1.0のポリシーコンプライアンスチェック：ファクトチェックと政策準拠確認を専用LoRAアダプタで行う設計は、監査AIにおける「根拠の検証」と「法規制・内部統制基準への照合」に対応できる可能性がある

## Yujiの取り組みへの示唆

Melliaのinstruct-validate-repairパターンは、YujiがLangGraph＋PydanticでReActエージェントを構築する際の出力検証・修復ループの設計参考になる。granitelib-guardian-r1.0の安全性・ポリシーコンプライアンスチェック機構は、内部監査エージェントが生成した所見や判断の根拠を自動検証する「LLM-as-judge」レイヤーとして流用できる可能性がある。制約付きデコーディングによるスキーマ正確性保証はPydanticモデルとの親和性が高く、監査レポートの構造化出力の信頼性向上に応用できる。
