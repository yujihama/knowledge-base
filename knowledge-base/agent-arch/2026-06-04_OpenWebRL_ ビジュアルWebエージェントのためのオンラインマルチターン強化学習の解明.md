## 要約

OpenWebRLは、実際のWebサイト上でビジュアルWebエージェントをオンラインマルチターン強化学習（RL）で訓練するためのオープンフレームワーク。従来の教師あり事後学習（SFT）に依存するアプローチは、高品質なデモンストレーションの収集コストが高く、静的データセットでは多様・変化するWebのカバレッジが限られるというスケーラビリティのボトルネックがあった。

本フレームワークは訓練パイプライン全体をカバーし、(1) スケーラブルなライブブラウザインフラ、(2) 教師あり初期化（SFT）、(3) マルチモーダルコンテキスト管理、(4) トラジェクトリレベルの成功判定、(5) 効率的なマルチターンポリシー最適化という5つのコンポーネントで構成される。

本フレームワークを用いて訓練されたOpenWebRL-4Bは、わずか0.4K本の初期化トラジェクトリと2.2Kのオープンエンドなタスクを用いたRL訓練のみで、ライブWebベンチマークにおいて新たなオープンソースSOTAを達成。Online-Mind2Webで67.0%、DeepShopで64.0%の成功率を記録し、同規模・大規模な既存オープンエージェントを上回るだけでなく、OpenAI CUAやGemini CUAといったプロプライエタリシステムとも競争力のある性能を示した。

ビジュアルWebエージェントにとってオンラインRLが有効な理由として、(1) 動的なWebの多様なシナリオに対してデモなしで適応できること、(2) トラジェクトリ全体の成功/失敗を報酬信号として使うことで長期ホライズン推論が強化されること、(3) データ収集コストを大幅に削減できることが挙げられる。また、RLがエージェントの推論プロセスをどのように改善するかも体系的に分析しており、再現性・コスト効率の高いWebエージェント開発への実践的な道筋を提供する。訓練データ・モデル・コードはすべて公開予定。

監査エージェント開発への示唆：長期ホライズン推論と動的環境への適応は監査エージェントにも共通の課題であり、少量のSFT初期化＋オンラインRLという組み合わせは、監査手続の多様なシナリオへの対応策として参考になる。また、トラジェクトリレベルの成功判定はLLM-as-judgeの実装設計にも応用可能。

## アイデア

- 0.4Kトラジェクトリという極めて少量のSFT初期化でオンラインRLに移行できる点は、データ収集コストとの兼ね合いでSFT依存からの脱却を示すベンチマーク的な知見
- トラジェクトリレベルの成功/失敗を報酬とするアプローチは、ステップ単位の報酬設計が困難な長期タスク（監査手続の自動化など）に適用できる設計パターン
- ライブブラウザインフラ上でのスケーラブルなオンライン訓練は、静的データセットでは捉えられないWebの動的変化への適応を可能にし、継続的改善ループの構築に繋がる
