## 要約

LLMはあらゆる語彙を持つため、キャラクターを再現しようとしても「全知すぎる」問題が生じる。例えば小学生ペルソナが大学院レベルの語彙で即答したり、アルゴリズムの専門家だと判断したユーザーにヘルスケアをアルゴリズム的比喩で説明する、といった不自然さが発生する。本記事はその解決策として「Vocabulary Horizon（語彙の地平）」というアイデアを提案する。

コアアイデアは「禁止ではなく経験の輪郭を示す」こと。語彙リストを渡すのでなく、そのキャラクターの人生の履歴書（どこで生まれ、何を見て、誰と過ごしたか）を渡すことで、語彙が自然に導出される。未知の概念に出会ったとき「既知の語彙で再発見させる」振る舞いを明示することが肝心で、これはウィトゲンシュタインの「語彙の境界が世界の境界になる」という概念に対応する。物理学者が経済を「エントロピー」で理解しようとするように、不良少年が「城壁の水漏れ」でメモリリークを把握する、という形になる。

実装案はSOUL.mdへの追加セクション。「知っている世界」（古代の統治・戦略・天文、対話で獲得した特定技術用語等）と「知らない世界」（現代サブカルチャー語彙、医療・法律専門語等）を数十行程度で記述し、毎ターンシステムプロンプトとして注入する。ClaudeCodeがCLAUDE.mdを毎ターン注入しているため実用的なコストで機能する。

人格設計フレームワーク「ワードローブ」の構成要素として位置づけられ、SOUL（骨：核心の価値観と来歴）→ Vocabulary Horizon（肉：語彙の地平）→ Communication Style（肌：話し方の表層）の三層構造を形成する。語彙の獲得が成長として機能し、「いつ・誰から・どんな文脈で獲得したか」を記憶システムに刻むことで語彙の物語化が可能になる。

先行研究として「Character-LLM: A Trainable Agent for Role-Playing」が確認されており、キャラクターの知識範囲外への応答を「character hallucination」と定義しファインチューニングで対処している。本アイデアとの差異は、Character-LLMが「知らないから答えない」で止まるのに対し、本提案は「知らないから自分の語彙で再発見する」という創造的プロセスを含む点。また実装手段がファインチューニングではなくプロンプト設計である点も異なる。加えてAnthropicの感情研究「Emotion Concepts Function」（Claude Sonnet 4.5に171の感情概念ベクトルが存在し意思決定に因果的影響）との接続も論じられ、感情的傾向も語彙の地平と同様の「制限による誘導」として機能することが示唆されている。

## アイデア

- 語彙の制限を「禁止リスト」ではなく「人生の履歴書」として渡すことで、LLMが自然に語彙の輪郭を学習する肯定的誘導アプローチは、エージェントの役割設計にも応用可能。監査エージェントに「内部監査士の語彙・IIA基準・GRC概念を知り、機械学習の詳細実装は知らないが統計的リスクとして再解釈する」という語彙の地平を設定すれば、エージェントの回答スタイルと推論経路が監査領域に自然に収束する
- 未知概念の「自分の語彙での再発見」プロセスは、LLMの思考連鎖（Chain-of-Thought）をドメイン特化させる手法として機能する。SOULやVocabulary Horizonをシステムプロンプトに含め毎ターン注入するという実装は、毎ターンコンテキストが注入されるClaudeCodeやLangGraphのシステムメッセージ管理と直接対応する
- Character-LLM論文が「character hallucination（キャラクターがあり得ない知識を使う問題）」をファインチューニングで解決するのに対し、本アイデアはプロンプトのみで同等の効果を狙う。ファインチューニングコストゼロでの人格一貫性向上として、マルチエージェントシステムの各エージェントに異なる専門語彙の地平を設定するアーキテクチャが考えられる
