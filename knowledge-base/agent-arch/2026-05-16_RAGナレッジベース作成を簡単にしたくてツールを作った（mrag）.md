## 要約

mrag（Micro RAG）は、ローカル環境で動作する小規模RAGパイプライン構築・運用のためのPython製CLIツール。作者はDifyを使ったRAGチャットボット開発を行ってきたが、Claude Desktop/Codex/Gemini CLIなどのデスクトップエージェントが普及する中で、それらに特化したナレッジベースを手軽に供給できるツールがなかったことが開発動機。技術スタックはPython + SQLite + Qdrant + Ollamaで完全ローカル動作。Tokenizerにはsqlite-vaporettoによる形態素解析を使用し、EmbeddingにはOllama上のbge-m3（多言語対応）を採用。検索戦略は4種類（hybrid: SQLite FTS5 BM25 + Qdrantベクトル検索、keyword: BM25のみ、vector: コサイン類似度のみ、parent_child: Parent-Child Retrieval）をYAMLベースのRetrieval Profileで切り替え可能。Chunkingは4戦略（recursive, markdown_recursive, block_aware, parent_child）があり、Markdownのテーブル・コードブロックをアトミック単位として保護するblock_aware戦略が特徴的。Contextual Augmentationとして、インデックス時にOllama（gemma4:e4b）でチャンク毎にコンテキストをLLM生成する機能（Anthropicのcontextual retrieval手法に準拠）も実装。Rerankerとしてsentence-transformersのCrossEncoderによる再スコアリングも対応。mrag serveコマンドでFastAPIサーバを起動しDify外部ナレッジAPIに準拠したエンドポイントを提供することで、既存DifyワークフローとのAPI連携も可能。基本操作はmrag init → mrag add → mrag index → mrag searchのみで、mrag evalで複数Retrieval Profileの検索品質を定量比較できる。SKILL.md（エージェント向け操作手順書）をClaude Code等に読み込ませることで、エージェントがmragコマンドを自律実行するAgentic RAGワークフローを実現できる点が独自性。監査エージェント開発の観点では、社内文書や監査調書をローカルでナレッジ化しつつ、Retrieval Profileで検索精度を定量評価・チューニングできる点が有用。クラウドに文書をアップロードせずにRAGを構築できるため、機密性の高い監査関連文書にも適用しやすい。

## アイデア

- SKILL.mdというエージェント向け操作手順書をツールに同梱し、Claude CodeやCodexがmragコマンドを自律実行できる設計にした点：ツール自体がエージェントフレンドリーなインターフェース仕様を持つという発想が、エージェントオーケストレーション設計に応用できる
- Retrieval ProfileをYAMLで定義しmrag evalで複数プロファイルの検索品質を並列比較できる構成：RAGの精度チューニングをコード変更なく設定ファイルの差し替えで実験できるため、監査エージェントのRAG精度改善サイクルを体系化できる
- インデックス時にLLMでチャンク毎のコンテキストを生成するContextual Augmentation（Anthropic手法準拠）と、teeによるログ監視でエージェントが進捗把握しながら長時間処理を自律実行する組み合わせ：長時間タスクにおけるエージェントの自律的な状態監視パターンとして参考になる
