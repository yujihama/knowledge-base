## 要約

社内ドキュメントをRAGに投入すれば「ベテラン社員」相当のAIになるという期待は誤りであり、その根本原因を「参照汚染（Reference Contamination）」として定義し、対策設計として「BRM（Branching Reference Model）」を提唱する記事。

RAGは意味的な近接性で文書を検索するが、「検索できること」と「正しく参照できること」は別問題である。社内に存在する文書は現在有効なポリシー、廃止済みルール、未承認ドラフト、顧客別例外対応、却下された提案など異なる権威・鮮度・適用範囲を持つ。これらを区別せず同一の検索空間に投入すると、AIは社内用語や過去事例を含む「もっともらしい混合回答」を生成する。この状態は純粋なハルシネーションより危険であり、利用者には「社内情報に基づいた回答」に見えてしまう。

BRMは思考・記憶・業務文脈を分岐構造（Branch）として扱う設計思想。構成要素は3つ：①Node（認知的動きの最小単位。大きすぎるチャンクは仮説・決定・反論を混在させるため、小さく切ることで状態変化を保存する）、②Edge（NodeをSequential/Thematic/Causal/Decision Linkの4種で型付けし、単なる近接検索ではなくナビゲーションを実現する）、③Decision Link（AIの提案と人間の承認済み意思決定を構造的に分離するアンカー）。

BRMにおけるRAGは文書ではなく「スコープされた参照」を取得する。クエリは「意味的に似た文書はどれか」ではなく「どの枝が関係するか、どのDecision Linkが適用されるか、どの参照を混ぜてはいけないか」に変わる。また生成物（Artifact：要約・ドラフト・レポート等）は常時アクティブコンテキストへ混入せずリンクとして保持し、生成物による推論汚染を防ぐ。

コンテキストウィンドウの拡大はこの問題を解決しない。分岐構造がなければ大きなウィンドウは「大きな沼」になるだけである。AIに必要なのは「より多くの文脈」ではなく「正しい枝から正しいタイミングで正しい権威レベルで渡される参照」である。設計パラダイムの転換として、memory accumulation→reference architecture、document retrieval→branch navigation、context expansion→context hygieneを提唱している。

監査エージェントへの示唆：監査証拠・ドラフト調書・承認済み調書・過去例外対応を同一RAG空間に投入することは参照汚染の典型例となる。Decision Linkによる承認済み/未承認の構造的分離は、監査調書の完全性・来歴管理の要件と直接対応する。

## アイデア

- 「参照汚染」という概念の明確化：RAGの失敗モードを情報量不足ではなく参照空間の境界崩壊として定義した点。ハルシネーションより危険な理由（社内用語・過去事例を含むため信頼されやすい）が論理的に整理されている
- Decision Linkによる提案と決定の構造的分離：AIの生成物と人間の承認済み意思決定を同一コンテキストに混在させないためのアンカー機構。監査・コンプライアンス領域でのAI活用に直接応用可能な設計原則
- Artifactをアクティブコンテキストから切り離す設計：要約・ドラフト等の生成物を常時混入させず「リンク」として扱うことで、モデルが自分自身の残渣から推論するドリフトを防ぐ考え方
