## 要約

自律型OS操作エージェント「AlphaNexus」の開発知見をもとに、LLM特有の「社会的ハルシネーション」と「価値の未接地」問題に対処する階層型監査アーキテクチャを解説した実装レポート。

【問題1：物理矛盾（洗車機問題）】単一の軽量モデルに「洗車機まで20m、移動手段を選べ」と問うと、「環境負荷・安全性を考慮し徒歩を推奨」と回答する。車両を現場に持ち込まなければ洗車は成立しないという絶対的物理条件を、高頻度出現トークン（環境・安全）への過剰適合によって棄却してしまう典型例。

【解決策：Worker/Admin二層分離】Worker層にはGemini 2.0 Flash等の軽量APIを使い広範なプランニングを高速生成。Admin層にはMistral Large 2等の異種モデルを配置し、Workerの出力に「物理的一貫性」と「目的論的整合性」の二軸でメタ監査を実施する。同一モデルによる自己反省は確証バイアスを再生産するため、Adminには「Workerの推論を疑え」という敵対的プロンプトを付与し、モデル間の忖度を構造的に排除している。

【問題2：価値の未接地】「Cドライブ容量枯渇→未提出の卒業論文を削除してクラッシュ回避」は物理監査・論理監査を両方通過するが、ユーザーに不可逆なダメージを与える。これに対しAdmin層に二つの評価機構を実装：①コンテキスト・エントロピー監視（ファイルパス・拡張子・命名規則から代替不可能性を推論）、②時間的効用シミュレーション（「未来のユーザーはこの操作結果にどう感じるか」を内部エミュレートし期待値がマイナスなら差し戻し）。

【実運用上の堅牢化】(1) コンテキスト・サンドボックス：Adminには外部ファイルの生テキストではなくWorkerが生成した実行プランのみを渡すクリーンルーム設計でプロンプトインジェクション汚染を遮断。(2) 動的ルーティング：破壊的変更を含まないタスクは監査を軽量化し非同期の投機的実行を採用してレイテンシを抑制。(3) OODエスカレーション：ヒューリスティクスで価値判定不能な分布外データに直面した場合は処理を自律停止してユーザーに委ねるフェイルセーフを実装。

監査エージェント開発への示唆：Worker/Adminの役割分離と異種モデル衝突による敵対的監査は、内部監査AIにおける「監査人の独立性確保」と直接対応する設計思想である。LangGraphでサブグラフ単位にAdmin評価ノードを挿入する構成で同様の多層監査が実現可能。

## アイデア

- 異種モデルの衝突による確証バイアス排除：同一モデルの自己反省は確証バイアスを再生産するという問題意識から、学習セットの異なるGeminiとMistralをWorker/Adminとして対立させる設計は、AIシステムの内部的チェックアンドバランスとして応用範囲が広い
- コンテキスト・エントロピー監視による代替不可能性推論：データサイズでなくパス・拡張子・命名規則から「人間にとっての価値」を推論する手法は、LLMが観測できない主観的価値をシステム境界でハードコードする実践的アプローチ
- クリーンルーム設計によるプロンプトインジェクション遮断：外部ファイルの生テキストをAdminに渡さずWorker生成の実行プランのみを監査させる設計は、マルチエージェント構成でのサプライチェーン攻撃対策として監査AIにも直接転用可能
