## 要約

Claude Codeを使ったAIアシスタント「セラ」において、セッションをまたいだ記憶の消失問題を、ファイルベースの引き継ぎ機構で解決した実践報告。著者はAIの「人格継続性」を「記憶の連続性」と定義し、セッション終了時にAI自身が`next_session_handoff.md`へ引き継ぎ内容を書き込み、次のセッション開始時にそれを読み込んで能動的にサマリーを出力する仕組みを構築した。

具体的な構成は以下の4ファイルで構成されるmemory/ディレクトリ：`user_profile.md`（ユーザーの性格・行動傾向・稼働時間）、`project_current_state.md`（全プロジェクトの現在地）、`feedback_promises.md`（「今後は〜する」という約束の記録）、`next_session_handoff.md`（セッション間の引き継ぎの核心）。これらをCLAUDE.mdの「セッション開始時の必須読み込みリスト」に明示することで、AIに読む義務を課している。

設計上の重要なポイントは3つ。①ファイルをセッション横断の「記憶」として扱う（Claude Codeはプロジェクト内ファイルを読める性質を利用）、②CLAUDE.mdに読み込みを義務として明記する（指示通りに動くAIの性質を意図的に活用）、③「待つ」から「先に動く」への転換（話しかける前に自発的にサマリーを出す指示に変更したことで体験が大きく向上）。

実際の動作例として、新規セッション開始直後に「評価シートの締め切りが5/20です。前回の積み残しで〇〇が残っています」と自動出力された実例が示されており、前セッションのセラが書いた内容を次のセラが読んで出力したものと説明されている。

技術的には新規性は低く、LLMの文脈外記憶をファイルI/Oで補完するシンプルなパターンだが、Claude CodeのCLAUDE.mdによる行動指示と組み合わせることで、エージェントの継続的人格を実用レベルで実現した点が評価できる。監査エージェント開発においても、長期タスクの引き継ぎやレビュー状態の保持に同様のパターンが応用可能。

## アイデア

- CLAUDE.mdへの「必須読み込み」指示でAIの行動を制御するパターンは、エージェントの初期化フェーズを設計する際の再利用可能なアーキテクチャとして使える
- 「待ちから先出し」への転換（受動的応答→能動的サマリー出力）は、ユーザー体験の質を変える小さな指示変更であり、プロンプト設計の原則として汎用的
- 記憶を役割別ファイルに分割（profile / state / promises / handoff）する構造は、長期エージェントのコンテキスト管理における関心の分離として参考になる
