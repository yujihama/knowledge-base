## 要約

本記事はAIエージェントを体系的に学ぶための用語集であり、基礎概念・構成要素・メモリシステム・計画推論・ツール使用の5章構成で100以上の用語を解説する。基礎層では、Agent Loop（観察→推論→行動→観察のサイクル）、BDI Agent（Belief-Desire-Intentionの3要素による意思決定モデル）、Policy（状態から行動を決定するルール）など、強化学習・古典AIの概念がLLMエージェントに接続される形で整理されている。構成要素層では、Perception Module・Memory Module・Planning Module・Action Moduleの4モジュール構造を提示し、それぞれの役割と連携を明確化。Tool Registry（利用可能ツールの定義を一元管理するカタログ）やScratchpad（推論中の作業メモ領域）といった実装レベルの概念も含む。メモリシステムは認知科学の分類（Working Memory・Episodic Memory・Semantic Memory・Procedural Memory）をLLM文脈に対応付けており、In-context StorageとExternal Storage（ベクトルストア・DB）の対比、Memory Consolidation（短期→長期記憶への定着処理）、Memory Compression（要約・重複排除によるウィンドウ対策）まで網羅する。Knowledge Graphをエンティティ＋関係の有向グラフとして記憶基盤に使う設計も紹介。計画推論層では、ReAct（推論と行動の交互ループ）・Plan-and-Execute・Hierarchical Planning・Dynamic Planningの違いを整理し、MCTS（選択・展開・シミュレーション・バックプロパゲーションの4フェーズ）・Beam Search PlanningなどのLLM推論改善手法も記載。Task Decomposition→Task Graph→Dependency Resolution→Subgoalという計画生成の流れが体系的に把握できる。ツール使用層では、Function Calling（OpenAI・Anthropicが標準提供）・Tool Schema（ツールのパラメータ定義）・Tool Selection・Tool Registrationの関係が整理されており、エージェント設計時のツール統合の全体像を把握できる。監査エージェント開発への示唆として、LangGraphで構築するReActループの各ノードがAgent LoopのObservation→Reasoning→Action構造に直接対応すること、Procedural MemoryとしてSOPをシステムプロンプトに定義するアプローチ、Function CallingによるAuditツール（DB照会・ログ検索・レポート生成）の統合設計、Episodic Memoryへの過去監査事例の蓄積による判断精度向上が特に応用可能な概念として挙げられる。

## アイデア

- BDI Agent（Belief-Desire-Intention）モデルは、監査エージェントの意思決定設計に直接適用可能：Beliefは現在の財務データ・ログの状態、Desireは監査目標（異常検知・コンプライアンス確認）、Intentionは現在実行中の監査手順として定義できる
- Procedural MemoryをSOPとしてシステムプロンプトに実装するアプローチは、監査手順書をエージェントの行動規範として形式化する際の設計パターンとして活用できる
- Memory Consolidation（短期→長期記憶への定着）とMemory Compression（要約・重複排除）の組み合わせは、長期監査セッションでコンテキストウィンドウを超えるログ・証跡を効率的に管理するための実装指針になる
