## 要約

本稿はAIエージェントサービスのUI設計における課題を、「怠惰なエンジニア視点」という逆張りの切り口で論じたオピニオン記事である。著者はAgent UIを「指示UI（人間からAIへのコンテキスト受け渡し）」と「編集UI（AIの成果物の確認・修正）」の2フェーズに分類する。前者はチャット入力が主体で、後者はWordのような従来UIが残存するとし、両者は見た目の変化は小さいが、要求される機能とコンテキスト設計は根本的に異なると指摘する。

著者が提示する3つの具体的要求は以下の通り。

①「パーソナライズされたコンテキストのサジェスト」：ユーザが何もしなくてもエージェントが関連情報（メール、会議履歴、仕事の流れ等）を先回りして収集・提示し、最終的な意思決定のみ人間に求めるべきとする。これを実現するにはAgentが社内情報含め多様なリソースにアクセスできるプラットフォーム整備が必要条件となる。企業のAI導入が進まない原因としてAgentのデータアクセス制限を挙げる点は、IT戦略上の重要な視点である。

②「入力コストの最小化」：コンテキスト入力量はタスク複雑化とともに増大しており、テキスト打鍵が主なインタラクション手段のままでは限界があると主張する。Coding Agentのファイルサジェスト機能やChatGPTの部分選択LLM呼び出しを「改善の萌芽」として評価しつつも不十分とし、「必要なコンテキスト項目をAIが全て列挙し入力フォームまで生成する」UIを理想とする。OpenAIがJony Iveと共同開発中の新デバイスも引用し、Speech-to-SpeechのマルチモーダルモデルによるUI革新へ言及する。

③「エージェントの接続先・権限の可視化」：AIが誰のメールを誰に送れるか、どのファイルを削除できるかが不明瞭なため信頼して使えないという問題提起。MCP（Model Context Protocol）やSkillの企業内蓄積が進む中、接続先ツールと許可操作の明示がUIレベルで必要と述べる。

余談として著者自身の書籍執筆体験を引き、AIは「正しいもの」は書けるが「自分が書きたいもの」は書けないと述べ、編集UIが残存する理由を「自己表現」という概念で説明している。AI時代における人間の役割を、欲求の保持・意図の言語化・AIとの協調による実現と定義する点は、エージェント設計における人間-AIインターフェースの哲学的枠組みとして参照価値がある。

## アイデア

- AgentのUIを「指示UI」と「編集UI」の2フェーズに分類し、それぞれで必要なコンテキスト設計が根本的に異なるという整理は、エージェントシステムの設計指針として応用しやすい
- エージェントのデータアクセス範囲（社内システム連携・MCP接続先）がユーザの使用可否判断に直結するという指摘は、監査エージェント設計においてツール権限とアクセス制御の可視化UIを検討する際に直接応用できる
- 「AIが先にコンテキストをサジェストしユーザは選択するだけ」というインタラクションモデルは、ReActループの入力フェーズを人間に最適化した設計パターンとして、エージェントのオンボーディングUI設計に示唆を与える
