## 要約

本記事はThe Gradient誌（2024年9月）に掲載された論考で、LLMベースのチャットボットが「目的志向の対話（Purposeful Dialogue）」を欠いているという問題を指摘する。MMLU・HumanEval・MATHといった既存ベンチマークはスコアの飽和が進む一方、実際のユーザー体験との乖離が拡大している。

目的志向の対話とは、旅行計画・心理療法・カスタマーサービスのような具体的なゴールを中心とした複数ターンの会話を指す。交渉理論における「反復的なバーゲニング」が一度きりのオファーより優れた結果をもたらすように、情報を段階的に交換する対話形式は、単一パス生成よりも複雑なタスクに適している。コード生成の文脈では、SWE-benchのようなGitHub Issue解決タスクが典型例であり、AIが人間のエンジニアと往復のコミュニケーションを行いながら要件確認・文書収集・支援要請を行う必要がある。

対話システムの変遷として、1970年代のSchankの「レストランスクリプト」、ELIZA、PARRYなどのルールベースシステムから、現代のLLMへの転換が整理される。現代のLLMは(1)大規模コーパスでの事前学習、(2)tokenizer.apply_chat_templateによるチャットフォーマット導入、(3)RLHFによる微調整、という3段階で対話能力を獲得するが、Yann LeCunの「ケーキのチェリー」比喩が示すようにRLHFはごく小規模な調整に過ぎない。

一貫性評価の実験として、著者らは2つのシステムプロンプト付きLMエージェントを相互に対話させ、最大8ターンの会話を生成するフレームワークを構築した。任意のターン（例：5ターン目）でコントロールプロンプトを注入し、ロール維持能力をLM-as-judgeで評価する手法を採用。結果、GPT-4を含む現行モデルは対話が長くなるにつれてシステムプロンプトへの追従が劣化し、ロールを完全に離脱するケースが確認された。

解決策として、(1)プロンプト工学：対話履歴の圧縮・要約・選択的除去、(2)長コンテキストファインチューニング：LongAlpacaやLongLoRA等、(3)外部メモリ機構：Memorybank、A-MEM、MemGPTなどのアーキテクチャ的手法が挙げられる。さらに対話の評価指標として、単なる応答品質を超えた「一貫性」「目的達成度」「適応性」を測る新しいベンチマーク設計の必要性を論じる。

監査エージェント開発への示唆：監査手続きは本質的に複数ターンの対話（証拠収集→仮説形成→追加質問→判断）で構成される。現行LLMのロール劣化問題はLangGraphベースの監査エージェントにも直接影響し、システムプロンプトの圧縮戦略やMemGPT型の外部メモリ設計が長期的な監査セッションの品質維持に不可欠となる。

## アイデア

- 2つのLMエージェントを相互対話させて任意ターンにコントロールプロンプトを注入することで、ロール一貫性をストレステストする評価フレームワークの設計
- 交渉理論の「反復的バーゲニング優位性」をAI対話設計に適用するアナロジー：情報の段階的開示が単一パス生成より複雑タスクに適する理論的根拠
- 長期対話におけるコンテキスト管理の3層アプローチ（プロンプト工学・長コンテキストFT・外部メモリ）の組み合わせが監査エージェントのセッション維持に応用可能
