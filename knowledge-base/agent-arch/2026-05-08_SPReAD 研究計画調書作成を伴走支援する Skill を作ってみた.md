## 要約

文部科学省が2026年度に開始した「AI for Science 萌芽的挑戦研究創出事業（SPReAD）」は、1課題あたり直接経費500万円以下・研究期間は交付決定日から2027年1月6日までという条件で、研究者のAI利活用を支援する事業。第1回公募期間は2026年4月17日〜5月18日正午。本記事では、この公募への応募を支援するAIスキル「spread-proposal-assistant」の設計と機能を解説する。

スキルの仕組みは、SKILL.md（指示書）と参照ドキュメント・スクリプト・テンプレートをまとめたフォルダをzipで配布する「Skills」形式。Kiro / Claude Code / Codex といった複数のAIエージェントプラットフォームで同一スキルが動作する点が特徴で、プラットフォーム間の移植性を確保している。

スキルの機能は4つ。①公募要領(PDF)と様式1（xlsx）をcall_materials/配下に同梱し、エージェント起動時から参照可能にすることで、ユーザーが毎回資料を提供する手間を省く。②研究計画調書の4シート（基本情報・研究目的/方法/妥当性・研究経費内訳・API/計算資源費用算定根拠）を対話形式でヒアリングしながら順次埋める。③AWS Price List JSONを参照してEC2・SageMaker・S3等のサービス単価を取得し、算定根拠文案を作成して様式4枚目の表形式テンプレに整形・埋め込む。④最終的に様式フォーマットに準拠したxlsxファイルを生成し、内部検証を実施する。

スキルのディレクトリ構成はSKILL.md・references/（シート別記入ガイド・選択肢マスタ）・scripts/（AWS料金試算・xlsx書き込み・検証スクリプト）・call_materials/（公募要領・様式xlsx等）からなる。GitHubでOSSとして公開されており（AWS公式プロダクトではなく個人開発）、~/.kiro/skills/や~/.codex/skills/に配置するか、Claude.aiにzipアップロードすることで利用できる。

制約として、AWS料金試算の精度は保証されず、生成AIの性質上ハルシネーション（存在しない文献・計算ミス・公募要領の解釈誤り）のリスクがある。また公募要領が改訂された場合は最新版との差分確認が必要。

監査エージェント開発への示唆：SKILL.md＋参照ドキュメント＋スクリプトをセットにした「タスク特化型Skillパッケージ」の設計は、監査チェックリスト作成支援や調書フォーマット自動生成にそのまま転用できる。複数エージェントプラットフォームで動作する移植性の高い設計は、企業内の多様なAIツール環境への対応という観点でも参考になる。

## アイデア

- SKILL.md＋参照ドキュメント＋スクリプトをzipで配布する『タスク特化型Skillパッケージ』の設計が、Kiro/Claude Code/Codexといった異なるエージェントプラットフォームで同一動作を実現する移植性の高いアーキテクチャになっている
- AWS Price List JSONをエージェントがリアルタイムで参照して料金試算を行い、その結果を申請書様式の表形式テンプレに直接整形・埋め込むという、外部APIデータ取得→構造化フォーマット変換のパイプライン設計
- 公募要領PDFと申請書xlsxテンプレートをcall_materials/としてスキル内に同梱することで、ユーザーのコンテキスト提供コストをゼロにするRAG的アプローチ（エージェント起動時から資料参照可能）
