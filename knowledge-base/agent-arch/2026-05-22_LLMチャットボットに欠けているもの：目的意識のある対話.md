## 要約

現在のLLMベースのチャットボットはMMIU、HumanEval、MATHなどのベンチマークでは改善が続いているが、インタラクティブな対話品質との乖離が問題視されている。本記事は「目的意識のある対話（Purposeful Dialogue）」という概念を提唱し、現行システムの限界と評価手法の不足を論じる。

目的意識のある対話とは、「旅行計画エージェント」「カスタマーサービスbot」のような具体的なゴールを持つ複数ターン対話を指す。交渉理論の反復的バーゲニングモデルと同様に、情報を1回で全て渡すよりも複数回のやりとりを通じて重要な情報のみを選択的にやりとりする方が効率的とされる。コード生成においても、SWE-benchのようなGitHub Issue解決タスクは単一アクションでは達成困難で、ペアプログラミング的な対話が必要となる。

対話システムの歴史的背景として、1970年代のRoger Schankの「レストランスクリプト」、ELIZA、PARRYから始まり、現代のLLMは(1)事前学習（次トークン予測）→(2)チャットテンプレートによる対話フォーマット導入（`<system>`や`<INST>`トークン）→(3)RLHFによる微調整という3段階で構築される。RLHFはLeCunの「ケーキのチェリー」に喩えられるほど学習データ規模は小さい。

現行システムの一貫性の問題として、著者らの実験ではシステムプロンプトを与えた2つのLMエージェントを互いに対話させ、途中でユーザーの割り込み発話（Perturbation）を挟む手法で長期対話における指示追従能力を評価した。具体的には対話の幹[a1,b1,...,a8,b8]に対し、異なるタイムステップでPerturbationを挿入し、その後の対話への影響を測定する。結果として、対話が長くなるにつれてモデルが役割から外れる傾向が確認された。

評価の困難さとして、インタラクティブ対話では相手の発話を事前に予測しにくく、MT-BenchやAlpaca-Evalのような1回完結型ベンチマークでは捉えられない問題がある。LLM-as-judgeを用いた評価でも対話全体の目的達成度を測る枠組みが欠如している。また、報酬の遅延問題（単発ターンの品質と最終ゴール達成の乖離）、対話エージェントの探索戦略（知識がない場合に質問を生成するか推測するか）、長期的な関係構築におけるユーザープロファイルの更新なども未解決課題として挙げられる。

監査エージェント開発への示唆として、ReActやLangGraphベースの監査エージェントも複数ターンの情報収集（証憑の追加要求、不整合箇所の確認）が本質的に必要であり、システムプロンプトによる役割維持の脆弱性は実務環境での信頼性リスクとなる。目的意識のある対話の評価フレームワークを監査プロセスのシミュレーション評価に応用できる可能性がある。

## アイデア

- 2つのLMエージェントを対話させてPerturbation（割り込み発話）を挿入することで、長期対話における指示追従の崩壊を定量評価する実験設計は、監査エージェントのロール維持テストに直接応用できる
- Terry Winogradの「発話は相手の世界モデルを変化させる意図的な行為」という枠組みでLLM対話を捉えると、各ターンの情報価値と戦略的設計が重要になり、RAGの検索タイミング設計にも影響する
- 交渉理論の反復バーゲニングモデルを対話設計に適用する視点は、監査における証拠収集プロセス（一度に全情報を要求せず段階的に絞り込む）のエージェント化に有用な理論的基盤を提供する
