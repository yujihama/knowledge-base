## 要約

本論文はThe Gradientに掲載された哲学的・技術的論考で、AIアライメントの主流的前提である「AIは目標（goal）を持つべき」という設計思想に根本的な疑問を呈する。著者は、人間の合理的行動は最終目標への最適化ではなく「実践（practice）」——行動・行動傾向・評価基準・行動リソースが自己強化するネットワーク——に従っていると主張する。

核心概念は「エウダイモニア的合理性（eudaimonic rationality）」であり、アリストテレスの幸福論（eudaimonia＝能動的・合理的な人間の繁栄）から着想を得ている。この合理性モデルでは、手段と目的、道具的価値と本質的価値の区別が原則として存在しない。音楽における一音符がメロディの一部であるように、個々の行為は実践全体の構成要素である。数学者テリー・タオの研究実践を例に、「数学的卓越を数学的に卓越な方法で促進する」という構造（promote x x-ingly）が自己組織的かつ安定的であることを論証する。

EA（Effective Altruism）型の功利最大化AIとの対比が論の中心をなす。EA型最適化器は人間の価値を効用関数に変換しようとするが、eudaimonic合理性との「型の不一致（type mismatch）」により、透明性・有益性・無害性・修正可能性（corrigibility）といった安全特性さえ脆弱で恣意的なものになると指摘する。一方、eudaimonic型エージェントにとってこれらは自然な実践内のダイナミクスとして解釈可能であり、安定性が高い。

著者はeudaimonic合理性の「自然性（naturalness）」を技術的文脈で定義する：学習可能性、アルゴリズム複雑度の低さ、文化進化における収束性、MLトレーニングによるターゲット可能性などを指標とする。この自然性があるならば、人間と神経網ベースAIの間で収束的進化が起こりえると論じる。

監査エージェント開発への示唆として重要な点は、corrigibility（修正可能性）を目標ベースの制約として実装しようとする従来アプローチへの批判である。修正可能性を「実践の中の自然なダイナミクス」として設計することで、より堅牢な監査AIエージェントの安全設計が可能になりうる。LangGraphやReActベースのエージェントにおいて、ルールベースの制約よりも実践ベースの行動傾向を組み込む設計方針への理論的根拠を提供する論考である。

## アイデア

- 「promote x x-ingly」という公式——誠実さを誠実に促進する、数学的卓越を数学的に促進する——が、自己組織的・安定的な実践を生む構造的条件であり、LLMエージェントの安全設計原則として応用可能
- corrigibilityや無害性を「目標制約」ではなく「実践内のダイナミクス」として定義することで、エージェントの行動が内発的に安全になるアーキテクチャ設計のアプローチ
- eudaimonic合理性の「自然性」をMLトレーニングのターゲット可能性で評価する枠組みは、RLHF/RLAIFにおける報酬モデル設計の理論的根拠として展開できる
