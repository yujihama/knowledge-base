## 要約

Nous ResearchがオープンソースとしてリリースしたHermes Agentは、セッションをまたいで自律的に学習・成長し続けるAIエージェントフレームワークである。全データをローカル（~/.hermes/配下）に保存し、クラウドへのデータ流出ゼロを実現している点が最大の特徴。

記憶システムは3層構造で設計されている。第1層「Prompt Memory（ホットメモリ）」は~/.hermes/memories/配下にMEMORY.md（上限約2,200文字）とUSER.md（上限約1,375文字）の2ファイルをMarkdown形式で保持し、セッション開始時に毎回システムプロンプトへ注入する。容量上限に達するとエージェントが自律的に古いエントリを統合・削除して肥大化を防ぐ。書き込みはイベント駆動（ユーザーの明示的指示、定期振り返りプロンプト、環境変化の検知）で行われるため、読み込みコストは低く保たれる。

第2層「Session Search（検索メモリ）」は全会話履歴をローカルのSQLite（~/.hermes/state.db）に保存し、SQLiteの全文検索エンジンFTS5とLLMによる要約を組み合わせて検索する。ベクトルDBやEmbeddingモデルを使わず、外部依存ゼロ・セットアップ不要のFTS5を選択した設計判断が特徴的。各ターン終了時にリアルタイムで書き込み、検索はオンデマンドのみ実行するため、第1層と異なりトークンコストが発生しない。

第3層「Skills（手続き記憶）」は~/.hermes/skills/配下にMarkdownファイルとして手順書を保存する。複雑なタスクの成功時にエージェントが自律的にSkillファイルを生成・更新し、失敗時の知見（例：「タイムアウトをデフォルト3秒→30秒に変更が必要」）も蓄積する自己改善ループを実現する。

この3層は「定期的な振り返りプロンプト」によって連携する。一定インターバルで「今の作業から永続化すべき知識はあるか？」という自己評価プロンプトをエージェントが受け取り、価値ある情報をMEMORY.md・USER.md・Skillsへ自動的にフィードバックする仕組みだ。ユーザーの明示的な指示なしに学習が進む点が監査エージェント開発においても参考になる。外部プロバイダーとしてMem0、Hindsight（ナレッジグラフ）、Honcho、Supermemoryとのプラグイン統合にも対応し、大規模知識管理やチーム横断利用に拡張できる。

## アイデア

- ベクトルDB（ChromaDB等）ではなくSQLite FTS5を選択した設計判断：外部依存ゼロ・Embeddingモデル不要・個人用途には十分な検索精度という割り切りは、監査エージェントのような内部情報を扱うシステムでも参考になる
- MEMORY.mdとUSER.mdに文字数上限（2,200字・1,375字）を設けてエージェント自身に圧縮・整理を委ねる設計：毎セッションのトークンコストを一定に保ちつつ、情報の鮮度管理を自動化する手法
- 定期振り返りプロンプト（Periodic Nudge）による自律学習：ユーザーが明示的に「覚えて」と指示しなくても、エージェントが自発的にMEMORY.md・Skills更新を判断するイベント駆動型の自己改善ループ設計
