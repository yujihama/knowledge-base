## 要約

multi-agent-shogunは、tmuxとClaude Codeを組み合わせた並列マルチエージェント開発基盤で、「将軍→家老→足軽×8＋軍師」の4層構造を持つOSS（GitHub ★1.3k超、Fork 275）。人間（上様）が将軍に命令すると、家老がタスクを分割して足軽に割り当て、並列実行が始まる戦国時代の軍制をモチーフにしたシステムである。著者は毎朝の起動時に戦国風バナーと「出陣じゃーーー！」という掛け声に脳内法螺貝が響くことに気づき、朝の心理的負担を軽減するために世界観をメイドお屋敷に丸ごと書き換えた。改造のスコープは完全にUIとテキストのみで、ロジックには一切手を入れていない。具体的な変更点は①役割名の置換（将軍→メイド長、家老→チーフメイド、足軽→メイド）、②tmuxセッション名・ペイン名の変更（shogun/multiagent→head-maid/multi-maids、ashigaru{N}→maid{N}）、③起動スクリプトのリネームと中身の書き換え（shutsujin_departure.sh→oshigoto.sh、掛け声・バナーASCIIアート・絵文字の全置換）、④CLAUDE.md・instructions/*.md・queue/配下のファイル名と中身の置換の4点。改造の要点は「戦国成分はロジックではなく語彙とテキストに偏在している」という発見であり、CLAUDE.md・指示書・キューファイル・起動スクリプト・設定ファイルを上から順に語彙置換するだけで世界観が完全に移植できる。結果として起動掛け声が「かしこまりました！お仕事開始です！」に変わり、毎朝の心拍数が下がったという。この手法はメイド以外にも海賊・宇宙船クルーなど任意のモチーフへの転用が可能。監査エージェント開発への直接的な示唆は薄いが、マルチエージェントシステムにおける「ロールとモチーフの分離」という設計観点は、エージェントの役割定義を柔軟にカスタマイズする際の参考になる。

## アイデア

- マルチエージェントシステムの「世界観（モチーフ）」はロジックと完全に分離可能であり、CLAUDE.md・指示書・スクリプトのテキスト層だけを差し替えることで任意のキャラクター・文化的文脈に移植できる
- エージェントの口調・役割名・掛け声といったUI文字列が開発者の心理状態（朝の気分・モチベーション）に与える影響は無視できず、UXとして意図的に設計する価値がある
- tmuxのセッション名・ペイン名もエージェント間通信のファイル名と連動しているため、リネームは単なる表示変更ではなく通信路の命名規約変更を伴う──この依存関係の把握が改造の技術的ポイント
