## 要約

MIT Technology Reviewの「今AIで重要な10のこと」シリーズの一編として、マルチエージェントシステムのオーケストレーション技術の現状と課題を概説した記事。

LLMは ChatGPT以降、会話AIとして普及したが、「世界を変える」には単なる対話を超えて実際のタスクを実行する必要がある。その担い手がAIエージェントであり、現在は単独エージェントからマルチエージェント協調へと進化している。

具体的なツールとして、Anthropic の Claude Code はコーディングエージェントを複数並列起動・調整でき、ユーザーによっては同時に数十のサブエージェントを稼働させた事例が報告されている。役割分担も可能で、コード生成・テスト・バグ修正をそれぞれ別エージェントが担う。Anthropic はさらに Claude Cowork（デスクトップ向け汎用マルチエージェント生産性ツール）を Claude Code 自体を使って10日間で開発したと主張しており、通常数ヶ月かかる開発期間を大幅に短縮した事例として紹介されている。OpenAI の Codex、Perplexity の Computer も同様の汎用ホワイトカラー向けワークフロー自動化ツールとして列挙されている。研究領域では Google DeepMind の Co-Scientist がチーム型エージェントにより文献調査・仮説生成・実験設計を協調実行する。

これらのマルチエージェントシステムは「新たな組み立てライン」に例えられており、ヘンリー・フォードの製造革命がホワイトカラー知識労働に起きる可能性が論じられている。コーダーはプロジェクトマネージャー的役割に移行し、より多くのタスクを委任・監督できるようになる。

一方でリスクも強調されている。LLMはそもそも予測困難（unpredictable）であり、画面内に閉じている間は「煩わしさ」で済むが、医療・金融・ソーシャルメディア・防衛インフラなどの実世界デジタルインフラと連携し始めると、誤動作が壊滅的な結果を招きうる。OpenClaw（記事中の架空/匿名化ツール名とみられる）はセキュリティ上の問題が指摘されつつも注目を集め、Nvidia・Tencent 等がそのオープンソースコードを基に安全性・信頼性を改善したボットを構築している。

監査エージェント開発への示唆：マルチエージェントの役割分担モデル（生成・検証・修正の分離）は監査ワークフローの自動化設計に直接応用可能。ただし、金融・内部統制領域では LLM の予測不確実性が規制・信頼リスクに直結するため、ガードレール設計とヒューマン・イン・ザ・ループの組み込みが不可欠。

## アイデア

- コーディングエージェントの並列化（数十サブエージェント同時稼働）は、監査手続の並列自動化（証拠収集・検証・レポート生成の同時進行）に直接転用できる設計パターンである
- Claude Cowork を Claude Code 自体で10日間開発したという事例は、エージェントによるソフトウェア開発の自己参照的加速（ブートストラップ）を示しており、開発速度のオーダー変化を示す実証例として注目できる
- マルチエージェントシステムを「組み立てライン」に例えるフレームは、タスク分解・専門化・調整コストのトレードオフ分析に有用で、エージェント設計時のアーキテクチャ評価軸として使える
