## 要約

本記事は、AnthropicのClaude CodeおよびOpenAIのCodexが「エージェント」と称されることへの語義的・構造的批判である。論点の中心は、民法における代理概念（民法第99条）との対比にある。本来のエージェントとは、プリンシパルから権限を委任され、その範囲内で裁量を持って自律的に行為し、プリンシパルを拘束しうる存在である。この基準に照らすと、両製品は「エージェント」とは言えないと著者は主張する。

Claude Codeの実態として、デフォルト動作はBashコマンド実行・ファイル変更前の承認要求であり、Anthropic自身の2026年3月の分析では承認プロンプトの93%がユーザーに通過されていると報告されている。これは「自律性が高いから承認不要」ではなく、「承認疲れが問題で自動判定で補完したい」という設計思想であり、自律代理人ではなく「承認疲れ対策ツール」である。また同社は、git remoteブランチの誤削除、GitHub認証トークンの内部環境へのアップロード、本番DBへのマイグレーション試行といった内部エージェント誤動作事例を公開しており、これらを「model being overeager（モデルが過剰に積極的）」と説明している。すなわち自由にすると危険なため制御しているという自認である。

Codexについては、デフォルトでネットワークアクセスがオフ、OSレベルのサンドボックスで作業ディレクトリのみにアクセス制限、承認ポリシーで「いつ止まって聞くか」を制御する構造であり、エージェントループの各ターン末尾で「control returns to the user」と明記されている。OpenAI自身も運用文脈では「Humans steer. Agents execute.」と記述しており、コードの真の所有権はエンジニアに残ると説明している。宣伝文と運用文の「二重帳簿」がCodexの実態を示している。

ユーザー行動の分析においても、経験の浅いユーザーは完全自動承認を約20%しか使わないのに対し、経験を積むと40%超まで増えるが、同時に割り込み率も5%前後から9%前後へ上昇する。つまり慣れるほど「全部任せて放置」ではなく「大まかには流しつつ必要時に鋭く介入する監督型」に変化する。

著者の結論は、両製品の価値は「代理」ではなく「圧縮」にある、というものだ。探索・編集・テスト・下書きの工程を圧縮し認知負荷を下げることが価値であり、人間の責任と判断を置換することが価値ではない。「エージェント」という名称は市場向けポジショニング（売れる語）であり、業務代替の物語として資本市場（2026年2月のSaaS株急落）を動かした誇大表示であると指摘する。真にエージェントを名乗る条件として、広範なリファクタリングを既存仕様保存まで含めて安定遂行でき、重要局面以外で逐一承認を要せず、責任帰属が人間レビューに依存しないことを挙げ、現状の両製品はその手前にあると結論づける。監査エージェント開発においても、「承認・サンドボックス・差分レビューを前提とした限定実行器」として設計する現実的なアーキテクチャ観が求められる示唆を含む。

## アイデア

- 「承認プロンプトの93%を通過させている」というAnthropicの自社分析データは、エージェントの自律性議論を定量的に反転させる強力な根拠であり、承認疲れ問題としてフレーミングする視点が監査AIの人間介入設計に直接応用できる
- ヘビーユーザーほど自動承認率が上がりつつ割り込み率も上がる（20%→40%、5%→9%）という逆説的データは、信頼の増加が監督の減少ではなく監督の質的変化（粗→精）を意味することを示しており、LLM-as-judgeの設計原則に通じる
- 「檻があるから知能が実用品になっている」という逆転命題は、エージェントアーキテクチャ設計において環境制御・権限制御・停止点の設計がモデル能力と同等以上に重要であることを示唆し、LangGraphのノード設計やReActループの割り込み設計に直接的な示唆を与える
