## 要約

非ITの公務員が、Claude Codeの6つの拡張機能（Hooks、Skills、MCP、Subagent、Memory、CronCreate）にLLMルーティングを加えた7つの仕組みを組み合わせ、月額$180で20以上のプロジェクトを同時開発・運用する環境を構築した実践事例。

最も経済的インパクトが大きいのはLLMルーティング。通常Opusのみで月$1,000以上かかるところを、GLM-5.1（85%）→MiniMax（14%）→Sonnet（1%未満）という3段階ルーティングで82%削減し$180/月を実現。高性能モデルは「難しい判断が必要な時だけ」に限定する。

Hooksは5種類を設定。PreToolUseで`rm -rf *`等19種類の危険コマンドをブロック、PostToolUseで作業内容をCSV自動記録、SessionStartで前回セッションの引き継ぎ情報を表示、StopでDiscord通知を送信する。

Memoryは3種類（ユーザー情報・フィードバック・プロジェクト文脈）を永続化。「featureブランチ禁止」「settings.json変更時はSSOTも更新」といった過去の指示方針や、各プロジェクトのバージョン・フェーズ情報をセッション間で引き継ぐ。

Subagentで最大4つのタスクを並行実行。コードレビュー・テスト実行・バグ調査を同時依頼することで、逐次実行の場合の4時間を1時間に短縮できる。

CronCreateで夜間・外出中も自動実行。5分ごとのドキュメント自動コミット、毎時の自律開発ループ、毎日0時の日記テンプレート生成、週3回のナレッジリント（情報整合性チェック）が稼働している。

MCPでBrave Search、GitHub、Playwright、Discord、Context7、Exa、GLM/MiniMaxなど15ツールを統合。Skillsでは「TDD」「体系的デバッグ」「コードレビュー」「プラン執行」の手順書をテンプレート化して再利用する。

監査エージェント開発への示唆：Hooksによる操作ブロック・ログ記録はエージェントのガバナンス制御に直接応用可能。CronCreateによる定期タスク＋Memoryによる文脈引き継ぎは、長期間稼働する監査エージェントの状態管理パターンとして参考になる。LLMルーティングはコスト制約の厳しい内部監査用途でも有効な設計思想。

## アイデア

- LLMルーティングで安価モデル（GLM-5.1）を85%使用し高性能モデル（Sonnet）を1%未満に抑える3段階コスト最適化戦略
- PreToolUseフックで危険コマンドを事前ブロックするエージェントガバナンス機構——LangGraphのguardrailノードと同等の制御をClaude Codeレイヤーで実現
- CronCreate＋Memory＋Subagentの組み合わせによる「自律開発ループ」——人間不在でもセッション間文脈を維持しながら並行タスクを継続実行する非同期エージェント設計
