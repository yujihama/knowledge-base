## 要約

Komlock labのエンジニアが2023〜2026年にかけてAIコーディングツールを乗り換えてきた経験を振り返った実践的なレポート。ツールの変遷はそのままAIの能力拡張の歴史と重なる。

最初はGitHub Copilotによる文脈を読んだコード補完に衝撃を受けた段階（2023〜2024年）。VSCode + Copilot + ChatGPT + GeminiがあればCursorやClineは不要という感覚だった。

転機はDevinの登場。複数リポジトリを横断して自律的にPRを出し、レビューまでこなす「実行するAI」という概念が初めてリアルになった。続いてMCPの台頭により、GitHub MCPやPlaywright MCPを活用してAIが外部ツールと連携しながら作業できるようになった。

Claude Codeの登場でIDE非依存のターミナルベース作業に移行。プロジェクトコンテキストをまるごと渡してタスクを委任できる感覚に加え、Skills・Hooksの仕組みによりMarkdownファイルへの指示記述だけでエージェントの動作をカスタマイズ可能になった。これによりMCPへの依存度が下がった。

OpenAI Codexも試したが、出力が硬く理解しづらい場面が多く、主に実装レビュー用に落ち着いた。

2026年2月にOpenClawを導入。Discord・Telegram・Slackなどのメッセージングプラットフォーム上でLLMベースのパーソナルAIエージェントを常時起動させるフレームワークで、cronによる定期タスク実行やヘルスチェック・情報収集の自動化が可能。「頼んだら動く」から「常時起動して自分から動く」への転換点となった。

現在の構成はOpenClaw + Claude Codeをメインに、コード確認にCursor、レビューにCodexを使い分ける体制。

全体の流れを整理すると「補完→指示実行→ツール連携→常駐→経済活動」という能力拡張の段階が見える。次のステップとして、エージェントが自律的にサービスへ課金し、エージェント間で仕事を依頼・受注し、収益を得る「エージェント経済」の到来を展望している。監査エージェント開発の観点では、Claude Code + Skills/Hooksによる作業自動化パターンや、OpenClawのような常駐型フレームワークが監査ワークフローの自律実行基盤として参考になる。

## アイデア

- Claude CodeのSkills/Hooksにより、Markdownで指示を記述するだけでエージェント動作をカスタマイズでき、MCPへの依存を大幅に削減できるという設計思想
- OpenClawのような「常時起動・cron駆動」エージェントフレームワークが、単発タスク実行型ツールとは質的に異なる「AIが一緒に働く」感覚をもたらす点
- AIツールの進化段階（補完→実行→外部連携→常駐→経済活動）を明示的にモデル化することで、次の技術トレンドを予測する視点
