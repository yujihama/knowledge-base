## 要約

本記事は、AIコーディングエージェントの出力をProduction環境に安全に反映するための「Verdict Layer」アーキテクチャを提唱する。背景として、AIがコード生成速度を飛躍的に向上させる一方、シニアエンジニアによるレビュー速度はボトルネックのままであり（アムダールの法則）、より高価なフロンティアモデルへの投資で解決しようとする業界の傾向を「誤ったアプローチ」と断じている。

著者は228回のエンタープライズ級コードベースへのパッチ適用サイクルを検証し、以下の定量的データを示した。①合計806回のLLM呼び出しのうち95.9%（773回）はdeepseek-v4-flashなどの安価なモデルで処理され、フロンティアモデルが必要だったのは4.1%（33回）のみ。②パッチ受容率（Accept Rate）は93.6%（206回承認、14回拒否）。③フラッシュモデルのワーカー失敗率は5.1%だったが、クリティカルな欠陥のProduction到達件数はゼロ。

アーキテクチャの核心は「Agent Capture」という概念で、「エージェントが提案し、境界が決定する（The agent proposes. The boundary decides.）」という原則に基づく。AIモデルにはコード生成権限のみを与え、実行・反映権限は決定論的（Deterministic）なソフトウェアレイヤー（Verdict Layer）が担う。この境界は、コミット前の静的解析・スコープ検証・Diff検査・テスト再実行で構成され、プロンプトインジェクションやハルシネーションの影響を受けない。

実行フローは「仮説立案→独立した検証→隔離ワーカーによるタスク実行→パッチ生成→テスト実行→人間による意味的レビュー→最終決定（マージ）」で構成される。フロンティアモデルのAPI費用は安価モデルの約50倍であり、モデルへの投資は単一タスクの品質向上に留まるが、堅牢な「境界」への投資は以降の全タスクを永続的に安全にするという主張は、監査エージェント設計においても直接応用可能。LangGraphベースの監査エージェントにおいて、エージェントの提案（調査結果・是正勧告）と実際のシステム状態変更の間に決定論的な検証レイヤーを挟む設計パターンとして参照価値が高い。アーキテクチャの青写真はGitHub上でOSSとして公開されている。

## アイデア

- 「提案と実行の分離」という原則：AIに生成権限のみ与え、実行権限は決定論的レイヤーに委ねることで、モデルの信頼性に依存しない安全性を実現する点が設計上の核心
- 95.9%のワークロードが安価なモデルで処理可能という定量データ：高精度な境界設計があれば、フロンティアモデルへの依存度を大幅に下げられることを実証しており、コスト最適化の根拠として活用できる
- 監査エージェントへの応用：調査結果や是正勧告の「提案」と、実際のワークフロー起動や記録への「反映」の間にVerdictレイヤーを挟む設計は、LangGraphベースの内部監査エージェントの信頼性・説明責任確保に直結する
