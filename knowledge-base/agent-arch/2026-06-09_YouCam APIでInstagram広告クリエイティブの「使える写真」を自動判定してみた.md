## 要約

YouCam API（Perfect Corp社の画像AI API）とLLMを組み合わせて、Instagram広告・PR投稿向けのクリエイティブ事前チェックCLIをTypeScript/Node.js 20系で実装した記事。

システムの処理フローは以下の通り：①YouCam APIのFile APIで画像アップロードURLとfile_idを取得、②画像をアップロード、③AIタスク（背景除去・肌解析等）を開始してtask_idを取得、④task_statusがsuccessになるまでポーリング、⑤結果をLLMに渡してSNS広告観点のJSON/Markdownレビューを生成。

LLMが出力するクリエイティブ評価の項目は、スコア（0〜100）、そのまま使えるか、推奨用途（フィード/ストーリーズ/リールサムネ等）、良い点、改善点、YouCam APIで試すべき補正、Instagramキャプション下書き、広告表現の注意点、判断できない不明点の9項目。サンプル出力ではスコア82/100、背景除去済み商品画像に対して「背景変更で使用シーンに近い背景を試す」「ストーリーズ用に縦長比率の余白を作る」等の具体的な改善案が得られた。

設計思想の核心は「画像加工とクリエイティブ判断の役割分離」。YouCam APIは背景除去・照明補正・肌解析等の画像処理を担い、LLMはその結果をSNS広告実務の観点で構造化する。LLMのみでは画像処理はできず、YouCam APIのみでは「どの用途に使うか」「どんな表現を避けるか」まで判断できないため、両者を連結することで投稿前レビューの補助体験を実現している。

実装上の課題として、APIはPlaygroundでは同期的に見えるが実際は非同期でポーリングが必要な点、高解像度画像のサイズ制限、結果画像URLの有効期限管理、LLMが「売れそう」等の根拠不明な評価を出す問題があった。LLMへの制約として「広告成果・審査通過を保証しない」「美容効果を断定しない」「改善点は背景・照明・余白等の制作観点に限定する」をプロンプトに明示した。

監査エージェント開発への示唆：「ツール連携による役割分担」パターンが参考になる。画像処理APIが構造化データを返し、LLMがそれをビジネスルール観点で評価するアーキテクチャは、監査エージェントにおける「データ抽出ツール＋LLM判断」の構成と同型。また、LLMの評価に「断定禁止・人間レビュー前提」の制約を課す手法は、監査報告の適切な表現管理にも直接応用できる。

## アイデア

- 画像処理APIとLLMを役割分離して連結するアーキテクチャ：外部ツールが構造化データを返し、LLMがビジネス観点で評価するパターンは、エージェント設計の汎用的な手法として応用範囲が広い
- LLMの評価出力に「断定禁止・人間レビュー前提」を明示的に制約するプロンプト設計：AI出力の信頼性・コンプライアンスリスク管理に有効で、監査・広告・医療等の規制領域で特に重要
- task_idとポーリングによる非同期API処理パターン：画像生成・動画変換等の重い処理を持つAPIを安全に扱う実装パターンとして、長時間タスク管理の基本設計として参照価値がある
