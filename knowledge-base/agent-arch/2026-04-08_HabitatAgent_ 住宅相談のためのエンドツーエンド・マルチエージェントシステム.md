## 要約

HabitatAgentは、住宅選択という「高リスクかつ基本的に不可逆な意思決定」を支援するために設計された、LLMベースのマルチエージェントアーキテクチャである。既存の住宅プラットフォームやLLMアシスタントはランキングや推薦に処理を単純化しがちで、推論の不透明性・複数制約の処理の脆弱性・ファクチュアリティ保証の欠如という問題を抱えている。HabitatAgentはこれらを解決するため、4つの専門エージェントで構成される。

**Memory Agent** は多層的なユーザーメモリを管理し、制約抽出（例:「予算3000万、駅徒歩10分以内」）→メモリ融合（既存情報との統合）→検証ゲート付き更新という内部ステージを経て、ユーザー要求の変化を追跡する。**Retrieval Agent** はベクトル検索とグラフ検索を組み合わせたハイブリッド方式（GraphRAG）を採用し、物件データベースから関連情報を高精度に取得する。**Generation Agent** は証拠参照付きの推薦と説明を生成し、なぜその物件が推薦されるかの根拠を明示する。**Validation Agent** は多段階の検証と対象を絞った修正を適用し、ハルシネーションや制約違反を排除する。

評価は100件の実ユーザー相談シナリオ（300ペアのマルチターン会話）に対してエンドツーエンド正解率プロトコルで実施。強力なシングルステージベースライン（Dense+Rerank）が75%の精度を達成する一方、HabitatAgentは95%に到達し、20ポイントの改善を示した。この結果は、単一パイプラインでは処理しきれない複合制約の同時充足と、回答の監査可能性（auditability）を実現したことを示す。

## アイデア

- Validation Agentによる多段階検証と「targeted remediation（対象箇所の修正）」の分離は、生成内容の品質保証を後工程で担保するパターンとして、監査レポート生成などへ転用できる
- Memory Agentの『制約抽出→融合→検証ゲート付き更新』という3ステージ設計は、ユーザー要件が会話を通じて変化するシナリオ（監査スコープの変更など）でのステートフルな要件管理に直接応用可能
- GraphRAGとDense Retrievalのハイブリッド構成が、単体Dense+Rerankに対して20ポイントの精度向上をもたらした点は、グラフ構造を持つ監査基準や規制文書の検索精度向上のヒントになる

## Yujiの取り組みへの示唆

HabitatAgentのアーキテクチャは、Yujiが開発する監査エージェントシステムと構造的に類似している。特にValidation Agentによる多段階検証と修正のパターンは、監査調書の根拠チェックや内部統制評価の正確性保証に直接転用できる。また、Memory Agentが複数ターンにわたって制約・要件を追跡・更新する仕組みは、LangGraphのステート管理と組み合わせることで、監査スコープの動的変更に対応するエージェント設計の参考になる。GraphRAGによる検索精度向上は、規制文書・基準書をグラフ構造で管理するRAG基盤の設計にも示唆を与える。
