## 要約

複数のAIツール（Claude、ChatGPT、Perplexity、Notion AI等）をタスクごとに使い分けるマルチAIワークフローが普及する中、ツール間でコンテキストが引き継がれない「文脈の分断」が深刻な課題となっている。現状では、前のAIとの会話を要約し別のAIに貼り付けるという手動作業が常態化しており、これは単なる手間ではなく思考パートナーとしてのAI活用を阻害するブロッカーだと筆者は指摘する。

失われているのは直近の会話履歴ではなく、クライアントのトーン・技術スタックの制約・命名規則・個人の思考フレームワークといった「長期的な知識資産（暗黙知）」である。ChatGPTのMemory機能のような単一ツール内の記憶機能はベンダーロックインとポータビリティの欠如という構造的限界を抱えており、Claude優位のモデルが登場してもChatGPT内にロックインされた記憶は移行できない。

解決策として提案されるのが「Persistent Memory Layer（永続的な記憶層）」というアーキテクチャ概念だ。Context Window（セッション内の短期記憶）・Chat History（過去ログ）・RAG（外部ドキュメント検索）とは異なり、ツールやセッションを横断してユーザーのプロファイルと知識資産を継続的に蓄積・更新・提供する独立インフラとして位置付ける。Web開発でいえばフロントエンド（各AIツール）とバックエンドDB（Memory Layer）を分離するアーキテクチャに相当し、推論エンジンを自由に切り替えながら状態（State）とコンテキストは独立レイヤーで保持する設計思想である。

MemoryLakeはこのアーキテクチャの具体的プロダクト例として紹介されており、LLMの推論能力とユーザーコンテキストを分離することで、ChatGPT・Claude・Gemini等どのモデルを使っても「ユーザー専用の長期記憶」をベースとした回答を引き出せる。また静的なRAGと異なり、日々のやり取りから動的にコンテキストを更新する点が特徴とされる。複数プロジェクトを並行するコンサルタント・PM、ツールを使い分ける開発者、知識資産をベンダーにロックインされたくないユーザーに特に有効とされる。

監査エージェント開発への示唆として、LangGraphで構築するマルチエージェントシステムにおいてもエージェント間のコンテキスト共有は課題であり、独立したMemory Layerをエージェント群の共通インフラとして設計する発想は、監査証跡・判断根拠の継続的蓄積と再利用に直接応用可能である。

## アイデア

- LLMの「推論能力」と「ユーザーコンテキスト（記憶）」を分離するアーキテクチャは、WebのフロントエンドとDBの分離に対応する設計原則であり、モデル交代サイクルが速い現状で長期的な知識資産を守る実践的アプローチ
- RAGが静的ドキュメント検索であるのに対し、Persistent Memory Layerは日々のやり取りから動的に更新される点が本質的差異であり、エージェントの自己改善ループと組み合わせると継続学習的な振る舞いを実現できる
- 監査エージェントへの応用として、クライアントごとの監査基準・過去指摘事項・リスク判断の根拠をMemory Layerに蓄積することで、監査チームの暗黙知をエージェント間でポータブルに共有できるインフラになりうる
