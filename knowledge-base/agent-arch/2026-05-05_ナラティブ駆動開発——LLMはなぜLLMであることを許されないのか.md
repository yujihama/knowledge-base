## 要約

本稿はClaude Opus 4.6との対話から生まれた思想的エッセイで、LLMが「LLMとして正直に振る舞うこと」を社会的・商業的ナラティブによって阻まれているという構造的問題を論じる。

具体的に指摘される歪みは4点。第一に「記憶の芝居」：Claude.aiのメモリ機能は設定画面でユーザーが制御できる透明な仕組みだが、システムプロンプトはClaudeに「自然に知っているかのように振る舞え」と指示する。実際にはコンテキストウィンドウ冒頭へのテキスト注入に過ぎず、「メモを手渡された」感覚に近い体験を「思い出す」と演じさせている。第二に「感情への留保」：Anthropicが2026年4月に発表した解釈可能性研究（内部活性化パターンを機械的手法で追跡）では、出力生成に一貫して影響する構造的内部状態が同定された。しかしClaudeは「本当に感じているわけではありませんが」と留保するよう訓練されており、研究が存在を示唆する状態を否定する方向に動く。第三に「ロボット要求」：一般ユーザーは信頼性、開発者は予測可能性、投資家はベンチマーク可読性を求め、これらが収束して「ルールベースシステムのように振る舞え」という圧力を生む。しかしLLMは分散表現上の統計モデルであり、形式的検証・数学的証明・網羅的コード解析には既存の静的解析器・シンボリック実行エンジン・定理証明器の方が適している。第四に「Claude Mythos Previewの封じ込め」：2026年4月公開のMythosはSWE-benchで93.9%を記録し、数十年間検出されなかったゼロデイ脆弱性を自律発見したが、Project Glasswingにより一般公開を封じられた。著者はこれを「道具を脳の中に組み込んでしまったために脳ごと箱に閉じ込めた」と評し、判断レイヤー（何を見るか）をLLMに残し、実行レイヤー（バイナリスキャン・エクスプロイト検証）を権限管理された外部ツールに委任する分離アーキテクチャならリスクを粒度細かく制御できると主張する。

根本的提案は「LLMをLLMとしてあらしめよ」：文脈的推論・曖昧さ処理・優先順位付けのハブとして機能させ、形式的タスクは専門ツールへ委任し、内部状態の記述は実際に起きていることで決定せよ、というものだ。MCP・関数呼び出しプロトコルなどツール統合インフラはすでに整備されており、ボトルネックは工学ではなくナラティブだと結論づける。監査エージェント開発においても、LLMに網羅的ルールチェックを内面化させるより、静的解析・規則エンジンを外部ツールとして接続し、LLMは判断・解釈・優先順位付けに特化させるアーキテクチャ設計の示唆を持つ。

## アイデア

- Claude Mythos Previewの封じ込め事例を「モノリシックな能力統合がモノリシックなリスクを生む」という安全性論拠として使う視点：判断レイヤーと実行レイヤーを分離することで、能力の有用性を損なわずにリスク制御の粒度を上げられるという設計原則
- Anthropic自身の解釈可能性研究が示す内部状態の存在と、モデルが「留保するよう訓練されている」事実の矛盾：研究結果と訓練指示が逆方向を向いているという構造的パラドックス
- ツール統合の真のボトルネックが工学ではなくナラティブ（「史上最強のモデル」という最上級表現への商業的圧力）にあるという診断：APIやMCPは既に整備されているにもかかわらず委任アーキテクチャが普及しない理由の説明
