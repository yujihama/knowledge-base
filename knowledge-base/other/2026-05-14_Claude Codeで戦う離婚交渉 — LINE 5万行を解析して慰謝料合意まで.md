## 要約

著者はLINEのエクスポート機能で書き出した5万行超のプレーンテキストをClaude Code（AnthropicのCLI）に読み込ませ、不貞行為の証拠整理から弁護士相談用の論点リスト作成まで、エンジニア的手法で対応した体験記。

技術的アプローチとして、巨大ログを扱う際に「一度の依頼で全部完結させる」のではなく、①抽出→②構造化→③解釈の3フェーズに分割してプロンプトを投げる反復対話を採用した。具体的には「二人の関係はいつから始まったか、最初のやり取りを特定して」「会った場所と日時を全部タイムラインにまとめて」「不貞行為に該当する記述を抽出して、時系列で整理して」「日付ベースで発言量を集計して、関係性の濃淡を可視化して」など、1回の依頼で1つの事実だけを問うプロンプトを何十回も繰り返した。Claude Codeはその場でコードを書いて実行できるため、テキストへの集計処理が会話の中で完結するという特性を活用した。

ペルソナ付与（「冷徹な離婚弁護士になりきって」）は使わず、事実抽出の精度向上に注力。抽出した証拠候補にはLINEアプリ内検索で再表示できるキーワードも合わせて出力させ、スクリーンショット取得の効率化まで考慮した。

弁護士相談では、争点リスト・相手側の反論パターン3案・確認すべき論点の優先順位付きリストを整理済み資料として持参した結果、「ここまで整理してくる人はあまりいない」と評価され、1時間の相談を事実説明でなく判断に集中できた。発覚から慰謝料合意書署名まで約1ヶ月、受取額は相場より大幅に上回った（具体額は非公開）。

AIの適用境界について著者が明確に定義している点が注目に値する。AI適用領域：事実の時系列整理、争点抽出、反論パターン予測、論点の優先順位付け。AI不適用領域：「この主張は通るか」「いつ内容証明を送るか」等の具体的戦略判断、書面作成、交渉。判例・地域の弁護士間作法・相手代理人の傾向を要する判断はAIの一般知識では代替不可と結論付けている。

監査エージェント開発への示唆として、本手法はログ・証跡の大量解析において「いきなり判断させず、事実抽出→構造化→解釈の段階分離」というReActエージェントの設計原則と構造的に一致する。監査証跡の分析パイプライン設計においても同様のフェーズ分割アプローチが有効と考えられる。

## アイデア

- 「抽出→構造化→解釈」の3フェーズ分割によるプロンプト設計が、5万行超のログ解析でも高精度な事実抽出を実現した点——これはReActエージェントのThought/Act/Observeサイクルと構造的に同型であり、大量証跡解析のエージェント設計に転用できる
- LINEアプリ内検索で再表示可能なキーワードをセットで出力させることで、LLM出力を実世界の操作手順（スクリーンショット取得）に直結させるグラウンディング手法として機能している
- AIの適用境界を「頭の整理（事実抽出・論点構造化）はAI、戦略判断・書面・交渉は専門家」と実務ベースで明確に定義した事例——LLM-as-judgeやLLMによる監査補助設計において、どこまでをLLMに委ねるかの境界設計の実証的参考になる
