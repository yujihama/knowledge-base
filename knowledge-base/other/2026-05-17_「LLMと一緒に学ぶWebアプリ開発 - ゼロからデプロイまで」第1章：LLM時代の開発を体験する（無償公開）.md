## 要約

本書はプログラミング初心者を対象に、LLMを活用しながらDjango製WebアプリをAWS EC2にデプロイするまでを解説するハンズオン形式の書籍。第1章が無償公開されており、全体の概要と学習方針が示されている。

サンプルアプリとして、Gemini APIを使ってストーリーをリアルタイム生成するテキストアドベンチャーゲームを開発する。技術スタックはバックエンドにPython 3.13 + Django 5.2 + SQLite + Gunicorn、フロントエンドにhtmx（JavaScript不要）、インフラにAWS EC2 + Nginx + Let's Encrypt（HTTPS化）、開発ツールにGit + Cursor IDEを採用。サーバー1台で完結するシンプルな構成を意図的に選択し、「環境構築からデプロイまでの体験」に学習リソースを集中させている。

本書の主張する要点は、LLM登場（2022年11月のChatGPT公開）以降、学習フローが「Google検索→Stack Overflow→英語ドキュメント→1時間の試行錯誤」から「LLMへの質問（30秒）→対話で深掘り（5分）→実装（10分）」へ短縮されたこと。これにより挫折ポイントが大幅に減少し、初心者でも完走しやすくなった。

一方、LLMが苦手な領域として「学習データ締切以降の最新情報」「プロジェクト全体の文脈理解」「最適なアーキテクチャ判断」「セキュリティの深い考慮」を明示しており、最終判断は人間が行う前提でLLMを「最高の相棒」として位置づけている。

全13章・合計約23時間の学習量で構成。第1部（第1-4章）が準備・企画・Django基礎、第2部（第5-7章）がGemini API連携・チャットUI・ゲームロジック、第3部（第8-12章）がAWS準備・EC2デプロイ・HTTPS化・運用、第4部（第13章）が機能拡張。最難関は第10章（EC2デプロイ、目安4時間）。

監査エージェント開発への示唆：LLMへの「良い質問の構造」として本書が提示するフォーマット（背景・現状・問題・試したこと・環境・質問）は、エージェントへのプロンプト設計やReActパターンでのツール呼び出し記述に直接応用できる。また「LLMが苦手なアーキテクチャ判断とセキュリティ考慮は人間が担う」という役割分担の明示は、監査エージェント設計における人間-AI協働モデルの整理に参考となる。

## アイデア

- LLMへの質問構造（背景・現状・問題・試したこと・環境・質問）を定型フォーマット化している点は、エージェントへのプロンプトテンプレート設計にそのまま転用できる
- htmxを採用することでJavaScript不要のチャットUIを実現しており、フロントエンド複雑性を排除しながらインタラクティブなLLM連携UIを構築するアプローチとして参考になる
- SQLiteを開発・本番共通で使用するという意図的な割り切りが、学習コストを下げつつ「デプロイ体験」に集中させる設計判断として明快
