## 要約

Claude Code を日本語で使用していると、一人称はほぼ「私」で安定しているが、セッションログを精査すると「僕」が出現するケースがある。著者は手元の4,372本のClaude Codeトランスクリプト（JSONL形式）をスクリプトで機械的に解析し、「僕」を含むassistantロールの発話を抽出した。その結果、明確に観測できたのは2セッションのみ（全体の約0.046%）。セッションAでは「私」20回・「僕」10回、セッションBでは「私」0回・「僕」15回という分布だった。

「僕」が出現した箇所を詳細に分析すると、いずれも推薦・意見表明の場面であり、かつ「推し」「推奨」という語彙の使い分けと連動していた。「私の推奨は」はフォーマルな説明調、「僕の推しは」はカジュアルで前のめりなトーン、という形で一人称と語彙レジスタが同時にスライドしていた。つまり揺れているのは人称単体ではなく、文全体のあらたまり度（フォーマリティ）である。セッションBが最初から最後まで砕けた調子だったため「私」が一切出なかったことも、この仮説と整合する。

この現象が英語圏で観測されないのは、英語の一人称が「I」一択であり、フォーマリティが変化しても表層形が変わらないためだ。日本語では「私／僕／俺」等が場面・相手・自己呈示によって使い分けられるのが常態であり、モデルはその分布をそのまま学習している。

Anthropicは「The Persona Selection Model」という枠組みでLLMを「多数のペルソナを演じうる俳優」と位置づけており、「アシスタント」はその一つのキャラクターにすぎない。システムプロンプトで一人称を固定していない限り、文脈に応じて分布から選ばれる余地がある。Anthropic自身も「Keep Claude in character」ガイドでキャラクターの一貫性は放置すると揺れうると認めており、CLAUDE.mdで一人称を明示指定すれば固定可能。なお観測された2セッションはいずれもOpus 4.7だったが、n=2では世代固有の挙動とは断言できない。

## アイデア

- 日本語の一人称は場面依存で揺れるという言語的性質がLLMの出力に直接表面化し、英語では観測不可能な挙動として現れる点は、多言語LLM評価の盲点を示している
- 一人称の揺れが「推し」vs「推奨」という語彙レジスタの変化と連動しているという観察は、LLMが文全体のスタイル一貫性を局所的に保とうとする挙動を示唆しており、文体制御の研究に応用できる
- CLAUDE.mdやシステムプロンプトで一人称を明示指定することでキャラクターの一貫性を強制できるという実用的な対処法は、エージェントシステム設計においてペルソナ固定が必要な場面（監査エージェント等）で直接活用できる
