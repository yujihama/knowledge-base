## 要約

大規模言語モデル（LLM）のパーソナライズには LoRA などのパラメータ効率的ファインチューニング（PEFT）が広く使われているが、複数ユーザー向けにアダプタを同時配信する際にスループットが大幅に低下するという問題がある。本論文はその根本原因を「プリフィル」と「デコード」の非対称性に求める。プリフィルは大量トークンを一括処理するため並列化が効きやすいが、デコードは1トークンずつ自己回帰的に生成するため、複数アダプタを同時処理するとスループットが極端に落ちる。

この問題を解決するために提案されたのが PreFT（Prefill-only Finetuning）である。PreFT はアダプタをプリフィルフェーズのトークンにのみ適用し、デコードフェーズでは廃棄する。これにより、アダプタによる計算オーバーヘッドをデコード段階から完全に除去できる。実装は vLLM 推論エンジン上で LoRA と ReFT（Representation Fine-Tuning）の2種類のプリフィル専用アダプタとして公開されている。

実験結果として、Llama 3.1 70B に512個のアダプタを同時配信した場合、従来の PEFT に比べてスループットが 1.9 倍に向上した。性能面では、SFT（教師あり微調整）タスクにおいて PreFT の評価ロスは通常の PEFT より高くなるが、アダプタのランクを増やすことでほぼスループット低下なく補償できることが示された。RL（強化学習）タスクでは PreFT は標準的な PEFT と同等の性能に一貫して近づくことが確認された。

本研究の核心的な主張は「パラメータ数に対する性能」ではなく「スループットに対する性能」を最適化指標とすべきという設計思想の転換にある。監査エージェント開発への示唆として、複数テナント・複数ユーザー向けに個別カスタマイズされた LLM を同一サーバーで低コスト配信するユースケース（例：組織ごとに微調整された監査アシスタント）において、PreFT は実用的なスループット改善手段となり得る。

## アイデア

- プリフィルとデコードの非対称性を利用し、アダプタ適用をプリフィル段階のみに限定することでデコードのボトルネックを根本的に回避するという発想の転換
- RL タスクでは SFT より PreFT の性能劣化が小さいという観測は、強化学習においてアダプタのデコード段階への影響が相対的に小さいことを示唆しており、RLAIF や GRPO 文脈での応用可能性がある
- 512アダプタ同時配信で 1.9 倍のスループット向上という具体的な数値が示すように、マルチテナント SaaS 型 LLM サービスの経済性を大きく変えうる実装レベルの提案
