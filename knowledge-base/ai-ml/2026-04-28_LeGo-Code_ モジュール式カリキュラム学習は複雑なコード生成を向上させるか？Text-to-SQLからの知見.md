## 要約

本論文は、Text-to-SQLタスクにおけるLLMのファインチューニング手法として「カリキュラム学習」の有効性を検証した研究。Text-to-SQLとは自然言語をSQL文に変換する技術で、非技術者がリレーショナルDBと対話するために使われる。SpiderおよびBIRDベンチマークを用いて実験が行われた。

課題として挙げられるのは、複数の結合（JOIN）や条件を含むネスト構造の深いSQLクエリの生成精度、およびノイズが多く構造化が不十分な実世界のDBスキーマへの対応である。まず「ナイーブなカリキュラム学習」として、訓練サンプルを複雑度順（Easy→Hard）に並べて1エポック学習させる手法を試みたが、これは壊滅的忘却（catastrophic forgetting）を引き起こし、通常のファインチューニングを上回れなかった。

これを克服するために提案されたのが**MAC（Modular Adapter Composition）戦略**である。複雑度レベル（Easy／Medium／Hard／Extra-Hard）ごとに専用のLoRAアダプタを段階的に訓練し、各階層のアダプタを「Lego」のように組み合わせて推論時に使用する。各アダプタはその難易度帯のサンプルのみで訓練されるため、前段階の知識を上書きする壊滅的忘却が発生しない。推論時はスキーマの難易度に応じて必要なアダプタを選択・合成することで柔軟なデプロイが可能となる。

Spider・BIRDベンチマーク上で測定可能な性能向上が確認されており、特に複雑クエリにおいてモノリシックなファインチューニングより優れることが示された。また「Lego-like」アーキテクチャの利点として、新たな難易度レベルへの対応にはその層のアダプタのみを追加訓練すれば良く、全体の再訓練が不要である点も強調されている。

監査エージェント開発への示唆としては、監査手続の複雑度をEasy（単一テーブル照会）からExtra-Hard（複数テーブル・複合条件）に段階分けし、それぞれ専用アダプタを訓練するMAC戦略を適用することで、監査クエリ自動生成の精度を段階的に高められる可能性がある。また、Pydanticによるスキーマ検証と組み合わせれば、難易度ルーティングのロジックをエージェント内に組み込める。

## アイデア

- 壊滅的忘却をアダプタ分離で回避するMAC戦略は、難易度が明確に段階化できるあらゆるコード生成タスク（例: 監査クエリ、API呼び出し生成）に転用できる
- 難易度ごとにアダプタを独立訓練することで、新しい複雑度レベルの追加が全体再訓練なしに済む「拡張可能な学習アーキテクチャ」という設計思想が新しい
- スキーマ難易度の自動判定ルーターと組み合わせることで、推論時に最適なアダプタ組み合わせを動的選択するエージェント的な推論システムに発展できる
