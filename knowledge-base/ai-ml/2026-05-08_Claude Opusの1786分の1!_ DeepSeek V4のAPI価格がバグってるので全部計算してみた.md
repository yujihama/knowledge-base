## 要約

2026年4月にリリースされたDeepSeek V4シリーズのAPI価格が、主要モデルと比較して桁違いに安い。DeepSeek V4 Flashの入力単価は100万Tokenあたり$0.14で、Claude Opus 4.6の$5.00と比べて約36倍安く、GPT-4oの$2.50と比べても約18倍安い。出力単価も$0.28と、Claude Opus 4.6の$25.00の約89倍安い。さらにDeepSeekはデフォルトで有効化されているContext Caching（KVキャッシュのディスク永続化）機能を持ち、キャッシュヒット時の入力単価は$0.0028（通常の1/50）まで下がる。Claude Opus 4.6の通常入力$5.00と比較すると約1786倍の価格差になる。キャッシュの仕組みは「プレフィックス一致」で、リクエスト境界での永続化・共通プレフィックスの自動検出・固定トークン間隔での永続化の3つのタイミングで機能する。コード変更不要でデフォルト有効のため、マルチターン会話や同一文書への複数クエリ（RAG等）は自動的に恩恵を受ける。ヒット率を最大化するには、静的コンテンツ（システムプロンプト・参考文書・Few-shot例）をプロンプト先頭に固定し、変動部分（ユーザー質問）を末尾に配置するのが鉄則。逆に、システムプロンプトにタイムスタンプ等の動的情報を混ぜると1文字の差でキャッシュミスするため注意が必要。APIレスポンスのusageフィールドにprompt_cache_hit_tokensとprompt_cache_miss_tokensが返されるため、定量的なモニタリングが可能。実用試算として、10万TokenのシステムプロンプトでAPI日100回コールするケースでは、Claude Opus 4.6が月約$1,500なのに対し、DeepSeek V4 Flash（キャッシュヒット）は月約$0.84。大量トークンを消費するエージェントやRAGシステムの運用コスト削減において、DeepSeek V4は有力な選択肢となる。ただしキャッシュはベストエフォートで100%ヒット保証はなく、未使用キャッシュは数時間〜数日で自動クリアされるため、散発的なリクエストでは恩恵が薄い点は留意が必要。監査エージェント開発への示唆として、長大なルールベースや監査基準文書をシステムプロンプトに固定配置するRAG構成と相性が良く、反復的な文書分析タスクにおいて大幅なコスト削減が見込める。

## アイデア

- KVキャッシュのディスク永続化をデフォルト有効化し、プレフィックス一致で自動的に課金を1/50にする設計は、プロンプトエンジニアリングの経済的インセンティブを根本から変える
- Claude Opus比1786倍の価格差は単なるコスト削減ではなく、エージェントのループ回数や試行錯誤の許容度を設計上のパラメータとして扱えるようになることを意味する
- APIレスポンスのusageフィールドでキャッシュヒット率を定量監視できる設計は、プロンプト設計のROI計測に直結し、MLOps的なコスト最適化サイクルを組み込める
