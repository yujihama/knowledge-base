## 要約

Moonshot AIが2026年6月12日にHugging Faceで公開したKimi K2.7 Codeは、思考モード（reasoning）を常時オンに固定したコーディング特化のオープンウェイトモデルだ。アーキテクチャはMixture-of-Experts（MoE）で、総パラメータ1兆に対して活性化パラメータは320億。384エキスパートから8つを選択し、共有エキスパートを1つ持つ構成。レイヤー数は61（うち1 dense）、コンテキスト長は256K、AttnはMLA（Multi-head Latent Attention）を採用し、テキスト・画像・動画の入力に対応する。ライセンスはModified MITで、帰属表示があれば商用利用が可能。

APIはOpenAI互換で、base_urlとmodelパラメータを差し替えるだけで既存クライアントから利用できる。価格は入力$0.95/MTok、キャッシュヒット時$0.19/MTok、出力$4.00/MTok。キャッシュヒット時の大幅値引きは、長いシステムプロンプトやコードベースを繰り返し参照するエージェント用途を明確に意識した設計だ。

このモデル最大の特徴は`preserve_thinking=True`固定という設計思想にある。推論チェーンはマルチターンをまたいで保持され、過去の推論を文脈として持ち越す。長丁場のソフトウェア開発（計画→編集→ツール実行→テスト→デバッグ）を一本の流れとして扱うことを前提とした設計だ。同時に、前世代K2.6と比較して思考トークン消費を約30%削減したとMoonshotは主張しており、「常時思考」と「トークン効率」の両立を図っている。また`message.reasoning`と`message.content`が分離して返るため、ログに推論だけ残してUIには結論だけ表示するといった出し分けがアプリ側で容易に実装できる。

一方、制約も明確だ。temperatureは1.0/top_p 0.95に固定されており、決定的な挙動が要るCI上の生成やインライン補完には不向き。重みはディスク上約595GBで、セルフホスト（vLLMまたはSGLang）にはマルチGPUノードが必要。ベンチマークはKimi Code Bench v2（62.0）やMLS Bench Lite（35.1）など自社定義が中心で、独立系への提出は見送られており、第三者検証待ちの状態。エージェント向けツール利用能力を測るMCPMarkでは81.1を記録している。

監査エージェント開発への示唆としては、長丁場の自律的作業（複数ファイルの参照・編集・ツール呼び出しを繰り返す）に推論文脈を持ち越す設計は、LangGraphベースの監査エージェントにおける多ステップ推論の品質向上に参考になる。ただし固定temperatureと巨大な重みサイズから、本番投入前にインフラコストと用途適合性の精査が必要。

## アイデア

- 思考チェーンをマルチターンで持ち越す`preserve_thinking=True`固定設計により、単発Q&Aではなく長丁場エージェント作業における推論の一貫性を担保する発想は、エージェントアーキテクチャ設計に直接応用できる
- キャッシュヒット時の入力単価を通常の約1/5（$0.95→$0.19/MTok）に設定することで、長いシステムプロンプトやコードベースを繰り返し参照するエージェント用途のコスト構造を意図的に最適化している
- `message.reasoning`と`message.content`を分離して返すAPIインターフェースにより、推論プロセスのロギング・デバッグと最終出力の提示を独立して制御できる設計パターンは、LLM-as-judgeや監査トレーサビリティに活用できる
