## 要約

本論文は、プログラム検証アーティファクト（検証結果・仕様・証明）の再利用を目的として、命令型プログラムとそのアノテーションをグラフ表現に変換するパイプラインを提案する。

**背景と課題**
ソフトウェア検証では、類似プログラムに対して同様の検証作業を繰り返すコストが高い。既存の検証アーティファクトを再利用するには、プログラム間の構造的・意味的類似性を自動的に識別する手段が必要だが、異なる言語や仕様記述スタイルをまたいだ統一的なグラフ構築手法は確立されていなかった。

**提案手法**
パイプラインは以下のステップで構成される：
1. **AST解析**：対象プログラム（C/ACSL、Java/JML、Dafny for C#）を抽象構文木（Abstract Syntax Tree）に変換し、構造的関係を抽出する。
2. **意味的埋め込み**：SentenceTransformerおよびCodeBERTを用いてコードトークンや仕様アノテーションのベクトル表現を生成し、ノード属性として付与する。
3. **型付き属性グラフの生成**：構造情報（AST由来）と意味情報（埋め込み由来）を統合した、typed attributed graphを出力する。

**対象データセット**
- C言語 + ACSL（ANSI/ISO C Specification Language）
- Java + JML（Java Modeling Language）
- C#向けのDafny

これら異なる言語・アノテーションスタイルにまたがる実験を行い、一貫したグラフ表現が構築できることを確認した。

**結果と意義**
異なるプログラミング言語・仕様言語の組み合わせに対して、統一フォーマットのグラフ表現が生成できることを実証。これはスケーラブルな検証アーティファクト再利用の基盤となる。今後の展開として、意味的エンリッチメントと近似グラフマッチングへの発展が示されており、類似プログラム検索や証明転移（proof transfer）への応用が期待される。

**監査エージェント開発への示唆**
LangGraphベースの監査エージェントにおいて、監査手順書や内部統制フローをグラフ構造に変換し、過去の類似監査ケースを検索・再利用する仕組みに応用できる。ACSLやJMLのようなフォーマル仕様と同様に、監査手順をアノテーション付き構造として表現し、CodeBERT等で意味エンベディングを付与することで、監査シナリオ間の類似度マッチングが可能になる。

## アイデア

- AST（構造）とCodeBERT埋め込み（意味）を同一グラフノードの属性として統合するアーキテクチャは、コード検索やクローン検出にも応用可能
- C/Java/C#という異言語間で統一グラフスキーマを保つ設計は、マルチ言語コードベースの静的解析ツール構築の参考になる
- 検証アーティファクト再利用という目的は、監査エージェントにおける過去監査手順の再利用・転移学習的アプローチに直結する
