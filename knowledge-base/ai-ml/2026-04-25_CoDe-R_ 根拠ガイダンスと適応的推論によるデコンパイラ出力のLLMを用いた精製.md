## 要約

バイナリデコンパイルは、ストリップされた実行ファイルから高レベルのソースコードを再構築するリバースエンジニアリングの基幹タスクである。コンパイル時に生じる不可逆な意味情報の喪失により、LLMを適用した既存手法では「論理的ハルシネーション」（アルゴリズム的に誤った構造の生成）と「意味的ミスアライメント」（元の意図と乖離したコード）が頻発し、生成コードが再実行不能になる問題があった。

本研究はこれに対処するため、2段階の軽量コード精製フレームワーク「CoDe-R（Cognitive Decompiler Refinement with Robustness）」を提案する。

第1段階は「意味的認知強化（SCE: Semantic Cognitive Enhancement）」であり、「根拠誘導型意味インジェクション（Rationale-Guided Semantic Injection）」戦略を採用する。これはモデルをトレーニングする際に、コード本体だけでなく高レベルのアルゴリズム意図（rationale）も同時に学習させる手法で、コンパイルで失われた高水準の意味情報を推論時に復元できるよう訓練する。

第2段階は推論時に適用される「動的デュアルパス・フォールバック（DDPF: Dynamic Dual-Path Fallback）」機構であり、意味的回復（semantic recovery）と構文的安定性（syntactic stability）をハイブリッド検証戦略によって適応的にバランスさせる。具体的には、生成候補ごとに再実行可能性を検証し、失敗した場合にはフォールバックパスへ切り替えることで出力の堅牢性を高める。

HumanEval-Decompileベンチマークでの評価では、1.3Bパラメータのバックボーンモデルを用いたCoDe-Rが軽量モデルカテゴリで新たなSOTA（最高性能）を達成。特に「平均再実行可能率（Average Re-executability Rate）」が50.00%を超えた最初の1.3Bモデルとなり、ベースラインを大幅に上回った。これは、7B・13Bクラスのモデルに匹敵するパフォーマンスを1.3Bで実現することを意味し、効率的な軽量モデルとエキスパートレベルの性能の間のギャップを埋める成果として位置づけられる。コードはGitHubで公開されており、再現・拡張が可能。

監査エージェント開発への示唆として、コード解析・証跡検証シナリオにおいてバイナリ解析や難読化コードの意味復元に本フレームワークの考え方（rationale注入＋フォールバック検証）が応用できる可能性がある。特に、軽量モデルで高い再実行可能率を達成するDDPF的な検証ループは、LLM-as-judgeによる自動コードレビューエージェントの設計にも参考になる。

## アイデア

- 1.3Bという小規模モデルで再実行可能率50%超を達成した点：「根拠（rationale）を明示的に学習させる」アプローチが、単純なコード生成より圧倒的に効果的であることを示しており、小型モデルでも意味理解を強化できる訓練設計の可能性を示す
- Dynamic Dual-Path Fallbackの設計思想：生成→検証→フォールバックというループ構造は、エージェント的な自己修正フローに近く、推論時の動的ルーティングがコード生成品質を底上げするパターンとして汎用性が高い
- コンパイルによる不可逆な情報損失への対処法として「rationale injection」を訓練に組み込む発想：これはRAGや思考連鎖（CoT）とは異なる、学習段階での意味情報埋め込み戦略であり、他のコード理解・変換タスクにも展開できる
