## 要約

GEM-RAG（Graph-Enhanced Memory RAG）のスペクトルグラフ理論的アイデアをPythonで簡略実装した実践記事。本家GEM-RAGはラプラシアン行列の固有ベクトルを使うが、本実装では「類似度グラフ＋固有ベクトル中心性（Eigenvector Centrality）」によるハイブリッドスコアリングに絞り、GemRagEngineクラスとして実装している。

処理フローは3ステップ。①sentence-transformers（paraphrase-multilingual-MiniLM-L12-v2）でチャンクをエンベディング、②コサイン類似度行列から閾値（デフォルト0.5）を超えるエッジのみ残した隣接グラフをNetworkXで構築、③nx.eigenvector_centrality()で各ノードの重要度スコアを算出。検索時はScore = α×類似度 + (1-α)×固有スコア（デフォルトα=0.6）のハイブリッドで順位付けする。

技術メモ6件での実験では、RAG/GEM-RAGの中核説明チャンク（ID 4）が固有スコア1.0、完全ノイズの「カレーライス」文（ID 5）が0.0となり、グラフ上のハブ構造がノイズ除去に機能することを確認。「GEM-RAGの仕組みは？」クエリに対し、直接類似度0.72のID 4が1位、0.77のID 3が2位、類似度は低い（0.32）がRAG文脈のハブであるID 1が3位に来る結果となった。

170チャンクに分割した1万字の物語テキストでも同様に検証。手紙の告白シーン・再会・死の直前など「物語の軸となるシーン」が固有スコア0.92〜1.0の高スコアを獲得し、序盤の日常描写は0.09〜0.47に留まった。ドメインと無関係のクエリを投げても固有スコアが高いシーンが引き上げられる挙動も確認。

監査エージェント開発への示唆として、この手法は「参照頻度の高い監査基準や内部統制記述をハブノードとして自動同定する」用途に応用可能。条文間の意味的類似グラフを構築し固有ベクトル中心性で重要条文をスコアリングすれば、RAG検索時の精度向上や証拠文書の優先度付けに使える。収束しない場合の次数ベースフォールバックも実装済みで、実用上の堅牢性も考慮されている。

## アイデア

- 類似度グラフ上の固有ベクトル中心性をチャンク重要度とみなすことで、直接類似度が低くても文脈的ハブとなる文書を検索上位に引き上げられる
- 物語テキストで「ストーリーの転換点・感情的クライマックス」が自動的に高スコアを得る挙動は、長文ドキュメントの要約・ハイライト抽出への応用可能性を示している
- 収束失敗時に次数重み和ベースのフォールバックを設けることで、疎なグラフや孤立ノードが多い実務データでもrobustに動作する設計になっている
