## 要約

2026年末にGoogleのPerspective APIが終了することを受け、本論文はNLP・計算社会科学（CSS）・LLM評価研究コミュニティがこの単一の独自ツールに構造的に依存してきた問題を体系的に記録・分析する。Perspective APIはテキストの毒性（toxicity）を自動測定するAPIとして事実上の業界標準となっており、多くのベンチマークやLLM評価パイプラインで採用されてきた。しかし論文は以下の深刻な問題点を指摘する。第一に、Perspectiveのモデルはバージョン管理や変更開示なしに定期的に更新されており、同じAPIを使った過去の実験が再現不可能になるリスクがある。第二に、アノテーション構造が単一企業（Google）の毒性概念の定義に依拠しており、「毒性」という争点ある概念を一企業の操作的定義に閉じ込めてしまっていた。第三に、Perspectiveのスコアが評価ターゲット（訓練目標）と評価基準（評価指標）の両方として同時に使用されており、循環的な評価構造が形成されていた。APIの終了により、既存のベンチマークは更新不可能となり、過去の研究結果は再現困難になる。また、この穴を埋めようとしてGPT-4などのクローズドソースLLMをそのまま代替測定ツールとして採用する動きが起きており、同様の依存構造が再生産されるリスクを論文は強調する。これに対し著者らは独立した・妥当性のある・適応可能な・再現可能な毒性・ヘイトスピーチ測定インフラの構築を提唱し、技術要件（オープンソース、バージョン管理、コミュニティガバナンス）とガバナンス要件を具体的に示している。監査AIの観点からは、外部の単一ベンダーツールへの依存がいかに評価の信頼性・再現性・独立性を損なうかという構造的教訓は、AIガバナンスや内部統制の評価フレームワーク設計にも直接応用できる示唆を持つ。

## アイデア

- 評価ツールが評価ターゲットと評価基準を兼ねる循環構造：Perspectiveスコアで訓練したモデルをPerspectiveで評価するという自己参照的な評価設計が、研究全体のエピステミックな健全性を損なっていた点は、LLM-as-judgeパイプライン設計でも同様の罠に陥りうることを示唆する
- 単一プロプライエタリツールへの依存リスクの定量化：コミュニティ全体がどれだけ一つのAPIに依存していたかを文献調査で可視化し、「インフラの脆弱性」として研究エコシステムレベルで問題を捉えるアプローチは、AI評価ガバナンスの新しい分析フレームとなりうる
- オープンで適応可能な毒性測定インフラの設計要件：バージョン管理・コミュニティガバナンス・概念の多元性（単一定義ではなく争点ある概念として扱う）を組み込んだ測定インフラの設計原則は、監査AIにおける評価指標の選定・管理プロセスにも応用できる
