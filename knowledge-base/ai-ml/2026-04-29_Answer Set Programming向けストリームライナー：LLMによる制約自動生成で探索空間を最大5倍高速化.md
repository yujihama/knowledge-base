## 要約

本論文は、組み合わせ問題の探索空間を絞り込む「ストリームライナー制約」を、Answer Set Programming（ASP）に対してLLMを用いて自動生成するアプローチ「StreamLLM for ASP」を提案する。

【背景】ASPは宣言的論理プログラミングの一形式で、組み合わせ最適化・プランニング・制約充足問題を解くために広く使われる。問題規模が大きくなると探索空間が爆発的に増大するため、解の一部を事前に排除する「ストリームライナー制約」を追加することで求解器の効率を上げる手法が研究されてきた。従来はこれを人手で設計する必要があったが、本研究ではLLMによって自動生成する。

【手法】入力はASPエンコーディングと少数の小規模訓練インスタンス。複数のLLMに候補制約を提案させ、以下のフィルタリングを行う：(1) 構文エラーを含むもの、(2) 充足可能なインスタンスを充足不能にするもの、(3) 訓練インスタンス全体で性能を低下させるものを除去。残った有効なストリームライナーを元のエンコーディングと組み合わせて評価する。最終的に各インスタンスで最速の変種を選ぶ「Virtual Best Encoding（VBE）」として結果を報告する。

【実験結果】ASP Competitionの3ベンチマーク（Partner Units Problem、Sokoban、Towers of Hanoi）で評価。VBEは元のエンコーディングに対して最大4〜5倍の高速化を達成した。また、異なるLLMが生成する制約は単なる構文的バリエーションではなく意味的に多様であり、それぞれが問題の異なる構造的側面を捉えていることが示された。

【監査エージェント開発への示唆】監査・GRCドメインのルールベース制約（コンプライアンスチェック、異常検出ルール等）をASPで記述し、LLMで自動的に探索空間を絞り込む制約を追加する設計が考えられる。特に制約の有効性を自動検証するパイプライン（構文チェック→充足性チェック→性能評価）の構造は、監査エージェントにおける仮説生成・検証ループの設計パターンとして参考になる。

## アイデア

- LLMによる制約自動生成を3段階フィルタリング（構文・充足性・性能）でロバストに選別するパイプラインは、LLM出力の品質保証手法として汎用性が高い
- 複数の異なるLLMを並列に使うことで意味的に多様な制約が得られ、アンサンブル的な探索空間削減が実現できる点は、マルチエージェント設計の一形態として興味深い
- Virtual Best Encoding（VBE）という概念は、単一の最適解ではなくインスタンスごとに最適な戦略を選ぶ「オラクル上界」として、ベンチマーク設計の評価フレームとして再利用できる
