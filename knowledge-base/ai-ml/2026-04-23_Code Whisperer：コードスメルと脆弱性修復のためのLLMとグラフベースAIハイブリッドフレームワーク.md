## 要約

本論文は「The Code Whisperer」と呼ばれるハイブリッドフレームワークを提案する。コードスメル（保守性の低下を招くコード構造上の問題）とセキュリティ脆弱性は、従来別々のツールで対処されてきたが、それぞれ構造的文脈の欠如やノイズの多い警告という課題を抱えていた。本手法はこれを単一のワークフローで統合的に検出・説明・修復する。

技術的な核心は、4種類のグラフ表現とLLMを組み合わせた「構造・意味の同時学習」にある。具体的には、抽象構文木（AST）、制御フローグラフ（CFG）、プログラム依存グラフ（PDG）、そしてトークンレベルのコード埋め込みをアライメントし、構造シグナルと意味シグナルを同時に学習させる。ASTはコードの構文構造を、CFGは実行フローを、PDGはデータ・制御依存関係を捉えることで、ルールベース解析器が見落としがちなコンテキストを補完する。

評価は複数言語のデータセットで実施され、ルールベース解析器および単一モデルベースライン（グラフのみ、LLMのみ）と比較。ハイブリッド設計が検出性能と修復提案の有用性の両面でそれらを上回ることを示した。特に「グラフのみ」アプローチがコード意味論を捉えきれない一方、「LLMのみ」が構造的依存関係を見落とすという弱点を、両者の統合で補完している。

実用面では、CI/CDパイプラインへの統合と説明可能性（explainability）も設計要件として明示的に組み込まれている。これは単なる研究提案にとどまらず、実際の開発ワークフローへの展開を意識した設計判断である。

監査エージェント開発への示唆として、本フレームワークのアーキテクチャパターン（複数の異種グラフ表現をLLMと融合させる手法）は、監査証跡・業務プロセスの依存関係グラフとLLMを組み合わせた異常検知・リスク評価システムの設計に直接応用できる。特にPDGに相当する「業務依存グラフ」を構築しLLMと統合することで、ルール違反の根本原因分析（RCA）を構造的文脈込みで行える可能性がある。

## アイデア

- AST・CFG・PDGの3種グラフをトークン埋め込みとアライメントする設計は、コード以外の構造化ドキュメント（契約書、監査手続書）への応用可能性がある
- 「グラフのみ」と「LLMのみ」の弱点を相互補完するハイブリッド設計は、監査証拠の構造解析と自然言語推論を組み合わせるReActエージェントの設計指針として参照できる
- CI/CD統合と説明可能性を最初から設計要件に含める姿勢は、監査AI開発における『説明責任』要件（監査調書への根拠記載義務）への対応パターンとして参考になる
