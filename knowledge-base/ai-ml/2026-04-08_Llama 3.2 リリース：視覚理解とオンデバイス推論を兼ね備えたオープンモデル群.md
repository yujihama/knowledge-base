## 要約

Meta が Llama 3.2 をリリースし、Hugging Face と協力して10種類のオープンウェイトモデルを公開した。内訳は、マルチモーダルモデル5種（Vision 11B/90B のベース・インストラクション版 + Llama Guard 3 Vision）とテキスト専用小型モデル5種（1B/3B のベース・インストラクション版 + Llama Guard 3 1B）。

【Llama 3.2 Vision】
アーキテクチャは Llama 3.1 LLM（8B または 70B）にビジョンタワーとイメージアダプターを追加した構成。テキストモデル部分はビジョン学習中に凍結され、テキスト性能を維持。60億枚の画像テキストペアで学習済み。コンテキスト長は128kトークン。11B はタイルサイズ448（ベース）または560（インストラクション）、90B はタイルサイズ560。主なベンチマーク：11B インストラクション版で MMMU 50.7（CoT）、DocVQA 88.4、AI2D 91.1。90B インストラクション版では MMMU 60.3、DocVQA 90.1、AI2D 92.3。画像＋テキスト入力は英語のみ対応、テキスト単独では英・独・仏・伊・葡・ヒンディー・西・タイ語の8言語対応。視覚的推論、ドキュメントQA、インフォグラフィック理解などに強みを持つ。

【Llama 3.2 1B・3B（テキスト専用）】
Llama 3.1 と同一アーキテクチャ。9兆トークンで学習、128kトークンコンテキスト対応、8言語サポート。プロンプト書き換え、多言語知識検索、要約、ツール使用、ローカルアシスタントなどオンデバイスユースケース向け。同サイズ帯のオープンモデルで最高クラスの性能を主張。

【インフラ・エコシステム】
Hugging Face Transformers・TGI による推論、Inference Endpoints・Google Cloud・Amazon SageMaker・DELL Enterprise Hub へのデプロイ統合、Llama.cpp・Transformers.js によるオンデバイス推論をサポート。TRL を用いて単一 GPU で Llama 3.2 11B Vision のファインチューニングが可能。

【ライセンス変更】
EU 居住者・EU 拠点企業はマルチモーダルモデル（Vision 系）のライセンス権を付与されない。ただし、Vision モデルを組み込んだ製品・サービスのエンドユーザーへの制限はなく、グローバルプロダクト構築自体は可能。テキスト専用モデルへの制限なし。

## アイデア

- テキストモデルを凍結したままビジョンアダプターのみを学習することで、既存の言語性能を損なわずにマルチモーダル能力を追加できる設計思想は、既存エージェントへの視覚機能後付けに応用できる
- 1B・3B モデルがツール使用（tool use）をオンデバイスで実行できる点は、エッジでの軽量エージェント実行の実現可能性を示す
- Llama Guard 3（1B・Vision対応版）を同一スタック内でサイドカーとして組み込み、マルチターン会話の安全性を低コストで評価できる構成は、エンタープライズ利用での内部コンプライアンス実装に直結する

## Yujiの取り組みへの示唆

監査エージェント開発において、Llama 3.2 Vision（11B）はドキュメントQA（DocVQA 88.4）やインフォグラフィック理解に強く、財務諸表や監査調書の画像を直接入力とするエージェントステップに活用できる。Llama Guard 3 を LangGraph のノードとして組み込むことで、エージェントの応答を自動分類・フィルタリングする LLM-as-judge パターンを低コストで実現できる。1B・3B モデルのツール使用対応は、ローカルLLMインフラ（RTX 3090）上で軽量サブエージェントを動かすユースケースにも適合する。
