## 要約

MAIC-UIは、教育者がHTMLやJavaScriptの知識なしに、教科書・PPT・PDFからインタラクティブなSTEM教材を作成・編集できるゼロコードオーサリングシステムである。

従来の課題として、生成AIによるHTML生成ツールは静的なプレゼンテーションにとどまり、インタラクティブなシミュレーションを生成できなかった。また、長文ドキュメントへの対応が困難で教育的な正確性の担保が不十分であり、修正のたびに200〜600秒のフル再生成が必要となり創作フローを阻害していた。

MAIC-UIはこれらの課題を以下の3つの技術的アプローチで解決する。

(1) **構造化知識分析（Structured Knowledge Analysis）**: マルチモーダル理解を活用し、入力ドキュメントから教育的に正確なコンテンツ構造を抽出する。テキスト・図表・数式等を統合的に解釈し、教育意図（ペダゴジー）を維持したコンテンツ分析を実現する。

(2) **2段階 Generate-Verify-Optimize パイプライン**: コンテンツ整合フェーズとビジュアル最適化フェーズを分離した2段階構造を採用。生成されたHTMLコードを検証フェーズで教育的正確性・インタラクション品質の観点で評価し、その結果を最適化フェーズにフィードバックすることで品質を担保する。

(3) **Click-to-Locate編集 + Unified Diff ベースの増分生成**: ユーザーが画面上の要素をクリックして編集対象を特定できる「Click-to-Locate」機能と、差分ベース（Unified Diff）の増分コード生成を組み合わせることで、1回の編集イテレーションを10秒未満で完了させる。従来のフル再生成（200〜600秒）と比べ、20〜60倍の速度改善を達成している。

評価実験として、40名参加者によるコントロールドラボスタディを実施。MAIC-UIは直接的なText-to-HTML生成と比較して、編集イテレーション回数を平均7.0回から4.9回に削減し、学習可能性（learnability）と制御可能性（controllability）においても有意に優れた結果を示した。

さらに3ヶ月間の高校（53名）への教室展開実証では、MAIC-UIを使用したパイロットクラスがSTEM科目で+9.21点の成績向上を達成した一方、対照クラスは-2.32点と低下しており、学習エージェンシー（自律的学習姿勢）の促進と成績格差の縮小効果が確認された。コードはGitHub（THU-MAIC/MAIC-UI）で公開されている。

## アイデア

- Unified Diffベースの増分生成により編集サイクルを200〜600秒から10秒未満に短縮するアーキテクチャは、コード生成系AIエージェント全般における「部分更新」問題に応用可能であり、監査レポート自動生成における差分編集ループにも転用できる
- Generate-Verify-Optimizeの2段階パイプライン設計は「LLM-as-Judge」パターンの一形態であり、生成と検証を分離することで品質担保とトークン効率を両立する構造は、エージェント設計において汎用性が高い
- Click-to-Locate編集はUIの視覚要素とソースコードの特定箇所を紐付けるグラウンディング技術であり、マルチモーダルLLMがDOM構造や画面座標からコード位置を特定するアプローチは、UIオートメーションエージェント研究と密接に関連する
