## 要約

著者が「SuccuLM（Succulent Language Model）」と名付けたミニ言語モデルをゼロから構築した実践記録。目的は、窓辺で育てている多肉植物キャラクター（Winy と Ruby）との会話ができるGPT-2風モデルの実装である。

データセットはTinyStories（arXiv:2305.07759）の手法に倣い、154語の限定語彙と厳密な世界観制約（舞台は窓辺の植木鉢のみ、登場キャラは2体のみ）のもとでGemini 3 FlashとClaude Sonnet 4.6を使って生成した。Gemini 3 Flashは複数回制約を破ったため手作業で修正・一貫性チェックが必要だったが、Claude Sonnet 4.6はCode execution toolをサンドボックス内で自律的に活用して制約チェックを行い、完全に制約を守ったデータを生成した（その分トークン消費は膨大）。

データパターンは「質問＆応答（1文＋1文）」と「ポエム的発話（通常文2文）」の2種類で、各キャラクターの性格差（WinyはBecause/sometimes等の論理・限定表現、Rubyはvery/always等の強調・断定）が統計的に現れるよう設計されている。

単語埋め込みにはGloVe（2024 Wikipedia + Gigaword 5、11.9Bトークン、50次元）を使用し、事前学習済み埋め込みを転用することで語彙の表現力を確保した。

モデルアーキテクチャはPyTorchで実装したGPT-2風のTransformerデコーダー。入力はBOS+トークン列+EOSの形式で、1トークンずらした次トークン予測（CrossEntropyLoss）で訓練する標準的な自己回帰構造。複数の設定プロファイル（n_layers、ffn_expansion等のハイパーパラメータ組み合わせ）を定義して比較実験を行っており、チェックポイント保存・復元の仕組みも実装している。最大系列長は29トークン。

監査エージェント開発への直接的な示唆は薄いが、「厳密な語彙・世界観制約のもとでLLMにデータ生成させる」手法はドメイン固有のファインチューニングデータ作成にそのまま応用可能。特にClaude Sonnet 4.6がサンドボックス内で制約検証ツールを自律構築・実行した点は、エージェント的なデータ品質管理の実例として注目に値する。

## アイデア

- Claude Sonnet 4.6がCode execution toolをサンドボックス内で自律的に開発・実行して制約チェックを行った点：データ生成エージェントとしてのLLMの品質担保手法として実用的
- 154語という超限定語彙でキャラクター性格差を統計的に設計する手法：ドメイン固有の小規模LLMやキャラクターLMのデータ設計に応用可能なフレームワーク
- GloVeの事前学習済み50次元埋め込みをミニモデルの初期化に使う構成：語彙数が少ない特化型モデルでも汎用埋め込みを転用することで表現力を補完できる設計判断
