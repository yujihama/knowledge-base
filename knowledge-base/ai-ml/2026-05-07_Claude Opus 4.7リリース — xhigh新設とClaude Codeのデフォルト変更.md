## 要約

Anthropicは2026年4月16日にClaude Opus 4.7をリリースした。主な変更点は以下の通り。

【コーディング性能の向上】SWE-bench Pro 64.3%（前世代比+10.9ポイント）、SWE-bench Verified 87.6%（+6.8ポイント）、CursorBench 70%（+12ポイント）と全指標で改善。特に視覚推論ベンチマークCharXivで+13.0ポイント（82.1%）、コンピュータ操作OSWorldで+5.3ポイント（78.0%）と大きな伸びを示した。SWE-bench Verifiedは実際のGitHub Issue修正に近い設定のため、実用上の改善として体感しやすい指標とされる。

【xhighエフォートレベルの新設】従来はlow / medium / high / maxの4段階だったエフォートレベルに、highとmaxの間に位置する「xhigh（extra high）」が追加された。推論トークン上限の目安は約100kで、Opus 4.6のmax（上限200k）より少ないトークン消費でmax相当以上のスコアを実現する設計。実測では約71%のスコアを記録している。

【Claude Codeのデフォルト変更】全プランでClaude Codeのデフォルトエフォートレベルがxhighに変更された。従来max品質を求める際は明示的に指定が必要だったが、今後は無指定でも同等以上の出力が返る。一方でProプランユーザーはトークン消費増加に注意が必要。合わせて/ultrareviewコマンドが追加され、Pro/MaxプランでPRのレビュー特化機能を月3回無料で利用できる。

【ビジョン性能の引き上げ】画像入力の対応解像度が約1.15メガピクセルから最大3.75メガピクセル（長辺2,576ピクセル）に拡張。一般的なノートPCのフルスクリーンショットをリサイズなしで渡せるレベルに達したため、エラー画面のスクショをそのままエージェントに入力するワークフローが現実的になった。

【コスト面の注意点】API価格は入力$5/百万トークン、出力$25/百万トークンとOpus 4.6から据え置き。ただしトークナイザーが更新され、同一テキストでも最大1.35倍程度トークン数が増加するケースがある。特にJSON等の構造化データで影響が大きい。実質コストは1〜3割増となる可能性があるため、移行直後はモニタリングが推奨される。セッション全体のトークン支出を管理できる「タスク予算ベータ」も同時導入された。

## アイデア

- xhighは「max未満のトークン消費でmax相当以上の品質」を実現する設計で、推論効率の改善がデフォルト体験に直結する点が興味深い。トークン予算と出力品質のトレードオフを明示的に制御できる粒度が増えた
- ビジョン解像度の3.75MP対応により、エラー画面スクショをそのままエージェントに渡すワークフローが実用化できる。監査エージェントがUIベースのシステムをスクリーンショット経由で検証するユースケースに直結する
- トークナイザー更新による最大1.35倍のトークン増は、JSON構造化データを多用するエージェント・パイプラインで無視できないコスト変動要因。タスク予算ベータによるセッション単位キャップ設定が長時間稼働エージェントの運用管理に有効
