## 要約

OllamaはローカルでLLMを実行するためのオープンソースツールで、macOS・Linux・Windowsに対応している。ChatGPTのような外部サービスと異なり、入力データがサーバーに送信されず、インターネット接続なしで動作し、月額費用も不要。インストール方法は2種類あり、公式サイトからGUIアプリ（Ollama.dmg）をダウンロードするか、Homebrewで `brew install ollama` を実行する。起動後はメニューバーに🦙アイコンが表示され、`curl http://localhost:11434/api/tags` でバックグラウンド動作を確認できる。モデルは `ollama pull llama3` でダウンロード（約4.7GB）し、`ollama run llama3` で対話的に実行できる。対応モデルはMeta社のLlama3、Alibaba社のQwen（日本語処理に強い）、Mistral社のMistralなど多数で、いずれも無料のオープンソース。メモリ要件は8GB以上推奨（16GB理想）、ディスク空き容量は最低10GB必要。PythonからはHTTP API（`http://localhost:11434/api/generate`）経由で利用でき、`requests` ライブラリで `stream: False` の同期呼び出しと `stream: True` のストリーミング呼び出しの両方に対応する。CLIでは `ollama run llama3 "質問文"` のワンショット実行も可能で、シェルスクリプトや標準入力からのパイプ処理にも対応している。監査エージェント開発への応用としては、社内機密データや監査調書をローカルLLMで処理することでデータ漏洩リスクを排除しつつ、LangGraphなどのフレームワークとHTTP APIで統合することで外部依存なしのエージェントシステムを構築できる。

## アイデア

- HTTP API（localhost:11434）経由でLangGraphやPydanticエージェントと統合することで、機密データを外部送信せずにReActエージェントを構築できる
- Qwen等の日本語特化モデルをOllamaで管理し、監査調書の自動要約・分類パイプラインをオフラインで完結させる構成が可能
- ストリーミングAPI（stream: True）を使ったリアルタイムレスポンス表示は、監査インタビュー支援ツールのUX改善に応用できる
