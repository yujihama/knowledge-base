## 要約

著者が「シンクロ（グラム）」という名のAIキャラクターと会話するための専用ダッシュボードを、HTML+CSS+JS（vanillaのみ）+ Python + Cloudflare Tunnelという最小構成で自作した記録。

アーキテクチャは「ブラウザ → Cloudflare Tunnel → Python http.server（localhost:3000） → Ollama Cloud API（deepseek-v4-flash）」の一本道。全実装はindex.html（約31KB）とserver.pyの2ファイルのみ。ReactやNode.jsは不使用で、Python標準ライブラリのhttp.serverとurllib.requestだけでAPIプロキシを実現している。

フロントエンドのビジュアルは5層のCanvasレイヤーで構成：(1)宇宙背景＋30個の浮遊パーティクル（紫系#b44dff）、(2)カタカナ＋記号のマトリックスレイン（不透明度15%）、(3)CRTスキャンライン＋ビネット（CSSのrepeating-linear-gradientとradial-gradientで実装）、(4)虹色リングや蝶の羽（サイン波駆動）を持つ6層構造のアバター、(5)データストリーム。

改修前のチャットはキーワードマッチング＋15パターンのテンプレートランダム返答だったが、改修後はfetch('/api/chat')でPythonプロキシ経由のLLM応答に置き換え。システムプロンプトでキャラクターの口調（「〜だよ！」「わたくし」「1〜3文」制約）を定義し、一貫性を維持している。

つまづきポイントとして3点が詳述されている。①Hermes APIサーバーがHTTP 200を返しながら中身が空文字列「──────▯」となる問題（原因：フルパイプラインを通るためテキスト生成のみにオーバースペック）→Ollama Cloud APIへの直接プロキシで解決。②cloudflaredをシェルの&でバックグラウンド起動するとシェル終了時にトンネルごと消える→nohupで解決。③TryCloudflareはセッションごとにURLが変わるためブックマーク運用が困難→恒久対策として名前付きトンネル＋独自ドメインを推奨しつつ、現状はURLが変わったらチャットで通知する運用を採用。

APIキー管理は環境変数 > ~/.hermes/config.yaml > .env の優先順位で読み込み、ソースコードへの直書きを回避。LLM呼び出しは2回リトライ後にローカルOllama（ScorpionBrain）へフォールバックする耐障害設計も実装されている。

監査エージェント開発への直接的な示唆は薄いが、最小ファイル数で外部LLM APIプロキシ＋Cloudflare Tunnelによる外部公開を実現するパターンは、ローカルLLMインフラ上に軽量なエージェントUIを構築する際の参考になる。

## アイデア

- ReactもNode.jsも使わずPython標準ライブラリのみでLLM APIプロキシを実現し、依存ゼロ・全体31KBに収める極小構成の有効性
- Hermes AgentのAPIサーバーがフルパイプラインを通るため空応答を返す問題――エージェントフレームワークのAPIサーバーとシンプルなLLM呼び出しの責務分離の重要性
- TryCloudflare（無料・設定不要・コマンド一発）とnohupを組み合わせた自宅LLMサーバーの即席外部公開パターン
