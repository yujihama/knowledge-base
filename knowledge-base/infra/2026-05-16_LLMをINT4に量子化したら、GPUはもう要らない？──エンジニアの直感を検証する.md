## 要約

LLMの重みをFP16からINT4に量子化するとモデルサイズが約1/4になる。この事実から「整数演算ならCPUで十分では？」という疑問を検証した記事。結論は「GPUは不要にはならないが、ハードウェア選択の自由度が大幅に広がる」。

LLM推論のボトルネックは計算量ではなくメモリ帯域にある。70Bモデルの場合、BF16では毎トークン生成ごとに140GBの重みをメモリから読み出す必要があり、INT4化でもこれが35GBになるだけで、依然として大量のデータ転送が発生する。CPUのメモリ帯域は数十GB/s程度であるのに対し、H100などのハイエンドGPUは数TB/sと約2桁の差がある。この帯域差がそのままスループット差に直結するため、GPUの優位性は量子化後も維持される。

GPU側では、Volta世代以降のNVIDIA TensorCoreがINT8・INT4・FP8の低精度整数行列乗算を直接加速できる。量子化はGPUを無効化するどころか、GPUのINT4専用回路を最大活用できる形態といえる。実測では、GPTQ-INT4化によりBF16比で2.69倍のスループット向上が報告されており、これはメモリ帯域削減の効果が直接現れたものだ。

CPU側では、llama.cpp・OpenVINO・ONNX RuntimeがAVX2・AVX-512・AVX-512 VNNIなどのSIMD命令を活用してINT8推論を実用化している。ただしINT4については8bit未満の整数演算に対するハードウェア命令サポートが不十分で、性能向上は計算高速化ではなくデータ量削減による間接的な効果にとどまる。7B〜13Bモデルであれば、llama.cppを使ったCPU単体運用でも実用速度が得られる。

第3の選択肢としてNPU（Neural Processing Unit）が台頭している。Intel Core Ultra・AMD Ryzen AI・Apple Neural EngineなどCPU統合型のNPUはINT8/INT4低精度演算に特化し、電力効率が高く、CPUと密結合しているためデータ転送オーバーヘッドが小さい。量子化技術の進化とNPUの普及は相互に加速する関係にあり、将来的にはスマートフォンで70Bモデルが動作する可能性も視野に入る。

監査エージェント開発への示唆：エッジ環境や社内オンプレ環境でのローカルLLM推論において、INT4量子化＋NPU/CPU構成はコスト・セキュリティ面で有力な選択肢になる。GPU調達が難しい環境でも、量子化済みモデルをllama.cppで動かす構成が現実的な代替となりうる。

## アイデア

- LLM推論のボトルネックは計算量ではなくメモリ帯域であり、量子化の本質的な効果は「計算の高速化」ではなく「転送データ量の削減」にある
- NVIDIAのTensorCoreはINT4演算も加速できるため、量子化はGPUを排除するのではなくGPUのINT4専用回路を活かす方向に働く
- Intel/AMD/Apple製NPUの台頭により、GPU・CPU・NPUという3択のハードウェア構成が現実化しており、量子化技術はこれら全プラットフォームへのLLM展開を可能にする共通技術として機能する
