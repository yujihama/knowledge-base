## 要約

研究室配属直後の学部生が感じた「論文にたどり着くまでのハードルの高さ」を解決するために開発されたWebサービスの紹介記事。英語の壁・abstractの難解さ・数式への苦手意識という三重のハードルにより、従来は「タイトルと図だけ見て終わる」状態が常態化していたという問題意識から出発している。LLMによるabstractの1件ずつの翻訳も試みたが、「読んでから判断する」という非効率さは解消されず、「読む前に読む価値を判断できる場所」の必要性を痛感したことが開発動機となっている。

構築したサービス「ML Paper Portal」（https://ml-paper-portal-web.pages.dev/）は、機械学習系論文を日本語の短い導入文付きで一覧表示し、新着論文チェック・タグ別閲覧（LLM/GNN/Transformerなど）・ランダム閲覧・原論文へのリンクという4機能を提供する。「読むかどうかを決めるための入口」として機能することを設計思想の中心に置いている点が特徴的。

技術スタックはフロントエンドにCloudflare Pages、バックエンドにCloudflare Workers＋D1（SQLiteベースのエッジDB）、論文メタデータの取得源にOpenAlex API、日本語要約生成にOpenAI APIを採用。エッジコンピューティング基盤上で論文取得・要約生成・配信を完結させる構成であり、サーバーレス・低コスト運用を実現している。

対象ユーザーは研究室配属直後の学部生を主ターゲットとしつつ、論文読解が苦手な社会人や隙間時間でのインプット層にも対応。初心者向けに「どこを見るか・どう判断するか」をまとめたガイドページも別途用意している。

監査エージェント開発への示唆：OpenAlex APIのような公開学術APIとOpenAI APIを組み合わせた「ドメイン特化の情報前処理パイプライン」は、監査領域においても規制文書・通達・判例の日本語要約・タグ付けパイプラインとして応用可能。Cloudflare Workers＋D1によるエッジDB構成は、社内監査ナレッジベースの軽量デプロイにも参考になる設計パターンである。

## アイデア

- 「読む前に読む価値を判断する」という設計思想：論文発見フローをプッシュ型に変換し、LLMによる日本語要約をフィルタリング層として位置づけることで、情報過負荷問題を解決するUX設計
- OpenAlex API + OpenAI API の組み合わせによる論文メタデータ自動収集・要約パイプライン：同パターンはドメイン特化（監査・法令・医療）の知識ベース自動構築にそのまま転用可能
- Cloudflare Pages + Workers + D1 のフルエッジ構成：サーバーレスかつ低コストで読み取り頻度の高いコンテンツを配信するアーキテクチャとして、RAGシステムのフロントエンド層設計の参考になる
