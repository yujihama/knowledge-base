## 要約

Anthropic Python SDK における`messages.stream`を使ったストリーミング実装を解説した入門記事。通常の`messages.create`との違いを「全文待機 vs リアルタイム表示」として整理し、ストリーミングが持続的なHTTP接続を維持する仕組みを説明。Pythonのコンテキストマネージャ（`with`文）を使うことで、正常終了・例外発生どちらの場合も接続が確実にクローズされることを強調。`stream.text_stream`イテレータで断片テキストを受け取り、`print(text, end="", flush=True)`で即時出力する実装パターンを示す。完了後に`stream.get_final_message()`でトークン使用量も取得可能。シリーズ構成はテキスト生成→マルチターン→ストリーミング→Function Calling→RAGの順。

## 要点

- `messages.create`を`messages.stream`に置き換えるだけでストリーミングが有効になる
- ストリーミングは接続を保持し続けるため、`with`文によるリソース管理が必須
- 完了後に`stream.get_final_message()`でトークン数を取得でき、コスト監視に活用可能
