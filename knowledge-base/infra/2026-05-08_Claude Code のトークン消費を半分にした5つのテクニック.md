## 要約

Claude Code を使う際のAPIコスト（トークン消費）を約50%削減した実践的な5つの手法を紹介する記事。コスト爆発の主因として、ファイル全体の毎回送信・システムプロンプトの肥大化・高価なモデルの過剰利用・キャッシュ未活用の4パターンを挙げている。

【テクニック1: 差分のみ送信】`git diff -U3 HEAD~1 HEAD` で変更箇所だけを抽出してClaudeに渡す。1000行ファイルで20行の変更なら送信トークンを最大98%削減可能。削減効果は60〜70%。

【テクニック2: システムプロンプト最小化】毎回500トークンのシステムプロンプトを送ると100リクエスト/日で月150万トークンの浪費になる。`.claude/CLAUDE.md`にプロジェクト規約を記載しておくことでAPIへの重複送信を回避。500トークン→50トークンへ削減で月135万トークン削減。

【テクニック3: モデルルーティング】タスク複雑度に応じてHaiku（$0.80/1Mトークン）・Sonnet（$3.00）・Opus（$15.00）を使い分ける。変数名修正や軽微なフォーマット修正はHaikuで十分。タスクの50%をHaikuに移行するだけでコストを約35%削減できる。

【テクニック4: Prompt Caching】Anthropic APIのPrompt Caching機能を使い、`cache_control: {"type": "ephemeral"}`を指定することでキャッシュヒット時にInputトークン料金が約90%オフになる。1024トークン以上・5分以内の再利用が条件。100トークンのプロンプトを100回送る場合、通常10,000トークン課金が1,900トークン相当に（81%削減）。

【テクニック5: バッチリクエスト】Anthropic の Message Batches APIを使うと通常の50%引きで処理可能。複数ファイルを個別リクエストではなく1リクエストにまとめることでオーバーヘッドも削減。大量処理タスクで40〜60%のコスト削減が見込める。

Before/Afterの試算では月$80の利用が約$38（52%削減）になる。監査エージェント開発への示唆として、コードレビューや差分チェックを繰り返すReActエージェントにPrompt CachingとモデルルーティングをかけたLangGraphノード設計が有効であり、Haikuで前処理・分類を行い、Sonnetで判断・出力する二段構成がコスト効率に優れる。

## アイデア

- Prompt Cachingの条件（1024トークン以上・5分以内再利用）を満たすようシステムプロンプトを設計することで、繰り返しパターンの強いエージェントループでは入力コストを最大90%カットできる
- タスク複雑度を自動分類してHaiku/Sonnet/Opusにルーティングするディスパッチャー層を設けることで、エージェントの品質を落とさずコストを35%以上削減できる設計パターンが実用的
- Message Batches APIはリアルタイム性不要な監査バッチ処理（大量ドキュメントのリスク分類・コメント自動生成）に最適で、通常APIの半額で非同期実行できる
