## 要約

MOPAC2016はかつて商用ライセンスが必要だった半経験的分子軌道法ソフトウェアだが、Apache License 2.0のもとでオープンソース化され、conda-forgeから`conda install mopac`一発でインストール可能になった。本記事はその第1回として、Python（RDKit + subprocess）を使った量子化学計算の完全自動化フローを解説する。

技術的な核心は4ステップのパイプライン構成にある。①RDKitのETKDGv3アルゴリズムで50個の配座（Conformer）を生成し、MMFF力場で粗い最適化を行って最安定配座を選抜する。②選抜した配座からMOPAC入力ファイル（.mop）を生成する際、「連結ジョブ（Linked Job）」機能を活用し、`&`区切りで「PM7 PRECISE STATIC EF（構造最適化）」と「PM7 PRECISE FORCE OLDGEO（振動計算）」の2段階計算を1ファイルにまとめる。③Pythonのsubprocessモジュールでmopacコマンドを呼び出す。④出力ファイル（.out）を正規表現で解析し、HOMO/LUMOエネルギー（eV）を抽出する。HOMO LUMO ENERGIESが直接見つからない場合、KoopmansのHOMO≒−IP近似でイオン化ポテンシャルから推定するフォールバックロジックも実装している。

PM7（Parametric Method 7）はPM6に比べ水素結合・分散力（ファンデルワールス力）の記述が改善されており、有機結晶や生体分子など非共有結合性相互作用が重要な系でも信頼性が高い。ab initio計算（Gaussian等）が指数関数的な計算コストを要するのに対し、半経験的手法は積分の一部を実験値パラメータで置換することで劇的な高速化を実現しており、大量分子スクリーニングや機械学習用記述子生成に適している。

フェノール（SMILES: Oc1ccccc1）を例題とし、SMILES文字列を入力するだけでHOMO-LUMOエネルギーを自動算出する完全なrun_mopac.pyを提示。連載は全3回で、第2回では機械学習向け大量分子データ生成と記述子算出、第3回ではPySide6によるGUIアプリ化を扱う予定。

監査AIエージェント開発への直接的な示唆は薄いが、「外部ツール（専門計算エンジン）をsubprocessで呼び出し、出力をパース・構造化する」設計パターンはエージェントのツール統合設計と同型であり、ツール呼び出し→出力解析→次ステップへの引き継ぎというフローはReActエージェントのツール使用サイクルと類似している。

## アイデア

- 連結ジョブ（Linked Job）パターン：1ファイル内に`&`区切りで複数計算ステップを定義し、前段の最適化済み構造をOLDGEOで引き継ぐ設計は、LLMエージェントのマルチステップツール呼び出しと同型の「状態引き継ぎ付きパイプライン」として汎用的に参照できる
- フォールバック付き出力解析：HOMO/LUMO直接抽出が失敗した場合にKoopmansの定理（HOMO≒−IP）でイオン化ポテンシャルから推定する堅牢化ロジックは、LLMの構造化出力解析における「プライマリパース失敗時の代替抽出戦略」として参考になる
- conda-forgeによる専門ソフトウェアのパッケージ化：商用ツールのオープンソース化と標準パッケージマネージャへの統合が、専門ドメイン計算の民主化を一行インストールで実現する事例として、AI/MLツールチェーン整備の文脈と対比できる
