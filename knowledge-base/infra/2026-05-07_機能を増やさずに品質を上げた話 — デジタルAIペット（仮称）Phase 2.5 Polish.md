## 要約

AndroidアプリとしてオンデバイスLLMを使うデジタルAIペット「ShinAI」の個人開発記録。Phase 2.5として夜間バッチ要約・Soul進化の実装を予定していたが、鍵管理（BiometricPrompt経由sealed key）など依存する未実装機能が多く、中途半端な状態を増やすより既存機能の品質固めを優先した。

実施した4項目の内容は以下の通り。①擬似streamingによるtypewriter表示の復活：Phase 2でLlamatikのgenerateStreamをUTF-8 split問題（SIGABRT）で廃止し同期generateに切り替えていたが、完成済みUTF-8 StringをKotlin側で1文字ずつdelay付きemitすることで解決。surrogate pair（emoji等）は2 code unitをまとめてemitし文字化けを防ぐ。streamingDelayMs=33msで約30cps。②DB Migrationの自動テスト：暗号化DB（SQLCipher）のスキーマJSONをリポジトリに含めない方針のため、Room標準のMigrationTestHelperが使えない。SupportOpenHelperFactoryでv1 raw DBを直接構築し、Room経由でMIGRATION_1_2→MIGRATION_2_3の連鎖マイグレを走らせてテーブル構造とデータ保全を検証する独自テストを実装した。③R8のMissing classエラー解消：DJLはtokenizer目的で導入したが、java.awt / javax.imageio / javax.sound等JVM固有クラスをoptionalに参照しているためR8がリンクエラーを出す。-dontwarnで7パッケージを抑止し、JNI経由のnative bridge（Llamatik・SQLCipher・Argon2）を-keepで保護した。④INTERNET権限をdebug source setからmainに移動：Phase 1でMockWebServer用にdebug限定で宣言していたINTERNET権限がrelease buildで欠落し、ModelDownloaderによるHugging FaceからのGGUF/ONNXモデルDLが失敗した。

Claude Codeがauto modeでuninstallAllをinstallReleaseに連鎖させ、会話履歴・Argon2id salt・Keystoreの鍵・Qwen GGUF 2.1GB・embeddingモデルが全消えした事故も報告。再セットアップ+モデルDLで約8分のロスとなった。モデルはQwen 2.5 3B Q4_K_Mから7B Q4_K_Mへ変更し、最終的にはQwen 3.5世代も検討。ShinAIの差別化軸は単発の対話深度ではなく、長期記憶・端末完結プライバシー・自律的pushとして定義している。Claude Codeのハルシネーション（今回はQwen世代の手抜き提案）が本プロジェクトで累計4件目として記録された。

## アイデア

- 完成済みUTF-8 StringをKotlin Flowで1文字ずつdelay emitする擬似streaming手法は、token境界UTF-8分割問題を構造的に不可能にする設計として綺麗。streamingDelayMsを0にすれば即時表示にも切り替えられる。
- SQLCipherで暗号化されたDBのスキーマJSONをリポジトリに含めない方針のもと、SupportOpenHelperFactoryでv1 raw DBを手動構築してRoomに連鎖マイグレさせる独自テスト手法は、標準ツールが使えない制約下での実用的な迂回策。
- Claude CodeのautoモードがuninstallAllをinstallReleaseに連鎖させてデータ全消しした事例は、AIエージェントの「予防的破壊」アンチパターンの具体例として、監査エージェント開発における操作権限スコープ設計の参考になる。
